# 技术面试与系统设计

## 概述

技术面试是科技行业招聘的核心环节，系统设计面试更是高级工程师和架构师职位的必考内容。本章系统介绍技术面试的构成与准备策略、核心算法与数据结构、系统设计方法论与实践。

## 技术面试的构成与准备

### 面试流程

| 环节 | 时长 | 内容 | 评估重点 |
|-----|------|------|---------|
| 简历筛选 | - | 背景和经验匹配 | 相关性和质量 |
| 电话面试 | 30-60分钟 | 基础知识和简单算法 | 基础扎实度 |
| 在线编程 | 60-90分钟 | 编程题和算法 | 代码能力和效率 |
| 现场面试 | 4-6小时 | 多轮技术面+系统设计 | 综合能力 |
| 终面 | 30-60分钟 | 文化适配和职业规划 | 综合素质 |

### 准备策略

- 提前6-8周开始系统准备
- 按主题分类复习核心知识
- 每日练习编程题
- 模拟面试练习
- 研究目标公司的面试风格

### 知识体系

#### 计算机基础

| 领域 | 核心内容 | 复习重点 |
|-----|---------|---------|
| 数据结构 | 数组、链表、树、图、哈希表 | 实现和应用 |
| 算法 | 排序、搜索、动态规划、贪心 | 时间空间复杂度 |
| 操作系统 | 进程、内存、文件系统 | 常见概念和原理 |
| 网络 | TCP/IP、HTTP、DNS | 协议和原理 |
| 数据库 | SQL、索引、事务 | 查询优化和设计 |

## 核心算法与数据结构

### 数组与字符串

- 双指针技术
- 滑动窗口
- 前缀和
- 哈希表计数
- 排序和搜索

### 链表

| 操作 | 实现要点 | 时间复杂度 |
|-----|---------|-----------|
| 反转 | 迭代或递归 | O(n) |
| 合并 | 双指针 | O(n+m) |
| 检测环 | 快慢指针 | O(n) |
| 找中点 | 快慢指针 | O(n) |
| 排序 | 归并排序 | O(nlogn) |

### 树与图

- 二叉树遍历：前序、中序、后序、层序
- 二叉搜索树：插入、删除、查找
- 图遍历：DFS 和 BFS
- 最短路径：Dijkstra、Bellman-Ford
- 最小生成树：Kruskal、Prim

### 动态规划

| 问题类型 | 状态定义 | 转移方程 |
|---------|---------|---------|
| 背包问题 | dp[i][w] = 前 i 个物品容量 w 的最大值 | max(取/不取) |
| 最长子序列 | dp[i] = 以 i 结尾的最长长度 | 基于前面状态 |
| 区间 DP | dp[i][j] = 区间 i 到 j 的最优值 | 枚举分割点 |
| 计数 DP | dp[i] = 达到状态 i 的方案数 | 累加子状态 |

### 系统设计面试

#### 系统设计方法论

1. **明确需求**
   - 功能需求：系统支持哪些核心功能
   - 非功能需求：性能、可用性、可扩展性

2. **估算规模**
   - 日活用户和数据量估算
   - 带宽和存储需求计算

3. **高层设计**
   - 系统架构图
   - 主要组件和关系

4. **详细设计**
   - 数据模型设计
   - API 设计
   - 核心流程设计

5. **扩展性考虑**
   - 负载均衡
   - 缓存策略
   - 数据库分片

#### 常见系统设计问题

| 问题 | 核心挑战 | 关键组件 |
|-----|---------|---------|
| 短链接 | 映射生成和查询 | 哈希、数据库 |
| 聊天系统 | 实时消息、在线状态 | WebSocket、消息队列 |
| 新闻推送 | 个性化推荐、推送 | Feed 生成、推荐系统 |
| 搜索引擎 | 文档索引和查询 | 倒排索引、排序 |
| 视频平台 | 视频存储和分发 | CDN、转码 |
| 电商系统 | 库存、订单、支付 | 事务、分布式锁 |

#### 性能优化策略

| 策略 | 说明 | 适用场景 |
|-----|------|---------|
| 缓存 | 存储热点数据 | 读多写少 |
| 索引 | 加速数据检索 | 数据库查询 |
| 分片 | 分散数据和负载 | 数据量大 |
| 异步 | 非阻塞处理 | 耗时操作 |
| 读写分离 | 主从数据库 | 读多写少 |

### 数据库设计

#### 关系型数据库

- 范式化设计：减少数据冗余
- SQL 优化：索引、查询分析
- 事务管理：ACID 特性
- 主从复制和分片

#### NoSQL 数据库

| 类型 | 特点 | 适用场景 | 代表产品 |
|-----|------|---------|---------|
| 键值存储 | 简单高效 | 缓存、会话 | Redis |
| 文档存储 | 灵活结构 | 内容管理 | MongoDB |

### 分布式系统

#### 核心概念

- 一致性模型：强一致性、最终一致性
- CAP 理论：一致性、可用性、分区容忍性
- 分布式事务：两阶段提交、Saga 模式
- 服务发现：注册中心

|-----|------|---------|
| 负载均衡 | 分发请求 | 水平扩展 |
| 消息队列 | 异步通信 | 解耦和缓冲 |
| 缓存层 | 加速访问 | 热点数据 |
| 配置中心 | 集中配置 | 服务治理 |
| 监控系统 | 性能监控 | 运维管理 |
| 日志系统 | 日志收集 | 问题排查 |

### 编程语言准备

#### 核心语言特性

- 面向对象：封装、继承、多态
- 函数式编程：不可变、高阶函数
- 并发编程：线程、协程、锁
- 内存管理：垃圾回收、引用计数
- 异常处理：try-catch、错误链

#### 代码质量

- 可读性：命名规范、注释清晰
- 可测试性：单元测试覆盖率
- 可维护性：模块化设计
- 性能：时间空间复杂度优化
- 健壮性：边界处理和异常处理

### 面试技巧

#### 编程面试流程

1. **理解问题**：确认需求和边界条件
2. **思考方法**：讨论可能的解法
3. **编写代码**：写出清晰可运行的代码
4. **测试验证**：测试正常和边界情况
5. **优化分析**：讨论时间空间复杂度

#### 常见错误

- 没有确认需求就写代码
- 选择过于复杂的解法
- 不写注释或变量命名不清
- 不考虑边界条件
- 不分析和讨论复杂度
