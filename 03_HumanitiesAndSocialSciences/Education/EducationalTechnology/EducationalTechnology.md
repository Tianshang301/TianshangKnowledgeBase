# 教育技术学

## 一、教育技术的定义与演进

### 1.1 AECT 定义

美国教育传播与技术协会（AECT）历次定义：

| 年份 | 定义 | 关键词 |
|------|------|--------|
| 1963 | 视听传播是教育理论与实践的一个分支 | 视听传播 |
| 1970 | 教育技术是根据具体目标，设计、实施和评价学与教全过程的系统方法 | 系统方法 |
| 1972 | 教育技术是设计、实施和评价学与教全过程的系统性方法 | 系统化 |
| 1977 | 教育技术是分析问题，设计和实施解决方案，评价结果的方法论 | 问题解决 |
| 1994 | 教育技术是为了促进学习，对有关的过程和资源进行设计、开发、利用、管理和评价的理论与实践 | 五个范畴 |
| 2008 | 教育技术是通过创造、使用和管理适当的技术性过程和资源，以促进学习和提高绩效的研究与符合伦理道德的实践 | 伦理、绩效 |
| 2017 | 教育技术是促进学习和改善绩效的研究与符合道德的实践 | 简化、道德 |

### 1.2 教育技术发展阶段

$$\text{教育技术演进} = \text{视觉教学} \rightarrow \text{视听教学} \rightarrow \text{视听传播} \rightarrow \text{教育技术}$$

- **视觉教学时代**（1900s-1930s）：幻灯片、无声电影
- **视听教学时代**（1930s-1960s）：有声电影、录音、电视
- **视听传播时代**（1960s-1970s）：系统方法引入
- **教育技术时代**（1970s-至今）：计算机、互联网、人工智能

## 二、学习理论与技术

### 2.1 行为主义与技术

行为主义学习理论强调刺激-反应联结，技术应用于：

| 技术类型 | 原理 | 应用 |
|---------|------|------|
| 程序教学 | 斯金纳的操作性条件反射 | 小步调、即时反馈、自定步调 |
| 计算机辅助教学（CAI） | 强化与塑造 | 练习与操练软件 |
| 教学机器 | 程序化学习材料呈现 | 早期程序教学机 |

**局限**：忽视认知过程与理解深度

### 2.2 认知主义与技术

认知主义关注信息加工过程：

- **智能导学系统**（ITS）：诊断学生知识状态，提供个性化指导
- **认知工具**：支持思维过程的外化与操作
- **信息加工模型**：注意、编码、存储、提取的技术支持

### 2.3 建构主义与技术

建构主义强调学习者主动建构知识：

| 技术环境 | 原理 | 示例 |
|---------|------|------|
| 微世界（Microworlds） | 在探索中发现规律 | Logo 编程、几何画板 |
| 认知学徒制 | 专家示范、支架式支持 | 情境化学习软件 |
| 协作学习环境 | 社会协商建构知识 | CSCL 平台 |
| 脚手架工具 | 逐步撤除支持 | 提示系统、模板 |

### 2.4 联通主义与技术

乔治·西蒙斯（George Siemens）提出联通主义：

$$\text{学习} = \text{连接节点} + \text{形成网络}$$

- **MOOCs**（大规模开放在线课程）：连接全球学习者
- **个人学习网络**（PLN）：利用社交媒体构建专业学习网络
- **知识管理**：在信息过载时代，学习即管道比管道中的内容更重要

## 三、教学设计模型

### 3.1 ADDIE 模型

ADDIE 是经典教学设计框架：

| 阶段 | 任务 | 产出 |
|------|------|------|
| Analysis（分析） | 需求分析、学习者分析、内容分析 | 分析报告 |
| Design（设计） | 目标设定、策略选择、媒体选择 | 设计方案 |
| Development（开发） | 材料制作、课件开发 | 教学材料 |
| Implementation（实施） | 教学交付、技术支持 | 教学实施 |
| Evaluation（评价） | 形成性评价、总结性评价 | 评价报告 |

### 3.2 迪克-凯瑞模型（Dick and Carey）

系统化教学设计流程：

1. 确定教学目标
2. 进行教学分析
3. 分析学习者与情境
4. 编写绩效目标
5. 开发评价工具
6. 开发教学策略
7. 开发与选择教学材料
8. 设计与实施形成性评价
9. 修改教学
10. 设计与实施总结性评价

### 3.3 SAM 模型（Successive Approximation Model）

敏捷教学设计方法：

$$\text{SAM} = \text{快速原型} + \text{迭代开发}$$

- **准备阶段**：信息收集、认知启动
- **迭代设计阶段**：设计、原型、评审的循环
- **迭代开发阶段**：开发、实施、评估的循环

### 3.4 布鲁姆分类法修订版

安德森（Anderson）与克拉斯沃尔（Krathwohl）修订：

| 维度 | 层次 | 关键词 |
|------|------|--------|
| 认知过程 | 记忆、理解、应用、分析、评价、创造 | 动词描述 |
| 知识类型 | 事实性、概念性、程序性、元认知 | 名词描述 |

## 四、多媒体学习

### 4.1 梅耶的认知理论

理查德·梅耶（Richard Mayer）提出多媒体学习认知理论，基于三个假设：

1. **双通道假设**：人类拥有独立的视觉/图像通道和听觉/言语通道
2. **容量有限假设**：每个通道的信息加工容量有限
3. **主动加工假设**：学习者主动选择、组织和整合信息

### 4.2 多媒体学习十二原则

| 原则 | 内容 | 应用 |
|------|------|------|
| 多媒体原则 | 文字+图像优于纯文字 | 图文结合呈现 |
| 空间邻近原则 | 相关文字与图像空间邻近 | 减少视觉搜索 |
| 时间邻近原则 | 相关言语与图像时间同步 | 同步呈现解说与画面 |
| 一致性原则 | 去除无关材料 | 避免冗余装饰 |
| 通道原则 | 动画+解说优于动画+文字 | 利用双通道 |
| 冗余原则 | 避免文字重复解说内容 | 解说与文字二选一 |
| 个体差异原则 | 效果因学习者知识水平而异 | 考虑先备知识 |
| 信号原则 | 突出关键信息 | 使用提示、颜色、箭头 |
| 分段原则 | 按学习者节奏分段呈现 | 学习者控制进度 |
| 预训练原则 | 先教关键概念名称与特征 | 降低认知负荷 |
| 具体先行原则 | 以具体案例引入抽象概念 | 从具体到抽象 |
| 人声原则 | 使用真人语音而非机器语音 | 增强社会存在感 |

### 4.3 认知负荷理论

约翰·斯威勒（John Sweller）提出三种认知负荷：

$$\text{总认知负荷} = \text{内在负荷} + \text{外在负荷} + \text{关联负荷}$$

- **内在负荷**：任务本身的复杂性
- **外在负荷**：不良教学设计增加的负荷
- **关联负荷**：促进图式建构的有效负荷

## 五、电子学习

### 5.1 学习管理系统（LMS）

| 平台 | 特点 | 适用场景 |
|------|------|---------|
| Moodle | 开源、灵活、社区支持强 | 高等教育、K-12 |
| Canvas | 云端、用户体验好、移动友好 | 高校在线教学 |
| Blackboard | 功能全面、企业级支持 | 大型机构 |
| 雨课堂 | 微信生态、轻量级 | 混合式教学 |
| 学习通 | 资源丰富、直播功能强 | 高校大规模教学 |

### 5.2 同步与异步学习

| 维度 | 同步学习 | 异步学习 |
|------|---------|---------|
| 时间 | 实时 | 灵活 |
| 交互 | 即时反馈 | 深度思考 |
| 技术 | 视频会议、直播 | 论坛、邮件、录播 |
| 优势 | 社会存在感强 | 时间灵活、可反复学习 |
| 劣势 | 时间约束 | 缺乏即时互动 |

### 5.3 混合式学习与翻转课堂

**混合式学习**（Blended Learning）：

$$\text{混合式学习} = \text{在线学习} + \text{面对面学习}$$

翻转课堂（Flipped Classroom）模型：

| 传统课堂 | 翻转课堂 |
|---------|---------|
| 课堂：知识讲授 | 课前：视频自学 |
| 课后：作业练习 | 课堂：互动深化 |

## 六、教育数据挖掘与学习分析

### 6.1 教育数据挖掘（EDM）

EDM 方法与技术：

- **预测建模**：预测学生成绩、辍学风险
- **聚类分析**：识别学习者类型与行为模式
- **关系挖掘**：发现学习变量间的关系
- **模型发现**：构建学习过程模型

### 6.2 学习分析（Learning Analytics）

学习分析循环：

$$\text{数据收集} \rightarrow \text{分析} \rightarrow \text{可视化} \rightarrow \text{干预} \rightarrow \text{评估}$$

应用：
- **早期预警系统**：识别学业困难学生
- **学习仪表盘**：可视化学习进度与表现
- **个性化推荐**：基于学习行为推荐资源

### 6.3 预测模型

学生成功预测模型：

$$P(\text{成功}) = f(\text{出勤率}, \text{作业完成率}, \text{在线活动}, \text{先备成绩}, \text{社会经济背景})$$

## 七、人工智能与教育

### 7.1 智能导学系统（ITS）

ITS 四大组件：

| 组件 | 功能 | 技术 |
|------|------|------|
| 领域模型 | 表示学科知识结构 | 知识图谱、本体 |
| 学生模型 | 诊断学习者状态 | 贝叶斯网络、项目反应理论 |
| 教学模型 | 决策教学策略 | 规则系统、强化学习 |
| 界面模型 | 人机交互 | 自然语言处理、多模态 |

### 7.2 自适应学习

自适应学习系统特征：

- **诊断性评估**：实时评估学生知识掌握程度
- **个性化路径**：根据学生水平调整内容序列
- **动态难度调整**：题目难度适应学生能力
- **即时反馈**：针对性反馈与提示

代表系统：Knewton、DreamBox、松鼠 AI

### 7.3 自动评分与智能评价

| 类型 | 技术 | 应用 |
|------|------|------|
| 客观题自动评分 | 模式匹配 | 选择题、填空题 |
| 作文自动评分 | NLP、机器学习 | E-rater、PEG |
| 口语自动评分 | 语音识别 | 发音评估、流利度 |
| 代码自动评分 | 程序分析 | 编程作业评测 |

### 7.4 教育聊天机器人

教育聊天机器人功能：

- **答疑助手**：回答学生常见问题
- **学习伙伴**：陪伴式学习支持
- **语言练习**：对话式语言学习
- **行政服务**：选课、查成绩、通知

## 八、虚拟现实与增强现实

### 8.1 VR/AR 教育应用

| 技术 | 特征 | 教育应用 |
|------|------|---------|
| VR（虚拟现实） | 沉浸式、完全虚拟环境 | 虚拟实验室、历史场景重建 |
| AR（增强现实） | 虚实叠加、真实环境增强 | 解剖学习、博物馆导览 |
| MR（混合现实） | 虚实交互、实时响应 | 协作设计、手术模拟 |

### 8.2 沉浸式学习优势

- **情境学习**：在真实或仿真情境中体验
- **安全练习**：高风险场景的低成本训练
- **空间理解**：三维空间概念的可视化
- **情感投入**：增强学习动机与记忆

### 8.3 虚拟实验室

虚拟实验室类型：

- **仿真型**：物理、化学实验的计算机模拟
- **远程型**：远程操控真实实验设备
- **虚拟-现实结合型**：虚实融合的实验环境

## 十、技术整合框架

### 10.1 TPACK 框架

科勒（Koehler）与米什拉（Mishra）提出整合技术的学科教学知识：

$$\text{TPACK} = \text{CK} + \text{PK} + \text{TK} + \text{交叉领域}$$

| 知识类型 | 缩写 | 内涵 |
|---------|------|------|
| 学科内容知识 | CK | 所教学科的知识 |
| 教学法知识 | PK | 教学策略与方法的知识 |
| 技术知识 | TK | 技术工具与资源的知识 |
| 学科教学知识 | PCK | 特定学科的教学方法 |
| 整合技术的学科知识 | TCK | 用技术表征学科内容 |
| 整合技术的教学法知识 | TPK | 用技术支持教学策略 |
| 整合技术的学科教学知识 | TPACK | 三者融合的最佳实践 |

### 10.2 SAMR 模型

普恩特杜尔（Puentedura）提出技术整合层次：

| 层次 | 特征 | 示例 |
|------|------|------|
| S-替代（Substitution） | 技术直接替代工具，功能无变化 | 用电子书替代纸质书 |
| A-增强（Augmentation） | 技术替代并增加部分功能 | 电子书+搜索、标注功能 |
| M-修改（Modification） | 技术允许任务重新设计 | 协作写作、多媒体创作 |
| R-重定义（Redefinition） | 技术创造先前不可能的任务 | 全球协作项目、虚拟实地考察 |

**目标**：从 S/A 层次向 M/R 层次发展
