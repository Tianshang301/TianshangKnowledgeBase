# 跨平台桌面应用开发

## 一、跨平台开发的必要性

桌面应用开发面临Windows、macOS、Linux三大平台的碎片化问题。跨平台框架能够降低开发成本、缩短交付周期、统一代码维护。

各平台原生技术栈对比：

| 平台 | 语言 | UI框架 | 包管理 | 发布渠道 |
|------|------|--------|--------|----------|
| Windows | C++/C# | WinForms/WPF/UWP | NuGet | MSIX |
| macOS | Swift/ObjC | Cocoa/SwiftUI | CocoaPods | App Store |
| Linux | C/C++ | GTK/Qt | apt/yum | 发行版仓库 |

## 二、Electron框架

Electron使用Chromium和Node.js构建跨平台桌面应用。

```javascript
// main.js - 主进程入口
const { app, BrowserWindow } = require('electron')

function createWindow() {
  const win = new BrowserWindow({
    width: 800,
    height: 600,
    webPreferences: {
      nodeIntegration: true,
      contextIsolation: false
    }
  })
  win.loadFile('index.html')
}

app.whenReady().then(createWindow)

app.on('window-all-closed', () => {
  if (process.platform !== 'darwin') app.quit()
})
```

进程间通信（IPC）：

```javascript
// main.js
const { ipcMain } = require('electron')
ipcMain.handle('read-file', async (event, filePath) => {
  const fs = require('fs')
  return fs.readFileSync(filePath, 'utf-8')
})

// renderer.js
const { ipcRenderer } = require('electron')
const content = await ipcRenderer.invoke('read-file', '/path/to/file')
```

Electron优势：Web技术栈复用、生态丰富、调试方便；劣势：内存占用高、启动慢、安装包体积大。

## 三、Qt框架

Qt是C++跨平台框架，提供原生级别性能和丰富的控件库。

```cpp
#include <QApplication>
#include <QPushButton>
#include <QVBoxLayout>
#include <QLabel>

int main(int argc, char *argv[]) {
    QApplication app(argc, argv);

    QWidget window;
    window.setWindowTitle("Qt Application");

    QVBoxLayout *layout = new QVBoxLayout();
    QLabel *label = new QLabel("Hello, Qt!");
    QPushButton *button = new QPushButton("Click Me");

    layout->addWidget(label);
    layout->addWidget(button);
    window.setLayout(layout);

    QObject::connect(button, &QPushButton::clicked, [&]() {
        label->setText("Button Clicked!");
    });

    window.show();
    return app.exec();
}
```

Qt信号与槽机制是其核心特性，支持类型安全的对象通信。Qt Quick和QML提供了声明式UI开发方式：

```qml
import QtQuick 2.15
import QtQuick.Controls 2.15

ApplicationWindow {
    visible: true
    width: 400; height: 300
    title: "QML Application"

    Button {
        anchors.centerIn: parent
        text: "Click Me"
        onClicked: label.text = "Clicked!"
    }

    Label {
        id: label
        anchors.top: parent.top
        anchors.horizontalCenter: parent.horizontalCenter
        text: "Hello, QML"
    }
}
```

## 四、Flutter Desktop

Flutter使用Dart语言和Skia渲染引擎，支持Windows、macOS、Linux桌面开发。

```dart
import 'package:flutter/material.dart';

void main() {
  runApp(const MyApp());
}

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: 'Flutter Desktop',
      home: Scaffold(
        appBar: AppBar(title: const Text('Flutter Desktop')),
        body: Center(
          child: ElevatedButton(
            onPressed: () => print('Button clicked'),
            child: const Text('Click Me'),
          ),
        ),
      ),
    );
  }
}
```

Flutter通过Platform Channel调用原生API：

```dart
import 'package:flutter/services.dart';

class PlatformBridge {
  static const platform = MethodChannel('com.example/app');

  Future<String> getPlatformVersion() async {
    try {
      final version = await platform.invokeMethod('getPlatformVersion');
      return version as String;
    } on PlatformException catch (e) {
      return "Failed: ${e.message}";
    }
  }
}
```

## 五、Tauri框架

Tauri是新兴的Rust框架，使用Web技术构建UI，后端用Rust提供系统调用。

```rust
// src-tauri/src/main.rs
#[tauri::command]
fn greet(name: &str) -> String {
    format!("Hello, {}!", name)
}

fn main() {
    tauri::Builder::default()
        .invoke_handler(tauri::generate_handler![greet])
        .run(tauri::generate_context!())
        .expect("error while running tauri application");
}
```

前端调用Rust命令：

```javascript
const { invoke } = window.__TAURI__.tauri
const result = await invoke('greet', { name: 'World' })
```

Tauri优势：安装包极小（通常小于5MB）、内存占用低、安全（Rust内存安全）；劣势：生态较新、社区较小。

## 七、平台适配策略

跨平台开发中的平台差异处理：

```cpp
// Qt平台条件编译
#ifdef Q_OS_WIN
    // Windows特定代码
    window.setWindowFlag(Qt::WindowStaysOnTopHint);
#elif defined(Q_OS_MAC)
    // macOS特定代码
    window.setAttribute(Qt::WA_MacAlwaysShowToolWindow);
#elif defined(Q_OS_LINUX)
    // Linux特定代码
    window.setWindowFlag(Qt::FramelessWindowHint);
#endif
```

Electron的平台检测：

```javascript
const { platform, arch } = process
const isWindows = platform === 'win32'
const isMacOS = platform === 'darwin'
const isLinux = platform === 'linux'
```

## 八、打包与分发

应用打包工具对比：

| 框架 | 打包工具 | 支持格式 |
|------|----------|----------|
| Electron | electron-builder | dmg/exe/AppImage/deb |
| Qt | windeployqt/macdeployqt | NSIS/AppImage |
| Flutter | flutter build | MSIX/dmg/AppImage |
| Tauri | tauri build | msi/dmg/deb |

Electron自动更新配置：

```javascript
const { autoUpdater } = require('electron-updater')

autoUpdater.checkForUpdatesAndNotify()

autoUpdater.on('update-downloaded', () => {
  autoUpdater.quitAndInstall()
})
```

## 九、性能优化建议

1. 避免在主线程执行耗时操作，使用Worker线程或异步处理
2. 延迟加载非关键资源，减少启动时间
3. 使用虚拟列表渲染大量数据
4. 合理管理内存，及时释放不再使用的对象
5. 使用原生模块替代纯JS实现计算密集型任务
6. 开启硬件加速（GPU）提升渲染性能
