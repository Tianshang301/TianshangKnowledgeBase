# 游戏引擎架构与渲染

## 一、游戏引擎架构概述

游戏引擎是游戏开发的底层框架，封装了图形渲染、物理模拟、音频处理、输入管理等核心模块。

典型游戏引擎分层架构：

```
应用层（游戏逻辑、脚本、UI）
-------------------------------
核心层（场景管理、资源管理、事件系统）
-------------------------------
渲染层（图形 API 封装、着色器、光照）
-------------------------------
平台层（操作系统抽象、输入、音频）
-------------------------------
硬件层（GPU、CPU、内存、外设）
```

## 三、渲染管线基础

渲染管线（Rendering Pipeline）将3D 场景转换为2D 图像：

```
应用阶段 -> 几何处理 -> 光栅化 -> 像素处理 -> 输出合并
```

顶点着色器（Vertex Shader）处理顶点变换：

```glsl
#version 450 core

layout(location = 0) in vec3 aPos;
layout(location = 1) in vec3 aNormal;
layout(location = 2) in vec2 aTexCoord;

uniform mat4 uModel;
uniform mat4 uView;
uniform mat4 uProjection;

out vec3 vNormal;
out vec2 vTexCoord;

void main() {
    gl_Position = uProjection * uView * uModel * vec4(aPos, 1.0);
    vNormal = mat3(transpose(inverse(uModel))) * aNormal;
    vTexCoord = aTexCoord;
}
```

片元着色器（Fragment Shader）计算像素颜色：

```glsl
#version 450 core

in vec3 vNormal;
in vec2 vTexCoord;

uniform sampler2D uTexture;
uniform vec3 uLightDir;

out vec4 FragColor;

void main() {
    vec3 lightDir = normalize(uLightDir);
    vec3 normal = normalize(vNormal);
    float diff = max(dot(normal, -lightDir), 0.0);
    vec4 texColor = texture(uTexture, vTexCoord);
    vec3 ambient = 0.1 * texColor.rgb;
    vec3 diffuse = diff * texColor.rgb;
    FragColor = vec4(ambient + diffuse, texColor.a);
}
```

## 四、光照模型

兰伯特漫反射模型（Lambertian Diffuse）：

$$
I_{diffuse} = k_d I_L \max(0, \hat{L} \cdot \hat{N})
$$

Phong 镜面反射模型：

$$
I_{specular} = k_s I_L \max(0, \hat{R} \cdot \hat{V})^{\alpha}
$$

Blinn-Phong 模型使用半向量简化计算：

$$
I_{specular} = k_s I_L \max(0, \hat{N} \cdot \hat{H})^{\alpha}
$$

其中 $\hat{H} = \frac{\hat{L} + \hat{V}}{\|\hat{L} + \hat{V}\|}$。

## 五、阴影与全局光照

阴影映射（Shadow Mapping）原理：

```cpp
// 第一趟：从光源视角渲染深度图
glBindFramebuffer(GL_FRAMEBUFFER, depthMapFBO);
glViewport(0, 0, SHADOW_WIDTH, SHADOW_HEIGHT);
renderScene(lightViewMatrix, lightProjectionMatrix);

// 第二趟：从相机视角渲染，比对深度
glBindFramebuffer(GL_FRAMEBUFFER, 0);
shader.use();
shader.setMat4("lightSpaceMatrix", lightSpaceMatrix);
glActiveTexture(GL_TEXTURE1);
glBindTexture(GL_TEXTURE_2D, depthMap);
renderScene(cameraViewMatrix, cameraProjectionMatrix);
```

## 六、场景管理与剔除

场景图（Scene Graph）使用树状结构组织场景对象：

```
Root
├── Camera
├── Light (Directional)
├── Terrain
└── Player
    ├── Weapon
    └── Character
        ├── Skeleton
        └── Mesh
```

视锥剔除（Frustum Culling）算法：

```cpp
struct Frustum {
    Plane planes[6]; // 左、右、上、下、近、远
};

bool isVisible(const Frustum& frustum, const BoundingBox& box) {
    for (int i = 0; i < 6; i++) {
        if (frustum.planes[i].distance(box.center) < -box.radius)
            return false;
    }
    return true;
}
```

## 七、物理引擎

物理引擎处理刚体碰撞、软体模拟、布料仿真等。常用物理引擎：

| 引擎 | 语言 | 集成引擎 | 特点 |
|------|------|----------|------|
| PhysX | C++ | Unity/Unreal | GPU 加速 |

| Bullet | C++ | Godot/Blender | 开源免费 |
| Box2D | C++ | Cocos2d | 2D 物理标准 |

碰撞检测算法：

$$
\text{AABB 相交测试: } overlap_x = (A_x < B_x + B_w) \land (A_x + A_w > B_x)
$$

## 九、性能优化策略

1. 合批（Batching）减少 Draw Call：静态合批、动态合批
2. LOD（Level of Detail）降低远处物体面数
3. Occlusion Culling 剔除被遮挡物体
4. GPU Instancing 加速重复物体渲染
5. 纹理压缩（BC/DXT/ETC）减少显存占用
6. 对象池（Object Pool）减少 GC 压力

优化前后的渲染统计：

| 指标 | 优化前 | 优化后 |
|------|--------|--------|
| Draw Call | 5000 | 200 |
| 三角形数 | 2M | 500K |
| 显存占用 | 2GB | 800MB |
| FPS | 30 | 120 |
