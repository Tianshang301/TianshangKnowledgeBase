# 排序与搜索算法深入

## 一、排序算法分类与理论下界

排序问题是计算机科学中最基本的问题之一，也是算法分析的经典范例。排序算法可分为比较排序和非比较排序两大类。比较排序的最优时间复杂度下界为 $\Omega(n \log n)$，这由决策树模型证明：对 $n$ 个元素的排序有 $n!$ 种可能排列，比较排序的决策树高度至少为 $\log_2(n!) = \Omega(n \log n)$。

非比较排序(计数排序、基数排序、桶排序)可以突破 $\Omega(n \log n)$ 下界，但需要对输入值的分布做出额外假设。

## 二、高效比较排序

### 2.1 快速排序

快速排序采用分治策略，平均时间复杂度 $O(n \log n)$，最坏情况 $O(n^2)$：

```python
def quicksort(arr, low, high):
    if low < high:
        pivot = partition(arr, low, high)
        quicksort(arr, low, pivot - 1)
        quicksort(arr, pivot + 1, high)

def partition(arr, low, high):
    pivot = arr[high]  # 选择最后一个元素作为基准
    i = low - 1
    for j in range(low, high):
        if arr[j] <= pivot:
            i += 1
            arr[i], arr[j] = arr[j], arr[i]
    arr[i + 1], arr[high] = arr[high], arr[i + 1]
    return i + 1
```

**优化策略**：三数取中法选择基准避免最坏情况、小数组切换为插入排序、三路切分处理重复元素。

### 2.2 归并排序

归并排序提供稳定的 $O(n \log n)$ 性能，但需要 $O(n)$ 额外空间：

$$
T(n) = 2T(n/2) + O(n) \Rightarrow T(n) = O(n \log n)
$$

### 2.3 堆排序

堆排序利用二叉堆实现在原位的 $O(n \log n)$ 排序。建堆操作可在 $O(n)$ 时间内完成：

```python
def heapify(arr, n, i):
    largest = i
    left, right = 2 * i + 1, 2 * i + 2
    if left < n and arr[left] > arr[largest]:
        largest = left
    if right < n and arr[right] > arr[largest]:
        largest = right
    if largest != i:
        arr[i], arr[largest] = arr[largest], arr[i]
        heapify(arr, n, largest)
```

| 算法 | 最坏时间 | 平均时间 | 最好时间 | 空间 | 稳定 |
|------|----------|----------|----------|------|------|
| 快速排序 | $O(n^2)$ | $O(n \log n)$ | $O(n \log n)$ | $O(\log n)$ | 否 |
| 归并排序 | $O(n \log n)$ | $O(n \log n)$ | $O(n \log n)$ | $O(n)$ | 是 |
| 堆排序 | $O(n \log n)$ | $O(n \log n)$ | $O(n \log n)$ | $O(1)$ | 否 |
| 插入排序 | $O(n^2)$ | $O(n^2)$ | $O(n)$ | $O(1)$ | 是 |
| 希尔排序 | $O(n^2)$ | $O(n^{1.3})$ | $O(n \log n)$ | $O(1)$ | 否 |

## 三、非比较排序

### 3.1 计数排序

计数排序适用于值域范围已知且较小的整数数组：

$$
\text{时间复杂度} = O(n + k), \quad \text{空间复杂度} = O(k)
$$

其中 $k$ 为值域大小。当 $k = O(n)$ 时，计数排序是线性时间的排序算法。

### 3.2 基数排序

基数排序从最低有效位到最高有效位逐位排序(低位优先LSD)。若数字有 $d$ 位，每位使用计数排序，总复杂度为 $O(d \cdot (n + k))$。

### 3.3 桶排序

桶排序将值域分成 $m$ 个桶，将元素分配到桶中，每个桶内单独排序后合并。当输入均匀分布时，期望时间复杂度为 $O(n)$。

## 四、搜索算法

### 4.1 二分搜索

二分搜索在有序数组中查找目标值：

$$
T(n) = T(n/2) + O(1) \Rightarrow T(n) = O(\log n)
$$

**变体问题**：
- 查找第一个/最后一个等于目标的位置
- 查找第一个大于等于/小于等于目标的元素
- 旋转有序数组中的搜索
- 峰值查找

### 4.2 三分搜索

三分搜索适用于单峰函数的极值查找。每次将区间划分为三段，比较两个中间点的函数值缩小范围：

$$
T(n) = T(2n/3) + O(1) \Rightarrow T(n) = O(\log_{3/2} n)
$$

### 4.3 跳表

跳表通过在有序链表上构建多层索引实现 $O(\log n)$ 的搜索、插入和删除。每层索引以概率 $p = 1/2$ 选择包含哪些元素。

## 五、第k大元素的选择算法

### 5.1 快速选择

快速选择是快速排序的变体，期望时间复杂度 $O(n)$，最坏情况 $O(n^2)$：

```python
def quickselect(arr, k):
    if len(arr) == 1:
        return arr[0]
    pivot = partition(arr, 0, len(arr) - 1)
    if k == pivot:
        return arr[pivot]
    elif k < pivot:
        return quickselect(arr[:pivot], k)
    else:
        return quickselect(arr[pivot+1:], k - pivot - 1)
```

### 5.2 BFPRT算法

BFPRT(Median of Medians)算法保证最坏情况线性时间的选择算法。其核心是将数组分为每组5个元素的子组，取每组中位数，再递归取中位数的中位数作为枢轴：

$$
T(n) \leq T(n/5) + T(7n/10) + O(n) \Rightarrow T(n) = O(n)
$$

## 六、排序算法的实际选择

在工程实践中选择排序算法需考虑：

1. **数据规模**：小数组(n < 50)用插入排序，大数组用快排或归并
2. **内存限制**：内存有限时用堆排序或快速排序(原地)
3. **稳定性需求**：需要稳定排序时用归并排序
4. **数据特征**：数据几乎有序时用插入排序，值域小用计数排序
5. **外部排序**：海量数据用多路归并

许多语言的标准库采用**内省排序(introsort)**——开始使用快速排序，当递归深度超过 $\log n$ 的某个倍数时切换为堆排序，小规模子数组用插入排序。
