# 网络流与匹配算法

## 一、网络流问题定义

网络流问题研究在存在容量约束的有向图中，从源点到汇点的流量传输。一个流网络 $G = (V, E)$ 是一个有向图，每条边 $(u, v)$ 具有非负容量 $c(u, v)$。网络流满足两个约束：

1. **容量约束**：$0 \leq f(u, v) \leq c(u, v)$
2. **流量守恒**：$\forall v \in V \setminus \{s, t\}, \sum_{(u,v) \in E} f(u, v) = \sum_{(v,w) \in E} f(v, w)$

最大流问题的目标是最大化从源点 $s$ 到汇点 $t$ 的总流量。最小割定理指出：最大流的值等于最小割的容量。

## 二、最大流算法

### 2.1 Ford-Fulkerson方法

Ford-Fulkerson通过在残留网络中反复寻找增广路径增加流量：

```
残留容量: c_f(u,v) = c(u,v) - f(u,v) + f(v,u)
```

```python
def ford_fulkerson(graph, s, t):
    n = len(graph)
    flow = [[0] * n for _ in range(n)]

    def bfs_find_path(parent):
        visited = [False] * n
        queue = [s]
        visited[s] = True
        while queue:
            u = queue.pop(0)
            for v in range(n):
                if not visited[v] and graph[u][v] - flow[u][v] > 0:
                    visited[v] = True
                    parent[v] = u
                    if v == t:
                        return True
                    queue.append(v)
        return visited[t]

    parent = [-1] * n
    max_flow = 0
    while bfs_find_path(parent):
        path_flow = float('inf')
        v = t
        while v != s:
            u = parent[v]
            path_flow = min(path_flow, graph[u][v] - flow[u][v])
            v = u
        v = t
        while v != s:
            u = parent[v]
            flow[u][v] += path_flow
            flow[v][u] -= path_flow
            v = u
        max_flow += path_flow
    return max_flow
```

Edmonds-Karp算法是Ford-Fulkerson的BFS实现，时间复杂度 $O(VE^2)$。

### 2.2 Dinic算法

Dinic算法通过分层图和阻塞流实现 $O(V^2E)$ 的复杂度，在单位容量图上达到 $O(\min(V^{2/3}, \sqrt{E}) \cdot E)$：

1. 使用BFS构建分层网络(level graph)
2. 在分层网络上使用DFS寻找阻塞流
3. 重复直到源点无法到达汇点

```python
def dinic(graph, s, t):
    n = len(graph)
    flow = [[0] * n for _ in range(n)]

    def bfs_level():
        level = [-1] * n
        level[s] = 0
        queue = [s]
        while queue:
            u = queue.pop(0)
            for v in range(n):
                if level[v] < 0 and graph[u][v] - flow[u][v] > 0:
                    level[v] = level[u] + 1
                    queue.append(v)
        return level[t] >= 0

    def dfs_flow(u, f, level, it):
        if u == t:
            return f
        for v in range(it[u], n):
            it[u] = v
            if level[v] == level[u] + 1 and graph[u][v] - flow[u][v] > 0:
                pushed = dfs_flow(v, min(f, graph[u][v] - flow[u][v]), level, it)
                if pushed > 0:
                    flow[u][v] += pushed
                    flow[v][u] -= pushed
                    return pushed
        return 0

    total = 0
    while bfs_level():
        it = [0] * n
        while True:
            pushed = dfs_flow(s, float('inf'), [0] * n, it)
            if pushed == 0:
                break
            total += pushed
    return total
```

| 算法 | 时间复杂度 | 特点 |
|------|-----------|------|
| Ford-Fulkerson | $O(E \cdot f_{\max})$ | 依赖容量大小 |
| Edmonds-Karp | $O(VE^2)$ | BFS增广 |
| Dinic | $O(V^2E)$ | 分层+阻塞流 |
| Push-Relabel | $O(V^3)$ | 预流推进 |
| HLPP | $O(V^2\sqrt{E})$ | 最高标号预流推进 |

## 三、最小费用最大流

### 3.1 问题定义

在每条边具有容量 $c(u, v)$ 和单位费用 $w(u, v)$ 的网络上，寻找给定流量下总费用最小的流：

$$
\min \sum_{(u,v) \in E} w(u, v) \cdot f(u, v)
$$

### 3.2 连续最短增广路算法(SSP)

使用Bellman-Ford或势能优化的Dijkstra在残留网络中寻找最短增广路：

```python
def min_cost_flow(graph, cost, s, t, max_flow):
    n = len(graph)
    flow, total_cost = [[0]*n for _ in range(n)], 0
    potential = [0] * n

    while max_flow > 0:
        dist = [float('inf')] * n
        dist[s], parent = 0, [-1] * n
        in_queue = [False] * n
        queue = [s]

        while queue:
            u = queue.pop(0)
            in_queue[u] = False
            for v in range(n):
                if graph[u][v] - flow[u][v] > 0:
                    reduced_cost = cost[u][v] + potential[u] - potential[v]
                    if dist[v] > dist[u] + reduced_cost:
                        dist[v] = dist[u] + reduced_cost
                        parent[v] = u
                        if not in_queue[v]:
                            queue.append(v)
                            in_queue[v] = True

        if dist[t] == float('inf'):
            break

        for v in range(n):
            if dist[v] < float('inf'):
                potential[v] += dist[v]

        pushed = max_flow
        v = t
        while v != s:
            u = parent[v]
            pushed = min(pushed, graph[u][v] - flow[u][v])
            v = u

        v = t
        while v != s:
            u = parent[v]
            flow[u][v] += pushed
            flow[v][u] -= pushed
            total_cost += pushed * cost[u][v]
            v = u
        max_flow -= pushed

    return total_cost
```

## 四、二分图匹配

### 4.1 最大匹配

二分图匹配可以在 $O(VE)$ 时间内使用匈牙利算法求解，也可以转化为最大流问题(源点到左部节点容量1，左到右容量1，右到汇点容量1)。

**匈牙利算法(DFS版本)**：

```python
def hungarian(n, m, adj):
    match = [-1] * m
    def dfs(u, seen):
        for v in adj[u]:
            if not seen[v]:
                seen[v] = True
                if match[v] == -1 or dfs(match[v], seen):
                    match[v] = u
                    return True
        return False

    result = 0
    for u in range(n):
        if dfs(u, [False] * m):
            result += 1
    return result
```

### 4.2 带权匹配(KM算法)

Kuhn-Munkres(KM)算法求解完全二分图上的最大权完美匹配，时间复杂度 $O(n^3)$。算法维护顶标 $lx[u] + ly[v] \geq w[u][v]$，在相等子图中寻找增广路。

### 4.3 匹配相关定理

- **König定理**：二分图的最小点覆盖等于最大匹配
- **Hall定理**：存在饱和左部所有点的匹配当且仅当任意左部子集 $S$ 满足 $|N(S)| \geq |S|$
- **Dilworth定理**：偏序集的最小链划分等于最大反链大小

## 五、网络流的应用

| 问题 | 建模方法 |
|------|----------|
| 最大二分图匹配 | 源→左部→右部→汇，容量1 |
| 最小路径覆盖 | 拆点为入点/出点，求最大匹配 |
| 最大闭合子图 | 正权连源，负权连汇，容量为权值 |
| 二分图多重匹配 | 源到左部容量为匹配上限 |
| 最大密度子图 | 转化为最小割问题 |
| 混合图欧拉回路 | 度平衡+网络流判定 |

最大闭合子图用于挖掘依赖关系中的最优子集：选择获利为正的项目依赖有成本的项目。连边方法为：

$$
\text{profit} = \sum_{w_i > 0} w_i - \text{minCut}
$$

## 六、进阶主题

### 6.1 上下界网络流

每条边有容量下限 $l(u, v)$ 和上限 $u(u, v)$，通过转换为标准网络流求解：

$$
c'(u, v) = u(u, v) - l(u, v)
$$

### 6.2 全局最小割(Stoer-Wagner算法)

使用Stoer-Wagner算法在 $O(V^3)$ 时间内求无向图的全局最小割，无需指定源点和汇点。
