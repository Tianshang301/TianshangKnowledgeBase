# 模式匹配与字符串处理

## 一、字符串匹配问题

字符串匹配是计算机科学中的基础问题：在文本 $T[1..n]$ 中查找模式 $P[1..m]$ 的所有出现位置。该问题在文本编辑、搜索引擎、生物信息学和网络安全中具有广泛的应用。

暴力匹配算法逐个检查每个可能的对齐位置，最坏情况时间复杂度为 $O(nm)$。高效的字符串匹配算法通过对匹配过程进行优化，将时间复杂度降低到 $O(n+m)$。

## 二、单模式匹配算法

### 2.1 KMP 算法

KMP(Knuth-Morris-Pratt)算法通过预处理模式串避免文本指针的回退。核心数据结构是**部分匹配表**(next 数组/失配函数)，记录模式串的前缀与后缀的最长公共长度。

```python
def build_lps(pattern):
    m = len(pattern)
    lps = [0] * m
    length = 0
    i = 1
    while i < m:
        if pattern[i] == pattern[length]:
            length += 1
            lps[i] = length
            i += 1
        else:
            if length != 0:
                length = lps[length - 1]
            else:
                lps[i] = 0
                i += 1
    return lps

def kmp_search(text, pattern):
    n, m = len(text), len(pattern)
    lps = build_lps(pattern)
    i = j = 0
    matches = []
    while i < n:
        if pattern[j] == text[i]:
            i += 1
            j += 1
        if j == m:
            matches.append(i - j)
            j = lps[j - 1]
        elif i < n and pattern[j] != text[i]:
            if j != 0:
                j = lps[j - 1]
            else:
                i += 1
    return matches
```

失配函数 $next[j]$ 满足当 $P[j]$ 不匹配时，将模式串右移 $j - next[j-1]$ 位。其定义：

$$
\text{lps}[j] = \max\{k < j+1 \mid P[0..k-1] = P[j-k+1..j]\}
$$

### 2.2 Boyer-Moore 算法

Boyer-Moore 算法从模式串的末尾开始匹配，利用**坏字符规则**和**好后缀规则**跳过尽可能多的字符：

- **坏字符规则**：当文本字符 $c$ 与模式字符不匹配时，将模式右移到 $c$ 在模式中最右侧出现的位置之后
- **好后缀规则**：利用已匹配的后缀信息决定移动距离

Boyer-Moore 在实践中通常比 KMP 更快，尤其是在大字母表和长模式的情况下。

### 2.3 Rabin-Karp 算法

Rabin-Karp 使用滚动哈希在 $O(n+m)$ 期望时间内完成匹配：

$$
h(k) = \sum_{i=0}^{m-1} P[i] \cdot d^{m-1-i} \mod q
$$

滚动更新哈希值：
$$
h_{\text{new}} = (d \cdot (h_{\text{old}} - T[i] \cdot d^{m-1}) + T[i+m]) \mod q
$$

哈希值相同时需验证字符串是否真正相等(解决哈希冲突)。

## 三、多模式匹配

### 3.1 Aho-Corasick 自动机

Aho-Corasick 将多个模式串构建成一个 Trie 树，添加失败指针实现线性时间多模式匹配：

```
构建步骤：
1. 将所有模式串插入 Trie 树
2. 通过 BFS 构建失败指针(fail link)
3. 同时收集输出(output link)
```

```python
class AhoCorasick:
    def __init__(self):
        self.trie = [{}]
        self.fail = [0]
        self.output = [[]]

    def add_pattern(self, pattern):
        node = 0
        for ch in pattern:
            if ch not in self.trie[node]:
                self.trie[node][ch] = len(self.trie)
                self.trie.append({})
                self.fail.append(0)
                self.output.append([])
            node = self.trie[node][ch]
        self.output[node].append(pattern)

    def build(self):
        from collections import deque
        q = deque()
        for ch, node in self.trie[0].items():
            self.fail[node] = 0
            q.append(node)
        while q:
            r = q.popleft()
            for ch, u in self.trie[r].items():
                q.append(u)
                v = self.fail[r]
                while v and ch not in self.trie[v]:
                    v = self.fail[v]
                self.fail[u] = self.trie[v].get(ch, 0)
                self.output[u] += self.output[self.fail[u]]

    def search(self, text):
        node = 0
        matches = []
        for i, ch in enumerate(text):
            while node and ch not in self.trie[node]:
                node = self.fail[node]
            node = self.trie[node].get(ch, 0)
            if self.output[node]:
                for pat in self.output[node]:
                    matches.append((i - len(pat) + 1, pat))
        return matches
```

复杂度 $O(n + \sum |P_i|)$，其中 $n$ 为文本长度。

## 四、后缀数据结构

### 4.1 后缀树

后缀树包含字符串所有后缀的压缩 Trie 树，可在 $O(m)$ 时间内完成模式匹配，$O(n)$ 时间内构建(Ukkonen 算法)。每个内部节点保存从根到节点的路径子串。

后缀树的应用：
- 查找最长重复子串
- 查找最长公共子串
- 统计子串出现次数

### 4.2 后缀数组

后缀数组 $SA$ 是字符串所有后缀按字典序排序后的起始位置数组。配合 LCP 数组(相邻后缀的最长公共前缀)可实现后缀树的大部分功能。

```python
def build_sa(s):
    n = len(s)
    sa = list(range(n))
    rank = [ord(c) for c in s]
    k = 1
    while k < n:
        sa.sort(key=lambda x: (rank[x], rank[x + k] if x + k < n else -1))
        temp = [0] * n
        for i in range(1, n):
            prev, cur = sa[i-1], sa[i]
            temp[cur] = temp[prev]
            if rank[prev] != rank[cur] or \
               (rank[prev + k] if prev + k < n else -1) != \
               (rank[cur + k] if cur + k < n else -1):
                temp[cur] += 1
        rank = temp
        if rank[sa[-1]] == n - 1:
            break
        k <<= 1
    return sa
```

后缀数组可在 $O(n)$ 时间(SA-IS 算法)或 $O(n \log n)$ 时间内构建。

| 数据结构 | 构建时间 | 空间 | 匹配时间 | 适用 |
|----------|----------|------|----------|------|
| 后缀树 | $O(n)$ | $O(n)$ | $O(m)$ | 功能丰富 |
| 后缀数组 | $O(n)$-$O(n \log n)$ | $O(n)$ | $O(m \log n)$ | 空间高效 |

## 五、近似字符串匹配

### 5.1 编辑距离(Levenshtein 距离)

编辑距离衡量将 $A$ 转换为 $B$ 所需的最少操作次数(插入、删除、替换)：

$$
dp[i][j] = \min(dp[i-1][j] + 1, dp[i][j-1] + 1, dp[i-1][j-1] + \text{cost}(A_i, B_j))
$$

**优化**：当允许 $k$ 个误差时，使用**带通配符的 DFA**或**位并行**算法可将搜索复杂度降低到 $O(kn)$。

### 5.2 正则匹配

正则表达式通过有限自动机实现模式匹配。Thompson 构造法将正则转换为 NFA，再通过子集构造法转换为 DFA 进行高效率匹配。

```regex
[a-z]+@[a-z]+\.[a-z]{2,}  # 简单邮箱正则
```

## 六、字符串处理实际应用

```python
# 最长的回文子串-Manacher 算法
def manacher(s):
    t = '#' + '#'.join(s) + '#'
    n = len(t)
    p = [0] * n
    center = right = 0
    for i in range(n):
        if i < right:
            p[i] = min(right - i, p[2 * center - i])
        while i - p[i] - 1 >= 0 and i + p[i] + 1 < n \
              and t[i - p[i] - 1] == t[i + p[i] + 1]:
            p[i] += 1
        if i + p[i] > right:
            center, right = i, i + p[i]
    max_len = max(p)
    center_idx = p.index(max_len)
    start = (center_idx - max_len) // 2
    return s[start:start + max_len]
```
