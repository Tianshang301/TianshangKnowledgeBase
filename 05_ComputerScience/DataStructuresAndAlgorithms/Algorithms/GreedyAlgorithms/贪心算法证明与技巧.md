# 贪心算法证明与技巧

## 一、贪心算法原理

贪心算法在每一步选择当前看来最优的决策，期望通过局部最优的选择达到全局最优。贪心算法的正确性依赖问题本身的特定性质——并非所有问题都适用贪心策略。

贪心算法的核心要素是**贪心选择性质**(greedy choice property)——全局最优解可以通过一系列局部最优选择达到。这一性质通常通过交换论证、数学归纳或反证法加以证明。

## 二、贪心正确性证明方法

### 2.1 交换论证(Exchange Argument)

交换论证是证明贪心算法正确性的最常用方法。基本思路：假设存在一个最优解，若其与贪心解不同，则可以通过一系列不降低质量的交换将其转换为贪心解。

**证明模板**：
1. 设贪心解为 $G$，任意一个最优解为 $O$
2. 找到 $G$ 和 $O$ 的第一个不同决策点
3. 在 $O$ 中交换为 $G$ 的决策，证明交换后不破坏可行性且不降低解的质量
4. 通过有限次交换，$O$ 变为 $G$，从而 $G$ 也是最优解

### 2.2 数学归纳法

通过归纳证明贪心算法在每一步后产生的部分解是所有可行解前缀中的最优解。

### 2.3 反证法

假设贪心解不是最优的，通过构造导出矛盾。

## 三、经典贪心问题证明

### 3.1 活动选择问题

**问题**：选择最大数量的互不重叠的活动。

**贪心策略**：按结束时间从小到大选择活动，选择第一个不冲突的活动。

**证明**：
- 设贪心选择的活动为 $a_1$(最早结束的活动)，$O$ 为某个最优解
- 若 $a_1 \notin O$，设 $O$ 中最早结束的活动为 $b_1$，必有 $f(a_1) \leq f(b_1)$
- 将 $O$ 中的 $b_1$ 替换为 $a_1$ 得到 $O'$，$O'$ 仍可行且大小不变
- 归纳可知贪心选择可以达到最优

```python
def activity_selection(activities):
    activities.sort(key=lambda x: x[1])  # 按结束时间排序
    selected = [activities[0]]
    last_end = activities[0][1]

    for start, end in activities[1:]:
        if start >= last_end:
            selected.append((start, end))
            last_end = end
    return selected
```

### 3.2 哈夫曼编码

**问题**：构造最优前缀码，最小化加权路径长度 $WPL = \sum f_i \cdot l_i$。

**贪心策略**：从最小频率的字符开始，重复合并两个频率最小的节点。

哈夫曼的正确性基于**贪心选择性质**：(反证)若最优编码树中两个最小频率字符不在最深层，交换它们可得到更优的编码。

### 3.3 最小生成树(MST)

| 算法 | 贪心策略 | 时间复杂度 |
|------|----------|-----------|
| Kruskal | 按边权升序选择，不形成环则加入 | $O(E \log E)$ |
| Prim | 从起点开始，每次加入最近的未连接点 | $O(V^2)$ / $O(E \log V)$ |

**Kruskal 正确性证明(交换论证)**：
- 设 Kruskal 生成的边集为 $K$，某个 MST 为 $T$
- 找到 $K$ 和 $T$ 中第一条不同的边 $e \in K \setminus T$
- 将 $e$ 加入 $T$ 形成环，环中必存在一条不在 $K$ 中的边 $e'$
- 替换 $e'$ 为 $e$ 不增加总权值(因为 Kruskal 选择 $e$ 时 $e'$ 尚未被选，说明 $w(e) \leq w(e')$)

## 四、贪心算法技巧

### 4.1 排序是关键

贪婪决策往往依赖某种排序规则：

| 问题 | 排序键 | 贪心规则 |
|------|--------|----------|
| 活动选择 | 结束时间 | 选最早结束 |
| 区间调度 | 结束时间 | 选最早结束 |
| 加油站问题 | 无(顺序处理) | 油不够就加油 |
| 最大相容区间 | 开始时间 | 尽量避免重叠 |
| 最小字典序 | 自定义比较器 | 每次选最小 |

### 4.2 数据结构辅助

- **优先队列**：动态选择当前最优(Huffman 编码、Dijkstra)
- **并查集**：维护连通分量(Kruskal)
- **线段树/树状数组**：加速区间选择

### 4.3 贪心+二分答案

当直接贪心难以确定最优值时，先二分答案再用贪心验证可行性：

```python
def can_achieve(target, data):
    # 贪心验证是否可以达到 target
    count = 0
    current = 0
    for item in sorted(data):
        if current + item <= target:
            current += item
        else:
            count += 1
            current = item
    return count <= limit
```

## 五、贪心算法的局限

并非所有问题都能用贪心求解。0-1背包问题是经典的反例——贪心按价值密度排序无法保证得到最优解，而分数背包问题贪心可以求解。区分贪心适用性的关键是问题是否具有贪心选择性质。

## 六、常见贪心问题对比

| 问题 | 贪心策略 | 正确性 | 证明方法 |
|------|----------|--------|----------|
| 找零钱(标准币制) | 用最大面额 | 正确 | 数学归纳 |
| 最小延迟调度 | 按截止时间排序 | 正确 | 交换论证 |
| 最长递增子序列 | 维护最小末尾值 | 正确(需理解) | 归纳 |
| 跳跃游戏 | 维护最远可达位置 | 正确 | 贪心选择 |
| 分数背包 | 按价值密度排序 | 正确 | 交换论证 |
| 0-1背包 | 按价值密度排序 | 错误 | 反例 |
| 图着色 | 按度数降序 | 不一定最优 | NP-hard 问题 |

## 七、贪心与动态规划的关系

贪心和 DP 都利用最优子结构性质，区别在于贪心假设贪心选择性质成立，而 DP 对子问题全面求值。DP 可以解决贪心不能解决的问题(如0-1背包)，但通常需要更高的时间复杂度。

选择贪心的依据：能够证明贪心选择性质成立，且贪心决策后剩余子问题与原问题同构且独立。若局部最优不能保证全局最优，则需要使用 DP。

```python
# 贪心算法模板
def greedy_solve(data):
    data.sort(key=custom_key)
    result = initial_solution()
    for item in data:
        if is_feasible(result, item):
            result = add_to_solution(result, item)
    return result
```
