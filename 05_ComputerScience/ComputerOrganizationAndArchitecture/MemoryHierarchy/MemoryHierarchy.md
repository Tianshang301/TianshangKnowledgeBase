# 存储器层次结构 (Memory Hierarchy)

## 1. 存储层次概念

### 存储金字塔

```
                 性能↑  容量↓  成本↑
              ┌──────────────────────┐
              │   CPU 寄存器          │  ~1ns, <1KB
              │  ≈ 1 周期             │
              ├──────────────────────┤
              │   L1 Cache (指令/数据) │  ~2-4ns, 32-64KB
              │  ≈ 3 周期             │
              ├──────────────────────┤
              │   L2 Cache            │  ~10ns, 256-512KB
              │  ≈ 10-20 周期         │
              ├──────────────────────┤
              │   L3 Cache (共享)     │  ~30ns, 4-32MB
              │  ≈ 30-50 周期         │
              ├──────────────────────┤
              │   主存 (DRAM)         │  ~100ns, 4-256GB
              │  ≈ 200 周期           │
              ├──────────────────────┤
              │   SSD (NAND Flash)   │  ~0.1ms, 256GB-2TB
              │  ≈ 100000 周期        │
              ├──────────────────────┤
              │   HDD (磁盘)          │  ~10ms, 1-20TB
              │  ≈ 10⁷ 周期           │
              ├──────────────────────┤
              │   云存储/磁带         │  ~s-min, PB-EB
              └──────────────────────┘
                 性能↓  容量↑  成本↓
```

### 局部性原理 (Principle of Locality)

```
时间局部性 (Temporal Locality):
  最近访问的数据很可能再次被访问
  → 循环中的变量、累加器

空间局部性 (Spatial Locality):
  相邻地址的数据很可能被访问
  → 数组遍历、顺序代码执行

例子：
  for (i = 0; i < N; i++) {
      sum += a[i];    // 时间: sum 每次循环都访问
  }                    // 空间: a[i], a[i+1], a[i+2] 相邻
```

## 2. Cache 映射方式

### 直接映射 (Direct-Mapped)

```
Cache 行 = 内存块地址 % Cache 行数

地址分解：
┌─────────────────┬──────────────┬────────────┐
│      Tag        │    Index     │ Block Offset│
│   (高位)        │  (Cache索引)  │ (块内偏移)  │
└─────────────────┴──────────────┴────────────┘

示例：64KB Cache, 64B 块, 32位地址
  块偏移: log₂(64) = 6 bit
  索引:   64KB / 64B = 1024 行 → log₂(1024) = 10 bit
  Tag:    32 - 10 - 6 = 16 bit

优点：实现简单，单周期比较
缺点：冲突率高（多个地址映射到同一行）
```

### 组相联 (Set-Associative)

```
n 路组相联：每组含 n 个 Cache 行

地址分解：
┌─────────────────┬──────────────┬────────────┐
│      Tag        │  Set Index   │ Block Offset│
└─────────────────┴──────────────┴────────────┘

组数 = Cache行数 / 相联度

2路组相联示例：
  Set 0: ┌──────┬──────┐
         │Way 0 │Way 1 │
         ├──────┼──────┤
  Set 1: │      │      │
         ├──────┼──────┤
  Set 2: │      │      │
         ├──────┼──────┤
  ...

命中率与相联度的关系：
  直接映射 ≈ 80%   2路 ≈ 85%   4路 ≈ 88%
  8路 ≈ 90%       全相联 ≈ 92%
```

### 全相联 (Fully Associative)

```
任何内存块可映射到任何 Cache 行

地址分解：
┌─────────────────────────────────┬────────────┐
│             Tag                 │Block Offset│
└─────────────────────────────────┴────────────┘

优点：命中率最高（无冲突缺失）
缺点：比较器极复杂（同时比较所有 Tag）
用途：TLB 通常为全相联或高相联
```

## 3. 替换策略 (Replacement Policies)

| 策略 | 原理 | 实现复杂度 | 命中率 | 说明 |
|------|------|-----------|--------|------|
| LRU | 替换最久未使用 | 高（需计数值或链） | 高 | 最常用策略 |
| FIFO | 替换最早进入 | 低（循环队列） | 中 | 可能替换常用块 |
| Random | 随机替换 | 极低 | 中 | 避免最坏情况 |
| LFU | 替换使用最少 | 中（计数） | 中 | 陈旧数据问题 |
| Belady MIN | 替换未来最久不用 | 不可实现（需预知） | 最优 | 理论下界 |

```
LRU 实现（4路组相联）：
  访问序列: A B C D A ... (A 被再次访问)

  初始: [A(0)][B(1)][C(2)][D(3)]  (数字=访问时间)
  访问A: [A(4)][B(1)][C(2)][D(3)]  (A移到最近)
  访问E: [A(4)][E(5)][C(2)][D(3)]  (替换最久B)

近似 LRU (Not Recently Used):
  使用 Reference Bit，定期清零
  选择 R=0 的行替换
  硬件成本低，接近 LRU 性能
```

## 4. 写策略 (Write Policies)

### 写命中

| 策略 | 操作 | 内存一致 | 优点 | 缺点 |
|------|------|---------|------|------|
| Write-Through | 写入 Cache 并同时写入内存 | 始终一致 | 简单 | 写带宽大 |
| Write-Back | 仅写入 Cache，标记 Dirty | 可能不一致 | 写带宽小 | 复杂（需 Dirty 位） |

### 写缺失

| 策略 | 操作 | 说明 |
|------|------|------|
| Write-Allocate | 先加载到 Cache，再写入 | 常与 Write-Back 配对 |
| No-Write-Allocate | 直接写入内存 | 常与 Write-Through 配对 |

```
Write-Back 写入流程：
  读命中：直接读 Cache
  写命中：写 Cache，设置 Dirty 位
  缺失时替换：
    若 Dirty=0 → 直接覆盖
    若 Dirty=1 → 先写回内存再覆盖

Write-Through 写入流程：
  写命中：写 Cache + 写内存
  写缺失：直接写内存（No-Write-Allocate）
  优点：内存始终最新，简化缓存一致性
```

## 5. Cache 性能

### 性能公式

```
平均访问时间 (AMAT: Average Memory Access Time):
  T_avg = T_hit + P_miss × T_miss

  其中:
    T_hit  = Cache 命中访问时间
    P_miss = 缺失率 (Miss Rate)
    T_miss = 缺失代价 (Miss Penalty)

示例：
  T_hit = 2 周期, T_miss = 200 周期, 缺失率 = 5%
  T_avg = 2 + 0.05 × 200 = 12 周期
  → 相比无 Cache (200 周期) 提升 16.7 倍

缺失类型 (3C):
  Compulsory (强制缺失)  ─── 首次访问，不可避免
  Capacity (容量缺失)    ─── Cache 不够大
  Conflict (冲突缺失)    ─── 映射冲突 (直接映射/组相联)
```

### 性能优化

| 方法 | 原理 | 效果 |
|------|------|------|
| 增大块大小 | 利用空间局部性 | 减少强制缺失，增加缺失代价 |
| 增大 Cache | 减少容量缺失 | 增加命中时间、成本 |
| 提高相联度 | 减少冲突缺失 | 增加命中时间 |
| 多级 Cache | 快速 L1 + 大容量 L2 | 平衡速度与容量 |
| 预取 (Prefetch) | 提前加载数据 | 隐藏延迟 |
| 编译器优化 | 循环交换、分块 | 改善局部性 |

## 6. 多级 Cache (Multi-level Cache)

### 层次设计

```
典型配置：
  L1 Cache (每核私有):
    大小: 32KB 指令 + 32KB 数据
    相联度: 4-8 路
    命中: 2-4 周期
    策略: Write-Through (或 Write-Back)

  L2 Cache (每核私有或共享):
    大小: 256KB - 1MB
    相联度: 8-16 路
    命中: 10-20 周期
    策略: Write-Back

  L3 Cache (所有核共享):
    大小: 4-32MB
    相联度: 16-32 路
    命中: 30-50 周期
    策略: Write-Back
```

### 局部与全局缺失率

```
L1 局部缺失率 = L1 缺失次数 / L1 访问次数
L1 全局缺失率 = L1 局部缺失率 (因为没有下级了)

二级 Cache 的 AMAT：
  T_avg = T_hit_L1 + P_miss_L1 × (T_hit_L2 + P_miss_L2 × T_miss_L2)

示例:
  L1: T_hit=2, 缺失率=10%
  L2: T_hit=10, 缺失率=20%
  L2 T_miss=200

  T_avg = 2 + 0.10 × (10 + 0.20 × 200)
        = 2 + 0.10 × (10 + 40) = 2 + 5 = 7 周期
```

## 7. TLB 与虚拟内存

### 虚拟地址转换

```
虚拟地址 ──→ TLB ──命中──→ 物理地址
               │
            未命中
               │
             查页表 (Page Table) ──→ 物理地址
               │
          更新 TLB  (TLB Miss 处理)
```

### 页表结构

```
传统二级页表 (32-bit, 4KB 页面)：

虚拟地址: [ 页目录索引(10) ][ 页表索引(10) ][ 页内偏移(12) ]

  CR3 ──► 页目录 (PD)
            ├── PDE[0] ──► 页表 (PT)
            ├── PDE[1]       ├── PTE[0] ──► 物理页框
            │                ├── PTE[1]
            └── ...          └── ...

x86-64 四级页表 (48-bit 地址):
  ┌──────┬──────┬──────┬──────┬──────────┐
  │ PML4 │ PDP  │  PD  │  PT  │  Offset  │
  │ 9bit │ 9bit │ 9bit │ 9bit │  12bit   │
  └──────┴──────┴──────┴──────┴──────────┘
```

### TLB

```
TLB 是页表的 Cache（地址转换缓存）：

  TLB 项: [虚拟页号(20)][物理页框(20)][权限][Dirty][Valid]

  TLB 未命中处理：
    x86: 硬件自动遍历页表 (Hardware Page Table Walk)
    MIPS: 触发异常，OS 处理 (Software TLB Miss Handler)

  TLB 与 Cache 的交互：
    物理地址索引 Cache (Physically Indexed)
    虚拟地址索引 Cache (Virtually Indexed) → 需要 TLB 在 Cache 之前

大页面 (Huge Pages):
  2MB / 1GB 页面 → TLB 覆盖更大内存 → TLB 缺失率↓
```

## 8. 主存组织

### DDR SDRAM

```
DDR 演进：
┌────────┬────────┬────────┬────────┬────────┐
│ DDR   │ DDR2  │ DDR3  │ DDR4  │ DDR5   │
├────────┼────────┼────────┼────────┼────────┤
│2000   │2003   │2007   │2014   │2020    │
│200-400│400-800│800-2133│1600-3200│3200-6400│
│2.5V   │1.8V   │1.5V   │1.2V   │1.1V    │
└────────┴────────┴────────┴────────┴────────┘
(第一行: 年份, 第二行: 频率 MHz, 第三行: 电压)

DRAM 内部结构：
  行缓冲 (Row Buffer) ─── 类似 Cache (行命中可快速访问)
  内存银行 (Banks) ─────── 并行访问
  Rank (DIMM 上的一组芯片)

访问延迟：
  tRCD (行地址到列地址) ─── 行激活延迟
  tCAS (列地址选通)     ─── 列访问延迟 (关键)
  例: DDR4-3200 CL22, tCAS = 22 × 0.625ns = 13.75ns
```

## 9. 虚拟内存与页面置换

### 页面置换算法

| 算法 | 原理 | 性能 | 实现 |
|------|------|------|------|
| FIFO | 最先进入的页面替换 | 差（Belady 异常） | 队列 |
| LRU | 最久未使用替换 | 好 | 计数器/矩阵 |
| Clock (Second Chance) | 循环扫描，参考位=1 的给第二次机会 | 近似 LRU | 循环缓冲区 |
| Working Set | 保留当前工作集 | 好 | 窗口跟踪 |
| MIN (Belady) | 替换最远未来使用 | 最优 | 无法实现 |

```
Clock 算法：
  物理页框循环排列，每项有 R (Reference) 位

  1. 指针指向第一个页框
  2. 若 R=1: 清零 R，指针前进 ← 给第二次机会
  3. 若 R=0: 替换此页

  优化: 增加 D (Dirty) 位 → 改进 Clock (避免写回干净页)

缺页率的影响因素：
  - 页大小 (4KB vs 2MB)  → 大页减少页表项
  - 工作集大小 (Working Set)
  - 物理内存大小
  - 程序局部性
```

## 10. 存储层次总结

| 层级 | 介质 | 典型容量 | 典型延迟 | 管理方式 |
|------|------|---------|---------|---------|
| 寄存器 | 触发器等 | <1KB | 0.3ns | 编译器/程序员 |
| L1 Cache | SRAM | 32-64KB | 1ns | 硬件 |
| L2 Cache | SRAM | 256KB-1MB | 4ns | 硬件 |
| L3 Cache | SRAM | 4-32MB | 12ns | 硬件 |
| 主存 | DRAM | 4-256GB | 50-100ns | 硬件+OS (页表) |
| SSD | NAND Flash | 256GB-2TB | 0.1ms | OS (文件系统) |
| HDD | 磁盘 | 1-20TB | 10ms | OS (文件系统) |

```
延迟比对（2024 典型值）：
  L1 命中:   4 周期    (1x)
  L2 命中:   12 周期   (3x)
  L3 命中:   40 周期   (10x)
  内存访问:  200 周期  (50x)
  SSD 访问:  200000 周期 (50000x)
  HDD 访问:  20000000 周期 (5,000,000x)

关键原则：
  每一级容量 ×10，延迟 ×3-5
  局部性 → 缓存的命中率是关键
```
