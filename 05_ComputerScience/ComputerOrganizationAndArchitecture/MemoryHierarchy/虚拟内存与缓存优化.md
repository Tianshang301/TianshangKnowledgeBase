# 虚拟内存与缓存优化

## 一、内存层次结构

现代计算机系统使用多层次存储结构平衡容量、速度和成本之间的矛盾。层次结构遵循"速度越快的存储器越靠近处理器、容量越小"的原则。

| 层级 | 容量 | 访问延迟 | 带宽 | 管理方式 |
|------|------|----------|------|----------|
| 寄存器 | 几百字节 | 0.3 ns | 最高 | 编译器 |
| L1缓存 | 32-64 KB | 0.5-1 ns | 1 TB/s | 硬件 |
| L2缓存 | 256-512 KB | 3-10 ns | 500 GB/s | 硬件 |
| L3缓存 | 2-32 MB | 10-40 ns | 200 GB/s | 硬件 |
| 主存(DRAM) | 8-512 GB | 50-100 ns | 50 GB/s | 硬件+OS |
| SSD | 256 GB-2 TB | 10-100 us | 5 GB/s | OS |
| 磁盘 | 1-10 TB | 5-10 ms | 200 MB/s | OS/用户 |

## 二、缓存基本原理

### 2.1 缓存组织

缓存以缓存行为单位存储数据。一个缓存行通常为64字节。缓存的三个关键设计参数决定其性能：

1. **缓存大小(C)**：总数据容量
2. **关联度(A)**：每个内存地址可映射到的缓存行数
3. **行大小(b)**：每个缓存行的字节数

缓存中的行数 $S = \frac{C}{A \times b}$。

### 2.2 缓存映射方式

| 映射方式 | 关联度 | 硬件复杂度 | 冲突失效率 | 典型场景 |
|----------|--------|-----------|-----------|----------|
| 直接映射 | 1 | 低 | 最高 | 简单嵌入式 |
| 组相联 | 2-16 | 中 | 中 | 主流通用 |
| 全相联 | 所有 | 最高 | 最低 | TLB, 小缓存 |

**组相联缓存**中，内存地址划分为：标记位(tag)、组索引(index)和块内偏移(offset)。

对于容量为 $C$、关联度为 $A$、行大小为 $b$ 的缓存，组数 $S = \frac{C}{A \times b}$，索引位数 $= \log_2 S$。

### 2.3 替换策略

- **LRU(最近最少使用)**：替换最久未访问的行，硬件实现成本随关联度增长
- **伪LRU**：使用树形近似，实现复杂度低
- **随机替换**：实现最简单，性能略低于LRU
- **NRU(最近未使用)**：近似LRU，周期性重置使用位

### 2.4 写策略

| 策略 | 行为 | 优点 | 缺点 |
|------|------|------|------|
| 写直达 | 同时写入缓存和下级 | 实现简单 | 带宽占用高 |
| 写回 | 仅写缓存，被替换时写下级 | 带宽效率高 | 实现复杂 |
| 写分配 | 写缺失时先加载到缓存 | 适合重复写入 | 浪费于一次性写入 |
| 非写分配 | 写缺失直接写入下级 | 避免缓存污染 | 不适合重复写入 |

## 三、虚拟内存

### 3.1 虚拟地址翻译

虚拟内存将进程的虚拟地址空间映射到物理地址。地址翻译通过页表(page table)实现：

$$
\text{虚拟地址} = \underbrace{\text{VPN}}_{\text{虚拟页号}} + \underbrace{\text{offset}}_{\text{页内偏移}}
$$

$$
\text{物理地址} = \text{PPN}(\text{VPN}) + \text{offset}
$$

典型页面大小为4 KB(小页)或2 MB/1 GB(大页)。

### 3.2 多级页表

64位地址空间使用多级页表减小页表占用的物理内存：

```python
# SV39页表遍历(RISC-V 39位虚拟地址)
def sv39_walk(virtual_addr, page_table_base):
    vpn = [
        (virtual_addr >> 30) & 0x1FF,  # 第2级
        (virtual_addr >> 21) & 0x1FF,  # 第1级
        (virtual_addr >> 12) & 0x1FF   # 第0级
    ]
    offset = virtual_addr & 0xFFF

    pte = read_pte(page_table_base + vpn[2] * 8)
    pte = read_pte(pte.ppn << 12 + vpn[1] * 8)
    pte = read_pte(pte.ppn << 12 + vpn[0] * 8)

    return (pte.ppn << 12) + offset
```

### 3.3 转换后备缓冲区(TLB)

TLB是最近使用的页面地址翻译的专用缓存。TLB缺失成本极高(需要多次内存访问遍历页表)，因此TLB命中率对性能至关重要。

```python
# TLB性能
AMAT_TLB = hit_time + miss_rate * miss_penalty
```

大页面(如2 MB透明巨页)是提高TLB覆盖范围的有效方法。

### 3.4 缺页处理

当访问的页面不在物理内存中时，产生缺页异常：

1. 操作系统确定页面在磁盘上的位置
2. 分配物理页面框
3. 从磁盘读取页面内容到内存(DMA)
4. 更新页表
5. 重新执行触发异常的指令

页面置换算法包括FIFO、LRU、时钟算法和工作集模型。

## 四、缓存优化技术

### 4.1 编译器辅助优化

- **循环交换**：改变循环嵌套顺序，使内存访问模式更连续
- **循环分块(tiling)**：将循环划分为小块，提高数据局部性
- **数组合并**：将多个独立数组合并为结构数组(SoA转AoS或反之)
- **预取**：提前发出内存加载请求

```python
# 循环交换示例：行主序 vs 列主序
# 友好访问方式(行主序)：连续内存访问
for i in range(N):
    for j in range(N):
        sum += A[i][j]

# 不友好访问方式：跨步大，缓存效率低
for j in range(N):
    for i in range(N):
        sum += A[i][j]
```

### 4.2 硬件预取

硬件预取器预测内存访问模式并提前加载数据到缓存：

- **流预取器**：检测顺序访问模式，预取下一个缓存行
- **步进预取器**：检测固定步长的访问模式
- **空间预取器**：利用空间局部性，预取相邻缓存行

### 4.3 缓存感知编程

```c
// 缓存行对齐结构体
struct __attribute__((aligned(64))) data_t {
    int value;
    char padding[60];  // 填充到64字节避免伪共享
};

// 伪共享问题：不同核心修改同一缓存行的不同部分
// 解决方案：每个线程的数据缓存行对齐
```

## 五、内存一致性模型

内存一致性模型定义多处理器系统中内存操作的可见性规则。x86使用**总存储顺序(TSO)**，而Arm/POWER使用**宽松一致性(relaxed consistency)**。

硬件内存屏障指令强制排序：x86的`mfence`、Arm的`dmb`和RISC-V的`fence`。

## 六、性能分析与调优

```python
# 使用perf工具分析缓存行为(Linux)
# perf stat -e cache-references,cache-misses,L1-dcache-load-miss ./program
```

缓存性能的关键指标：

- **缺失率(miss rate)**：未命中次数/总访问次数
- **平均内存访问时间(AMAT)**：$AMAT = \text{命中时间} + \text{缺失率} \times \text{缺失惩罚}$
- **每指令缺失数(MPKI)**：每千条指令的缓存缺失数
