# 超标量与乱序执行

## 一、指令级并行

指令级并行(ILP)是提高处理器性能的核心方法之一，指在单个指令流中同时执行多条不相关的指令。挖掘ILP的主要技术包括流水线、超标量发射和乱序执行。

理想情况下，处理器每个周期执行 $N$ 条指令(其中 $N$ 为发射宽度)，性能可达单发射处理器的 $N$ 倍。然而实际ILP受到多种限制：数据依赖、控制依赖和资源冲突使实际加速比远低于理论值。

## 二、超标量架构原理

### 2.1 基本结构

超标量处理器每个时钟周期取出、译码和发射多条指令。关键组件：

- **指令预取与I-Cache**：提供足够宽度的指令流
- **译码器**：多个并行译码单元，每个周期译码4-8条指令
- **发射队列**：暂存译码后的指令，等待操作数就绪
- **执行单元**：多个功能单元(ALU、FPU、Load/Store单元、分支单元等)

### 2.2 发射策略

| 策略 | 发射条件 | 硬件复杂度 | ILP利用 |
|------|----------|-----------|---------|
| 顺序发射顺序完成 | 严格按序 | 低 | 低 |
| 顺序发射乱序完成 | 按序发射，任意完成 | 中 | 中 |
| 乱序发射乱序完成 | 操作数就绪即发射 | 高 | 高 |

### 2.3 指令窗口

指令窗口是乱序处理器的核心结构——从指令队列中选取就绪指令发射。窗口大小直接影响ILP发掘能力：更大的窗口允许处理器跨越更多指令寻找并行性，但增加硬件复杂度和功耗。

现代处理器的指令窗口规模从Intel Sandy Bridge的168条到Apple M1的超过600条。

## 三、乱序执行

### 3.1 寄存器重命名

寄存器重命名消除WAR(写后读)和WAW(写后写)伪相关。物理寄存器的数量通常远多于架构寄存器：

```python
# 寄存器重命名消除WAW相关
# 代码: ADD R1, R2, R3; SUB R1, R4, R5
# 架构R1在两次写入时映射到不同物理寄存器
rename_table[R1] = p6  # 第一次写R1
# ...执行ADD...
rename_table[R1] = p7  # 第二次写R1重命名为不同物理寄存器
# 两条指令可以乱序执行，不产生冲突
```

**重排序缓冲(ROB)**按程序顺序提交指令结果，确保精确异常。

### 3.2 Tomasulo算法的现代实现

Tomasulo算法通过预留站(reservation stations)和公共数据总线(CDB)实现乱序执行：

```
步骤1: 取指译码
  指令进入译码队列
  分配ROB条目和物理寄存器(重命名)

步骤2: 发放(dispatch)
  指令进入功能单元的预留站
  如果所需操作数在CDB上广播，则直接捕获

步骤3: 发射(issue)
  预留站中所有操作数就绪的指令发送到执行单元

步骤4: 执行
  功能单元计算结果

步骤5: 写结果(CDB广播)
  结果通过CDB广播到所有预留站和ROB

步骤6: 提交(commit)
  ROB按序提交结果到架构寄存器
```

### 3.3 推测执行

处理器在执行分支指令时无法确定正确的执行路径，使用分支预测器预测方向并推测性执行。如果预测正确，推测执行超前于程序计数器；如果预测错误，需回滚推测状态(清空ROB中推测部分并恢复寄存器映射表)。

正确预测率对性能影响巨大：99%的预测率下，平均每100条指令浪费1条错误推测；95%的预测率下，每100条指令浪费5条。

## 四、取指与分支预测

### 4.1 分支预测器

现代预测器的准确率超过97%，关键设计包括：

- **BTB(分支目标缓冲)**：缓存分支指令的目标地址
- **分支方向预测**：使用两级历史表和TAGE预测器
- **间接分支预测**：预测间接跳转的目标(如虚函数调用)
- **返回地址栈(RAS)**：精确预测函数返回

### 4.2 取指带宽

维持超标量发射宽度需要从指令缓存中取出足够数量的指令。挑战在于：

- 分支指令后的对齐问题
- 跨缓存行的取指
- 指令预译码

## 五、资源管理与调度

### 5.1 功能单元配置

| 执行单元 | 数量(典型) | 延迟(周期) | 流水线化 |
|----------|-----------|-----------|----------|
| 整数ALU | 4-6 | 1 | 是 |
| 整数乘法 | 1-2 | 3-5 | 是 |
| 浮点加法 | 1-2 | 3-5 | 是 |
| 浮点乘法 | 1-2 | 4-6 | 是 |
| Load单元 | 2-3 | 3-4(延迟) | 是 |
| Store单元 | 1-2 | - | - |
| 分支单元 | 1-2 | 1 | - |

### 5.2 调度器设计

调度器在操作数就绪的指令中选择发射。选择策略影响性能和公平性：

- **最老优先**：公平，无饥饿风险
- **关键路径优先**：可改善整体性能，但实现复杂
- **混合策略**：综合性能和公平性考虑

### 5.3 内存依赖预测

Load指令的操作数(内存地址)可能依赖于之前尚未执行的Store指令。Load推测执行时，如果地址命中Store指令，需回滚。内存依赖预测器预测哪些Load可能依赖于哪些Store，帮助调度决策。

## 六、功耗与热管理

超标量和乱序执行消耗大量功耗。现代处理器集成动态电压频率调整(DVFS)、指令窗口调整、选择性关闭功能单元等功耗管理技术。暗硅(dark silicon)问题——芯片上无法同时全部通电的晶体管比例不断增加——限制了ILP的扩展。

## 七、性能模型与局限

处理器的性能模型简化：
$$
\text{IPC} = \frac{\text{发射宽度}}{\text{1 + 分支误测惩罚 + 缓存缺失惩罚 + 数据相关延迟}}
$$

实际ILP限制因素：
1. **程序固有ILP**：不同程序存在根本差异(科学计算通常高于数据库应用)
2. **数据依赖深度**：长串的数据依赖链限制并行性
3. **存储墙**：内存访问延迟远高于运算延迟
4. **功耗墙**：复杂调度逻辑消耗大量功耗
5. **延迟面积墙**：宽发射和大型ROB占用大量芯片面积
