# 现代处理器微架构

## 一、处理器微架构概述

处理器微架构是处理器内部的具体实现方式，在给定的指令集架构(ISA)约束下通过硬件设计追求性能、功耗和面积的最优平衡。微架构设计涉及流水线组织、执行单元设计、缓存系统、分支预测和指令调度等核心问题。

摩尔定律的放缓迫使微架构创新从单纯依赖工艺进步转向架构层面的深度优化。现代处理器微架构的设计空间包括：功耗效率优先(手机 SoC)、单线程性能优先(高性能桌面 CPU)和吞吐量优先(GPU/AI 加速器)。

## 二、流水线微架构

### 2.1 经典五级流水线

经典的 RISC 五级流水线包括：
1. **IF(取指)**：从指令缓存读取指令
2. **ID(译码)**：解析指令、读取寄存器
3. **EX(执行)**：ALU 运算或地址计算
4. **MEM(访存)**：数据缓存加载/存储
5. **WB(写回)**：结果写回寄存器

理想情况下每个时钟周期完成一条指令，但流水线冒险会打乱执行节奏。

### 2.2 流水线冒险

| 冒险类型 | 原因 | 解决方案 |
|----------|------|----------|
| 结构冒险 | 硬件资源冲突 | 资源复制(多端口、多缓存) |
| 数据冒险 | 指令间数据依赖 | 转发、旁路、流水线停顿 |
| 控制冒险 | 分支跳转 | 分支预测、延迟槽 |

数据冒险的三种类型(Read After Write, Write After Read, Write After Write)通过转发网络解决：

```python
# 数据转发示例：ADD R1, R2, R3; SUB R4, R1, R5
# EX 阶段结果直接转发到下一个指令的 EX 阶段输入
sub_rs1 = EX_result  # 从前一条指令的 EX 输出转发，无需等待 WB
```

## 三、超标量与乱序执行

### 3.1 超标量实现

超标量处理器每个周期发射多条指令。指令发射宽度从早期的2路(如 Pentium)发展到现代处理器的6-8路(如 Apple M 系列)。超标量设计面临发射槽利用率、指令窗口大小和寄存器文件端口等物理限制。

### 3.2 Tomasulo 算法

Tomasulo 算法是乱序执行的核心技术，通过**寄存器重命名**消除 WAR/WAW 伪相关：

1. **预留站**：每个执行单元关联的缓冲区，保存等待发射的指令
2. **公共数据总线(CDB)**：将计算结果广播给所有等待单元
3. **寄存器重命名**：将架构寄存器映射到更多的物理寄存器

提交阶段使用**重排序缓冲(ROB)**按程序顺序提交指令，确保精确异常。

### 3.3 寄存器重命名示例

```
指令序列：          寄存器重命名后：
ADD R1, R2, R3     ADD p6, p4, p5    # R1 → p6
SUB R4, R1, R5     SUB p7, p6, p8    # 通过 p6读取 R1的值
ADD R1, R6, R7     ADD p9, p10, p11  # R1 → p9, 与之前不再相关
```

## 四、分支预测技术

### 4.1 静态与动态预测

```python
# 2位饱和计数器分支预测器
class SaturatingCounter:
    def __init__(self):
        self.state = 0b01  # 弱不取: 00=强不取, 01=弱不取, 10=弱取, 11=强取

    def predict(self):
        return self.state >= 0b10

    def update(self, taken):
        if taken:
            self.state = min(3, self.state + 1)
        else:
            self.state = max(0, self.state - 1)
```

### 4.2 高级分支预测器

现代处理器使用两级自适应预测器：

- **局部历史表**：为每个分支维护单独的历史模式
- **全局历史**：所有分支的全局历史寄存器
- **选择器**：根据预测准确性在局部和全局预测之间选择

TAGE 预测器使用多个不同历史长度的预测表，性能接近理论极限，广泛应用于商用处理器中。

## 五、缓存系统微架构

| 层级 | 容量 | 延迟 | 关联度 | 策略 |
|------|------|------|--------|------|
| L1 I-cache | 32-64 KB | 2-4 周期 | 4-8路 | 虚拟索引物理标记 |
| L1 D-cache | 32-64 KB | 3-5 周期 | 4-8路 | 写直通或写回 |
| L2 | 256-512 KB | 10-15 周期 | 8-16路 | 统一、写回 |
| L3 | 2-32 MB | 30-50 周期 | 12-24路 | 共享、非包含 |

**预取(prefetching)**技术缓存可能被访问的数据：

- **硬件预取**：检测内存访问模式(如顺序流、步进模式)
- **软件预取**：编译器插入预取指令
- **空间预取**：访问一个缓存行时预取相邻行

## 六、现代处理器实例

| 微架构 | 发布年 | 制造工艺 | 流水线阶段 | 发射宽度 | 核心特性 |
|--------|--------|----------|-----------|----------|----------|
| Intel Golden Cove | 2021 | Intel 7 | 18 | 6 | 混合架构，P-core |
| AMD Zen 4 | 2022 | TSMC 5nm | 19 | 6 | 大缓存，AVX-512 |
| Apple M2 | 2022 | TSMC 5nm | 16 | 8 | 宽发射，低功耗 |
| Arm Cortex-X3 | 2022 | TSMC 4nm | 12 | 6 | 高性能 Arm 核心 |

Apple M 系列的处理器的性能/功耗比表现突出，得益于其超宽发射和巨大的 ROB(超过600条目)。

## 七、特殊化微架构

### 7.1 GPU 微架构

GPU 采用 SIMT(单指令多线程)架构，通过大规模线程并行隐藏延迟。核心特征包括：数百个 CUDA 核心、分层线程组织结构(线程束 warp)、高带宽显存(HBM 或 GDDR)。

### 7.2 AI 加速器

神经网络加速器针对矩阵乘法优化。张量处理单元(TPU)的核心是脉动阵列(Systolic Array)：

$$
C_{i,j} = \sum_k A_{i,k} \times B_{k,j}
$$

脉动阵列通过规整的数据流减少内存带宽需求，每个处理单元(PE)执行简单的乘加运算。
