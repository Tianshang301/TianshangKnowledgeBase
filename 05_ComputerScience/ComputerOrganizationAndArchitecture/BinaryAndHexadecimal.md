# 二进制与十六进制 (Binary and Hexadecimal)

## 一、数制基础 (Number System Basics)

计算机内部使用二进制（Binary, Base-2），因为电子电路只有两种稳定状态：高电平（1）和低电平（0）。十六进制（Hexadecimal, Base-16）作为二进制的人可读简化表示，每 4 位二进制对应 1 位十六进制，在计算机工程中广泛使用。

### 位置计数法 (Positional Notation)

任意进制数值可表示为按权展开式：

$$N = \sum_{i=-m}^{n-1} d_i \times b^i$$

其中 $b$ 为基数（Radix），$d_i$ 为第 $i$ 位的数字（$0 \leq d_i < b$），$n$ 为整数位数，$m$ 为小数位数。例如十进制 $123.45_{10}$：
$$123.45 = 1 \times 10^2 + 2 \times 10^1 + 3 \times 10^0 + 4 \times 10^{-1} + 5 \times 10^{-2}$$

二进制 $1011.01_2$ 的展开：
$$1011.01_2 = 1 \times 2^3 + 0 \times 2^2 + 1 \times 2^1 + 1 \times 2^0 + 0 \times 2^{-1} + 1 \times 2^{-2} = 11.25_{10}$$

### 为什么计算机用二进制

- **物理实现简单**：晶体管只有导通/截止两种状态
- **抗干扰性强**：电压高低易于区分，允许较大噪声容限
- **布尔代数完备**：逻辑代数 $AND/OR/NOT$ 可直接用电路实现
- **可靠性高**：二值信号检测简单，错误率低

## 二、进制转换 (Base Conversion)

### 完整转换方法表

| 转换方向 | 方法 | 示例 |
|---------|------|------|
| 二进制→十进制 | 按权展开求和 | $1011_2 = 1\times2^3 + 0\times2^2 + 1\times2^1 + 1\times2^0 = 11_{10}$ |
| 十进制→二进制（整数） | 除 2 取余，倒序排列 | $13 \div 2 = 6\cdots1 \rightarrow 6 \div 2 = 3\cdots0 \rightarrow 3 \div 2 = 1\cdots1 \rightarrow 1 \div 2 = 0\cdots1 \rightarrow 1101$ |
| 十进制→二进制（小数） | 乘 2 取整，顺序排列 | $0.6875 \times 2 \rightarrow 循环取整$ |
| 二进制→十六进制 | 每 4 位一组，高位补 0 | $1101\;1011_2 = DB_{16}$ |
| 十六进制→二进制 | 每位转 4 位 | $A3_{16} = 1010\;0011_2$ |
| 十进制→十六进制 | 除 16 取余 | $255 \div 16 = 15\cdots15 \rightarrow FF_{16}$ |
| 八进制→二进制 | 每 3 位一组 | $73_8 = 111\;011_2$ |
| 二进制→八进制 | 每 3 位一组 | $110\;101_2 = 65_8$ |

### 小数转换详细过程 (Fractional Conversion)

以 $0.6875_{10} \rightarrow$ 二进制为例：
$$0.6875 \times 2 = 1.375 \rightarrow \text{整数部分 } 1$$
$$0.375 \times 2 = 0.75 \rightarrow \text{整数部分 } 0$$
$$0.75 \times 2 = 1.5 \rightarrow \text{整数部分 } 1$$
$$0.5 \times 2 = 1.0 \rightarrow \text{整数部分 } 1$$

结果：$0.6875_{10} = 0.1011_2$

### 快速转换技巧 (Quick Conversion Tricks)

```text
二进制 ↔ 十六进制:
  0x4A = 0100 1010
  二进制: 从右向左每 4 位一组
  十六进制: 每位展开成 4 位二进制

常用 2 的幂:
  2^0  = 1        2^10 = 1024 (1 KB)
  2^1  = 2        2^16 = 65536 (64 KB)
  2^2  = 4        2^20 = 1,048,576 (1 MB)
  2^3  = 8        2^30 = 1,073,741,824 (1 GB)
  2^4  = 16       2^32 = 4,294,967,296 (4 GB)
  2^8  = 256      2^40 = 1 TB
```

## 三、二进制运算 (Binary Operations)

### 算术运算 (Arithmetic)

二进制加法规则：
$$0+0=0, \quad 0+1=1, \quad 1+0=1, \quad 1+1=0\ \text{进位 }1$$

```text
  101101  (45)
+ 011011  (27)
--------
 1001000  (72)
```

二进制乘法：移位相加
```text
    101   (5)
  × 011   (3)
  -----
    101   (1 × 101)
   1010   (1 × 101, 左移 1 位)
+ 00000   (0 × 101, 左移 2 位)
--------
   1111   (15)
```

### 位运算 (Bitwise Operations)

| 运算 | 符号 | 规则 | 示例（8 位） | 用途 |
|------|------|------|-------------|------|
| AND（与） | `&` | 全 1 为 1 | $1010 \;1100 \;\&\; 1100\;1010 = 1000\;1000$ | 掩码/清零 |
| OR（或） | `\|` | 有 1 为 1 | $1010 \;1100 \;\|\; 1100\;1010 = 1110\;1110$ | 置位 |
| XOR（异或） | `^` | 不同为 1 | $1010 \;1100 \;\text{^}\; 1100\;1010 = 0110\;0110$ | 翻转/加密 |
| NOT（非） | `~` | 逐位取反 | $\sim 1010\;1100 = 0101\;0011$（取所有位）| 补码 |
| 左移 | `<<` | 低位补 0 | $0000\;1010 \ll 2 = 0010\;1000$ | 乘 $2^n$ |
| 算术右移 | `>>` | 高位补符号位 | $1000\;1010 \gg 1 = 1100\;0101$ | 有符号除 $2^n$ |
| 逻辑右移 | `>>>` | 高位补 0 | $1000\;1010 \gg\gg 1 = 0100\;0101$ | 无符号除 $2^n$ |

### 位运算经典应用

```text
判断奇偶:      n & 1              (n % 2 的快速替代，n 为奇数则返回 1)
去除末位 1:     n & (n-1)          (统计二进制中 1 的个数)
取最低位 1:     n & (-n)           (Lowbit 操作，BIT 树状数组)
判断 2 的幂:    n > 0 && (n & (n-1)) == 0
交换 a 和 b:    a ^= b; b ^= a; a ^= b;  (无临时变量)
计算 2 的 n 次幂: 1 << n
```

统计二进制中 1 的个数（Popcount）：
```text
n = 10110110 (182)
n & (n-1) = 10110110 & 10110101 = 10110100  (去掉了最后一个 1)
重复直到 n = 0，计数次数
CPU 内置指令: POPCNT (x86), CNT (ARM)
```

## 四、补码表示 (Two's Complement)

补码是计算机表示有符号整数的标准方式，几乎所有现代 CPU 都使用补码。

### 转换规则

正数补码 = 原码本身
负数补码 = 原码取反加一（即 $2^n - |x|$）

示例（8 位）：$-5$ 的补码
- 原码 $+5$: $0000\;0101$
- 取反: $1111\;1010$
- 加一: $1111\;1011 = -5$（十六进制: $0xFB$）

验证：$(-5) + 5 = 1111\;1011 + 0000\;0101 = 1\;0000\;0000 \equiv 0 \pmod{2^8}$

### 补码的性质

$$[x]_补 + [-x]_补 \equiv 0 \pmod{2^n}$$

- $n$ 位有符号数范围：$-2^{n-1}$ 到 $2^{n-1}-1$
  - 8 位：$-128$ 到 $127$
  - 16 位：$-32768$ 到 $32767$
  - 32 位：$-2,147,483,648$ 到 $2,147,483,647$
- $0$ 的表示唯一（$0000\;0000$），没有正零/负零之分
- 最高位为符号位（0 正 1 负）
- 溢出判断：两个正数相加得负数（正溢出），或两个负数相加得正数（负溢出）

### 符号扩展 (Sign Extension)

将 $n$ 位有符号数扩展为 $m$ 位（$m > n$），高位填充符号位：
$$6_{10} = 0110 \rightarrow 0000\;0110$$
$$-6_{10} = 1010 \rightarrow 1111\;1010$$

### 加减法统一电路

补码的设计使得减法可以转化为加法，无需专门的减法器：
$$A - B = A + (-B)$$

CPU ALU 的实现：
```text
加法:  result = A + B
减法:  result = A + (~B + 1)  即 A + (-B 的补码)
```

## 五、十六进制的工程应用 (Hexadecimal in Engineering)

十六进制在计算机工程中无处不见：
- **内存地址表示**：`0x7FFF_FFFF` 比十进制 2147483647 更直观
- **颜色值**：`#FF6600`（RGB 各 8 位，红色 FF，绿色 66，蓝色 00）
- **MAC 地址**：`00:1A:2B:3C:4D:5E`
- **IPv6 地址**：`2001:0db8:85a3::8a2e:0370:7334`
- **机器码/汇编**：`0x89 0xE5`（x86 MOV EBP, ESP 指令编码）
- **Unicode**：`U+4E2D` 表示汉字"中"
- **调试信息**：错误码 `0xC0000005` 表示访问违规

## 六、实际工程中的二进制应用案例

### 网络掩码 (Subnet Mask)

IP 地址与子网掩码按位 AND 得到网络地址：
```text
IP:       192.168.1.130   = 11000000.10101000.00000001.10000010
Mask:     255.255.255.128 = 11111111.11111111.11111111.10000000
Network:  192.168.1.128   = 11000000.10101000.00000001.10000000
Host:     0.0.0.2         = 00000000.00000000.00000000.00000010
```

### 权限掩码 (Bitmask Permission)

用位表示权限，每个位代表一个权限开关：
```text
权限定义:
  READ    = 1 << 0 = 0001 (1)
  WRITE   = 1 << 1 = 0010 (2)  
  EXECUTE = 1 << 2 = 0100 (4)
  DELETE  = 1 << 3 = 1000 (8)

检查权限: user_permission & READ
设置权限: user_permission |= WRITE
取消权限: user_permission &= ~EXECUTE

示例:
  permission = READ | WRITE = 0011 (3)
  has_exec = (permission & EXECUTE) != 0  → False
  has_read = (permission & READ) != 0     → True
```

### 位图 (Bitmap) 在数据库中的应用

PostgreSQL 使用位图索引和位图扫描加速低基数列查询：
```text
性别列: M(0), F(1), Unknown(2)
位图索引: 每个性别的数据对应一个 1/0 位数组
  过滤 WHERE gender = 'M' → 通过位图快速定位所属行
  多条件 AND/OR → 位图快速位运算合并结果
```

## 七、不同进制在编程语言中的字面量表示

```python
# Python 进制字面量
binary  = 0b1010           # 10
octal   = 0o12             # 10
decimal = 10               # 10
hex     = 0xA              # 10

# 格式化输出
bin(10)  # '0b1010'
oct(10)  # '0o12'
hex(10)  # '0xa'
int('FF', 16)  # 255

# Bit 运算
a = 0b1100
b = 0b1010
c = a & b   # 0b1000 (8)
d = a | b   # 0b1110 (14)
e = a ^ b   # 0b0110 (6)
f = ~a & 0xF  # 0b0011 (3) — 注意取反后高位补 1，需掩码
g = a << 2  # 0b110000 (48)
h = a >> 1  # 0b0110 (6)
```
