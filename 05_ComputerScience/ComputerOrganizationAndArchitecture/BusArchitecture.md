# 总线架构

## 1. 系统总线层次结构

### 1.1 总线分类

```
┌─────────────────────────────────────────────────────────────────┐
│                        系统总线层次                              │
├─────────────────────────────────────────────────────────────────┤
│  处理器总线 (Processor Bus)                                      │
│  ├── Front-Side Bus (FSB) [已淘汰]                              │
│  ├── Ring Bus (Intel Sandy Bridge+)                             │
│  ├── Mesh Interconnect (Intel Xeon Scalable)                    │
│  └── Infinity Fabric (AMD)                                      │
├─────────────────────────────────────────────────────────────────┤
│  内存总线 (Memory Bus)                                           │
│  ├── DDR4/DDR5 Channel                                          │
│  ├── LPDDR4/LPDDR5 (移动端)                                     │
│  └── HBM (高带宽内存, GPU/HPC)                                  │
├─────────────────────────────────────────────────────────────────┤
│  I/O 总线 (I/O Bus)                                             │
│  ├── PCIe (主要I/O互连)                                         │
│  ├── USB (通用串行总线)                                          │
│  ├── Thunderbolt                                                │
│  └── NVMe (存储)                                                │
├─────────────────────────────────────────────────────────────────┤
│  片间互连 (Inter-Chip)                                          │
│  ├── CXL (Compute Express Link)                                 │
│  ├── UCIe (Universal Chiplet Interconnect Express)              │
│  └── EMIB/CoWoS (先进封装)                                      │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 传统 Front-Side Bus (已淘汰)

```
┌─────────┐    FSB (64-bit, up to 1600 MT/s)    ┌──────────────┐
│   CPU   │◄────────────────────────────────────►│  Northbridge │
└─────────┘                                      │  (MCH/GMCH)  │
                                                 └──────┬───────┘
                                                        │
                          ┌─────────────────────────────┼─────────────┐
                          │                             │             │
                    ┌─────┴──────┐              ┌───────┴──────┐  ┌───┴───┐
                    │ Memory Bus │              │   PCIe x16   │  │ DMI   │
                    │  DDR2/DDR3 │              │   (GPU)      │  │       │
                    └────────────┘              └──────────────┘  └───┬───┘
                                                                     │
                                                              ┌──────┴──────┐
                                                              │ Southbridge │
                                                              │   (ICH)     │
                                                              └──────┬──────┘
                                                         ┌───────┬───┴───┬───────┐
                                                       USB    SATA    Audio   LAN
```

**FSB 问题**：
- 带宽共享，所有设备竞争同一总线
- 延迟高，信号需经过北桥
- 功耗大，高频并行总线

### 1.3 Ring Bus (Intel)

```
┌─────────────────────────────────────────────────────┐
│                    Ring Bus                          │
│                                                     │
│    ┌──────┐    ┌──────┐    ┌──────┐    ┌──────┐    │
│    │ Core │    │ Core │    │ Core │    │ Core │    │
│    │  0   │    │  1   │    │  2   │    │  3   │    │
│    └──┬───┘    └──┬───┘    └──┬───┘    └──┬───┘    │
│       │           │           │           │         │
│   ◄───┴───────────┴───────────┴───────────┴───►     │
│   │              Ring (双向)                    │    │
│   ►─────────────────────────────────────────────◄   │
│       │           │           │           │         │
│    ┌──┴───┐    ┌──┴───┐    ┌──┴───┐    ┌──┴───┐    │
│    │  L3  │    │  L3  │    │  L3  │    │  L3  │    │
│    │Slice │    │Slice │    │Slice │    │Slice │    │
│    └──────┘    └──────┘    └──────┘    └──────┘    │
│                                                     │
│    ┌──────────┐  ┌──────────┐  ┌──────────┐        │
│    │  IMC     │  │  IMC     │  │  PCIe    │        │
│    │ (Memory) │  │ (Memory) │  │  Agent   │        │
│    └──────────┘  └──────────┘  └──────────┘        │
└─────────────────────────────────────────────────────┘
```

**Ring Bus 特点**：
- 双向环形拓扑，每个节点有独立接入点
- 延迟可预测：最坏情况 = N/2 跳
- 适合中等核心数（4-24核）

### 1.4 Mesh Interconnect

```
┌──────┐    ┌──────┐    ┌──────┐    ┌──────┐
│Core 0│◄──►│Core 1│◄──►│Core 2│◄──►│Core 3│
│ + L3 │    │ + L3 │    │ + L3 │    │ + L3 │
└──┬───┘    └──┬───┘    └──┬───┘    └──┬───┘
   │           │           │           │
   ▼           ▼           ▼           ▼
┌──────┐    ┌──────┐    ┌──────┐    ┌──────┐
│Core 4│◄──►│Core 5│◄──►│Core 6│◄──►│Core 7│
│ + L3 │    │ + L3 │    │ + L3 │    │ + L3 │
└──┬───┘    └──┬───┘    └──┬───┘    └──┬───┘
   │           │           │           │
   ▼           ▼           ▼           ▼
┌──────┐    ┌──────┐    ┌──────┐    ┌──────┐
│ IMC  │    │ IMC  │    │ PCIe │    │ IIO  │
│      │    │      │    │      │    │      │
└──────┘    └──────┘    └──────┘    └──────┘
```

**Mesh 优势**：
- 更好的可扩展性（支持 56+ 核心）
- 更低的平均延迟（局部通信更快）
- 更高的聚合带宽

## 2. PCIe 协议

### 2.1 PCIe 架构

```
┌─────────────────────────────────────────────────────────┐
│                    Root Complex                          │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐   │
│  │  CPU    │  │  CPU    │  │  IMC    │  │  IMC    │   │
│  │ Core 0  │  │ Core 1  │  │ Ch 0-1  │  │ Ch 2-3  │   │
│  └─────────┘  └─────────┘  └─────────┘  └─────────┘   │
└───────────────────────┬─────────────────────────────────┘
                        │ PCIe Root Port
          ┌─────────────┼─────────────┐
          │             │             │
    ┌─────┴─────┐ ┌─────┴─────┐ ┌─────┴─────┐
    │  PCIe x16 │ │  PCIe x4  │ │  PCIe x1  │
    │  (GPU)    │ │  (NVMe)   │ │  (NIC)    │
    └───────────┘ └───────────┘ └───────────┘
```

### 2.2 PCIe Lane 结构

```
每个 Lane (差分对):
┌─────────────────────────────────────────┐
│              PCIe Lane                   │
│  ┌───────────────────────────────────┐  │
│  │  TX+ (发送差分对正极)              │  │
│  │  TX- (发送差分对负极)              │  │
│  │  RX+ (接收差分对正极)              │  │
│  │  RX- (接收差分对负极)              │  │
│  └───────────────────────────────────┘  │
│                                         │
│  全双工: 同时收发                        │
│  编码: 128b/130b (Gen3+)               │
└─────────────────────────────────────────┘

带宽计算:
Gen3: 8 GT/s × 128/130 ≈ 7.88 Gbps per lane
Gen4: 16 GT/s × 128/130 ≈ 15.75 Gbps per lane
Gen5: 32 GT/s × 128/130 ≈ 31.51 Gbps per lane
Gen6: 64 GT/s × 242/256 ≈ 60.5 Gbps per lane (PAM4编码)

x16 双向带宽:
Gen3: 7.88 × 16 × 2 ≈ 31.5 GB/s
Gen4: 15.75 × 16 × 2 ≈ 63.0 GB/s
Gen5: 31.51 × 16 × 2 ≈ 126.0 GB/s
```

### 2.3 TLP (Transaction Layer Packet)

```
TLP 包格式:
┌──────────┬──────────┬──────────┬──────────┬──────────┐
│  Byte 0  │  Byte 1  │  Byte 2  │  Byte 3  │  ...     │
├──────────┴──────────┴──────────┴──────────┴──────────┤
│ Fmt[2:0] │ Type[4:0]│ TC[2:0]  │ Attr     │ Length   │
│ (格式)    │ (类型)   │ (流量类) │ (属性)   │ (长度)   │
├──────────────────────────────────────────────────────┤
│ Requester ID (16-bit) │ Tag (8-bit)  │ Last BE/First BE │
├──────────────────────────────────────────────────────┤
│ Address (32/64-bit) 或 Completer ID                   │
├──────────────────────────────────────────────────────┤
│ Data Payload (0-4096 bytes)                           │
├──────────────────────────────────────────────────────┤
│ ECRC (可选, 32-bit)                                   │
└──────────────────────────────────────────────────────┘

TLP 类型:
  Memory Read/Write (MRd/MWr)
  I/O Read/Write (IORd/IOWr)
  Configuration Read/Write (CfgRd/CfgWr)
  Completion (Cpl/CplD)
  Message (Msg/MsgD)
```

### 2.4 Configuration Space 与 BAR

```
PCIe Configuration Space (256 bytes, 可扩展到 4KB):
┌─────────────────────────────────────────────────────┐
│ Offset │ 内容                                       │
├────────┼─────────────────────────────────────────────┤
│ 0x00   │ Vendor ID / Device ID                      │
│ 0x08   │ Class Code / Revision ID                   │
│ 0x0C   │ Header Type / Latency Timer                │
│ 0x10   │ BAR0 (Base Address Register 0)             │
│ 0x14   │ BAR1                                       │
│ 0x18   │ BAR2                                       │
│ 0x1C   │ BAR3                                       │
│ 0x20   │ BAR4                                       │
│ 0x24   │ BAR5                                       │
│ 0x28   │ CardBus CIS Pointer                        │
│ 0x2C   │ Subsystem Vendor ID / Subsystem ID         │
│ 0x30   │ Expansion ROM Base Address                 │
│ 0x34   │ Capabilities Pointer                       │
│ 0x3C   │ Interrupt Line / Interrupt Pin             │
└────────┴─────────────────────────────────────────────┘

BAR 机制:
  系统启动时, BIOS/OS 枚举所有PCIe设备
  读取 BAR 确定设备需要多少地址空间
  分配物理地址并写回 BAR

  BAR[0]: Memory Base (32/64-bit)
  BAR[1]: Memory Limit
  BAR[2]: Prefetchable Memory Base
  BAR[3]: Prefetchable Memory Limit
  BAR[4]: I/O Base
  BAR[5]: I/O Limit
```

### 2.5 DMA (Direct Memory Access)

```
DMA 传输流程:
┌──────┐                    ┌──────────┐                    ┌──────────┐
│  CPU │                    │   DMA    │                    │  Memory  │
└──┬───┘                    │Controller│                    └────┬─────┘
   │                        └────┬─────┘                         │
   │ 1. 配置DMA                   │                               │
   │   - 源地址                    │                               │
   │   - 目的地址                  │                               │
   │   - 传输长度                  │                               │
   │   - 传输方向                  │                               │
   │──────────────────────────────►                               │
   │                             │                               │
   │ 2. DMA 读取源数据            │                               │
   │                             │──── Memory Read TLP ──────────►│
   │                             │◄─── Completion with Data ─────│
   │                             │                               │
   │ 3. DMA 写入目的地址          │                               │
   │                             │──── Memory Write TLP ─────────►│
   │                             │                               │
   │ 4. DMA 完成中断              │                               │
   │◄────────────────────────────│                               │
   │   (MSI/MSI-X Interrupt)     │                               │

DMA 优势:
  - CPU 不参与数据搬运, 可执行其他任务
  - 支持 Scatter-Gather (分散-聚集)
  - 零拷贝网络传输
```

## 3. 内存总线

### 3.1 DDR4 信号与时序

```
DDR4 Channel:
┌─────────────────────────────────────────────────────────┐
│                    Memory Controller                     │
└───────────────────────┬─────────────────────────────────┘
                        │
    ┌───────────────────┼───────────────────┐
    │                   │                   │
    │   ┌───────────────┴───────────────┐   │
    │   │     DDR4 Channel (64-bit)     │   │
    │   │  ┌─────────────────────────┐  │   │
    │   │  │ Data Bus: DQ[63:0]     │  │   │
    │   │  │ Data Strobe: DQS[8:0]  │  │   │
    │   │  │ Address: A[17:0]       │  │   │
    │   │  │ Bank Address: BG[1:0]  │  │   │
    │   │  │ Bank Group: BA[1:0]    │  │   │
    │   │  │ Command: CAS, RAS, WE  │  │   │
    │   │  │ Clock: CK_t, CK_c      │  │   │
    │   │  └─────────────────────────┘  │   │
    │   │                               │   │
    │   │  ┌─────────┐   ┌─────────┐   │   │
    │   │  │  DIMM 0 │   │  DIMM 1 │   │   │
    │   │  │ (Rank 0)│   │ (Rank 1)│   │   │
    │   │  │ 8 chips │   │ 8 chips │   │   │
    │   │  └─────────┘   └─────────┘   │   │
    │   └───────────────────────────────┘   │
    │                                       │
    └───────────────────────────────────────┘

DDR4 时序参数:
  tCAS (CL): CAS Latency - 列地址选通延迟
  tRCD: RAS to CAS Delay - 行到列延迟
  tRP: Row Precharge - 行预充电时间
  tRAS: Row Active Time - 行激活时间

  总延迟 = tRCD + CL
  DDR4-3200: CL=22, tRCD=22, 延迟 ≈ 13.75ns
```

### 3.3 Rank Interleaving

```
Rank 交错访问:
时间轴:
  Cycle:  1    2    3    4    5    6    7    8
  Rank0: [RAS] [CAS] [DATA] [PRE] [RAS] [CAS] [DATA] [PRE]
  Rank1: [PRE] [RAS] [CAS] [DATA] [PRE] [RAS] [CAS] [DATA]
         └─────────────────────┘   └─────────────────────┘
              Rank0 传输                Rank0 传输

  交错允许 Rank1 在 Rank0 数据传输时激活
  隐藏行激活和预充电延迟
  有效带宽提升 20-30%
```

### 4.3 CXL.cache 一致性

```
CXL.cache 缓存状态:
  - UD (Unique Dirty): 独占且脏
  - UC (Unique Clean): 独占且干净
  - SD (Shared Dirty): 共享且脏
  - SC (Shared Clean): 共享且干净
  - I (Invalid): 无效

M2S (Master to Slave) 请求:
  - RdClean: 读取干净副本
  - RdOwn: 读取并获取独占权
  - RdShared: 读取共享副本
  - WrInv: 写入并使其他副本无效
  - CLFlush: 刷新缓存行

S2M (Slave to Master) 响应:
  - D2S Data: 数据响应
  - NCB (Non-Coherent Bypass): 非一致性数据
  - Completion: 操作完成确认
```

## 5. 缓存一致性协议

### 5.1 MESI 协议

```
MESI 状态机:
每个缓存行有 4 个状态:

┌─────────────────────────────────────────────────────────┐
│                   MESI 状态转换                          │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  M (Modified):  本地修改, 其他副本无效                   │
│  E (Exclusive): 独占, 未修改, 其他副本无效              │
│  S (Shared):    共享, 多个副本存在                      │
│  I (Invalid):   无效, 数据不可用                        │
│                                                         │
│  状态转换:                                              │
│  I → E: 本地读取, 无其他副本 (BusRd, 无响应)           │
│  I → S: 本地读取, 有其他副本 (BusRd, 有响应)           │
│  E → M: 本地写入 (无需总线事务)                        │
│  M → I: 其他处理器读取 (BusRd, 需写回)                 │
│  S → M: 本地写入 (BusUpgr, 使其他副本无效)             │
│  M → S: 其他处理器读取 (BusRd, 写回并共享)             │
│  E → S: 其他处理器读取 (BusRd, 转为共享)               │
│  S → I: 其他处理器写入 (BusUpgr)                       │
└─────────────────────────────────────────────────────────┘

状态转换图:
        BusRd/Flush
    ┌──────────────────┐
    │                  ▼
┌───┴───┐  PrWr    ┌───────┐  PrRd    ┌───────┐
│   I   │────────►│   M   │────────►│   E   │
└───┬───┘         └───┬───┘         └───┬───┘
    │                 │                 │
    │ PrRd            │ BusRd           │ BusRd
    │ (其他有副本)     │ (写回)          │ (无响应)
    ▼                 ▼                 ▼
┌───────┐  BusUpgr ┌───────┐  PrRd    ┌───────┐
│   S   │◄────────│   M   │────────►│   E   │
└───────┘         └───────┘         └───────┘
```

### 5.2 MOESI 协议 (AMD)

```
MOESI 增加 O (Owned) 状态:
  O: 拥有最新数据, 其他副本可能为 S (脏)

  优势: S → O 不需要写回内存
  减少内存带宽消耗

  ┌─────────────────────────────────────────┐
  │  M: 本地修改, 独占                      │
  │  O: 拥有最新数据, 其他有S副本 (脏)     │
  │  E: 独占, 干净                          │
  │  S: 共享, 可能过时 (如果O存在)          │
  │  I: 无效                                │
  └─────────────────────────────────────────┘
```

### 5.3 Snoop-based vs Directory-based

```
Snoop-based (监听协议):
┌──────┐    ┌──────┐    ┌──────┐
│Core 0│    │Core 1│    │Core 2│
│Cache │    │Cache │    │Cache │
└──┬───┘    └──┬───┘    └──┬───┘
   │           │           │
   └─────────┬─┴───────────┘
             │
      ┌──────┴──────┐
      │  Snoop Bus  │ (广播)
      └──────┬──────┘
             │
      ┌──────┴──────┐
      │   Memory    │
      └─────────────┘

特点: 所有请求广播到所有核心
优点: 延迟低, 实现简单
缺点: 扩展性差 (总线带宽限制)

Directory-based (目录协议):
┌──────┐    ┌──────┐    ┌──────┐
│Core 0│    │Core 1│    │Core 2│
│Cache │    │Cache │    │Cache │
└──┬───┘    └──┬───┘    └──┬───┘
   │           │           │
   └─────────┬─┴───────────┘
             │
      ┌──────┴──────┐
      │  Directory  │ (集中式状态)
      │  ┌────────┐ │
      │  │Cache   │ │
      │  │Line    │ │
      │  │State   │ │
      │  └────────┘ │
      └──────┬──────┘
             │
      ┌──────┴──────┐
      │   Memory    │
      └─────────────┘

特点: 维护每个缓存行的共享者列表
优点: 扩展性好 (无广播风暴)
缺点: 额外存储开销, 延迟略高
```
