# XR交互设计

XR（Extended Reality）交互设计是为虚拟现实、增强现实和混合现实环境设计用户交互方式的学科，需要考虑三维空间、沉浸感和自然交互等独特因素。

## 3D UI 设计原则

### 空间层次

3D 用户界面需要建立清晰的空间层次关系：

- **世界空间 UI**：固定在虚拟世界中的界面元素
- **头部锁定 UI**：跟随用户头部移动的 HUD 式界面
- **手部锁定 UI**：附着在手部控制器的菜单
- **世界锚定 UI**：固定在现实世界位置的 AR 界面

### 深度感知

- **视差线索**：双眼视差提供深度信息
- **运动视差**：头部移动产生的深度变化
- **遮挡关系**：近物遮挡远物
- **相对大小**：物体随距离缩小
- **大气透视**：远处物体对比度降低

### 设计尺度

- **人体尺度**：界面元素应符合人体工程学
- **可达范围**：常用交互元素在手臂可达范围内
- **舒适视角**：重要内容在自然视线范围内（水平线下 15°-30°）
- **信息密度**：避免 3D 空间中的信息过载

### 布局策略

```
舒适区域布局：
┌─────────────────────────────────┐
│          次要信息区域            │
│    ┌───────────────────────┐    │
│    │     主要信息区域       │    │
│    │   (视线自然落点)       │    │
│    └───────────────────────┘    │
│          交互操作区域            │
│      (手臂自然可达范围)          │
└─────────────────────────────────┘
```

## 手势交互设计

### 手势分类

- **静态手势**：保持特定手型（握拳、张开、指向）
- **动态手势**：包含运动轨迹（挥手、画圆、捏合）
- **双手手势**：需要双手配合（缩放、旋转）
- **触觉手势**：与虚拟物体接触（触摸、抓取、推动）

### 设计原则

- **自然映射**：手势动作与期望结果自然关联
- **可发现性**：用户能自然发现可用手势
- **容错性**：识别算法应有一定容错范围
- **反馈及时**：手势识别后立即提供视觉/听觉反馈
- **疲劳最小化**：避免需要长时间维持的手势

### 常用手势模式

| 手势 | 动作 | 应用场景 |
|------|------|---------|
| 捏合 | 拇指+食指接触 | 选择、确认 |
| 挥手 | 手掌水平移动 | 翻页、切换 |
| 抓取 | 握拳动作 | 抓取物体 |
| 指向 | 食指伸出 | 指示方向 |
| 缩放 | 双手距离变化 | 调整大小 |

### 手势交互状态机

```
空闲状态 → 检测到手 → 手势识别
                ↓
        ┌───────┴───────┐
        ↓               ↓
    静态手势         动态手势
        ↓               ↓
    持续检测       轨迹追踪
        ↓               ↓
    手势确认       手势完成
        ↓               ↓
    执行动作       执行动作
        ↓               ↓
        └───────┬───────┘
                ↓
            返回空闲
```

## 语音交互

### 语音命令设计

语音交互是 XR 环境的重要补充：

- **唤醒词**：触发语音识别的关键词
- **命令语法**：简洁明确的命令结构
- **上下文感知**：根据当前状态理解语音意图
- **多模态融合**：语音 + 注视/手势的组合

### 设计原则

- **简短命令**：避免冗长的语音指令
- **自然语言**：支持多种表达方式
- **确认机制**：重要操作需要语音确认
- **错误恢复**：提供清晰的错误反馈和重试方式
- **隐私考虑**：提供静音选项

### 语音 UI 反馈

```
用户：「打开菜单」
系统：[视觉] 菜单淡入显示
      [音频] 「菜单已打开」提示音
      [触觉] 控制器轻微震动
```

### 语音交互流程

- **监听状态**：等待唤醒词或直接命令
- **识别处理**：语音转文本 + 语义理解
- **意图解析**：确定用户意图和参数
- **执行反馈**：执行操作并提供多模态反馈

## 触觉反馈

### 触觉技术

触觉反馈（Haptic Feedback）增强交互的真实感：

- **振动反馈**：通过振动电机产生触觉感受
- **力反馈**：模拟物体的阻力和重量
- **热反馈**：模拟温度变化
- **纹理反馈**：模拟表面纹理

### 控制器触觉

- **振动模式**：不同强度、频率、持续时间的振动组合
- **脉冲模式**：短促的脉冲振动
- **连续振动**：持续的振动反馈
- **自适应触发**：如 PS5 DualSense 的自适应扳机

### 设计应用

- **碰撞反馈**：物体碰撞时的冲击感
- **抓取反馈**：抓取物体时的握持感
- **工具反馈**：使用工具时的阻力感
- **环境反馈**：雨滴、风等环境效果

### 触觉设计原则

- **适度使用**：避免过度使用导致疲劳
- **语义一致**：触觉反馈与视觉/听觉一致
- **层次分明**：不同操作使用不同的触觉模式
- **性能优化**：避免触觉计算影响帧率

## 交互模式

### 直接交互

直接用手触摸和操作虚拟物体：

- **触摸交互**：手指直接触摸虚拟按钮
- **抓取交互**：用手抓取和移动虚拟物体
- **推动交互**：用手推动虚拟物体
- **捏合交互**：用手指捏合小物体

### 间接交互

通过控制器或其他设备间接操作：

- **射线交互**：从手部发射射线选择远处物体
- **控制器映射**：物理按键映射到虚拟操作
- **手势快捷键**：特定手势触发快捷操作
- **语音命令**：语音控制虚拟环境

### 混合交互

结合直接和间接交互的优势：

- **近处直接 + 远处间接**：根据距离切换交互方式
- **粗略直接 + 精确间接**：根据精度需求切换
- **多模态组合**：手势 + 语音 + 注视的组合

## 注视交互

### 注视设计

眼动追踪使注视成为重要的交互通道：

- **注视选择**：注视目标后确认选择
- **注视预览**：注视目标显示预览信息
- **注视滚动**：注视边缘区域触发滚动
- **焦点渲染**：注视区域高质量渲染

### 设计原则

- **避免误触**：注视不应直接触发重要操作
- **确认机制**：注视 + 手势/语音确认
- **隐私保护**：不记录敏感的注视数据
- **舒适性**：避免要求精确注视的交互

## 用户体验设计

### 舒适性设计

- **运动病预防**：避免强制的虚拟运动
- **视野管理**：运动时收缩视野
- **稳定参考**：提供稳定的参考框架
- **休息提示**：长时间使用后提醒休息

### 引导设计

- **首次引导**：新用户教程和引导
- **交互提示**：可视化显示可用交互
- **进度反馈**：操作进度的可视化
- **错误引导**：错误发生时的恢复引导

### 无障碍设计

- **可调节性**：支持不同能力用户的调节
- **替代交互**：为不同需求提供替代交互方式
- **视觉辅助**：高对比度、大字体等选项
- **听觉辅助**：字幕和视觉提示

## 设计工具与流程

### 原型工具

- **3D 原型工具**：Shapes、Gravity Sketch
- **交互原型**：Unity、Unreal Engine
- **快速原型**：Figma 3D、Spline
- **用户测试**：VR 原型测试平台

### 设计流程

1. **用户研究**：了解目标用户和使用场景
2. **概念设计**：确定交互模式和关键流程
3. **原型制作**：创建可交互的 3D 原型
4. **用户测试**：在 XR 环境中测试原型
5. **迭代优化**：根据反馈持续改进
6. **开发实现**：与开发团队协作实现
