# 51 单片机

## 一、8051 架构概述

8051 是 Intel 于 1980 年推出的 8 位微控制器，采用哈佛架构（程序存储器和数据存储器分开编址）。

| 参数 | 值 |
|------|-----|
| CPU | 8 位 |
| 架构 | Harvard |
| 程序存储器 (ROM) | 4 KB (内部) |
| 数据存储器 (RAM) | 128 B (内部) |
| 定时器/计数器 | 2 个 16 位 |
| 中断源 | 5 个 |
| I/O 引脚 | 32 个 (4 组 × 8 位) |
| 串行接口 | 1 个 UART |

## 二、引脚配置

| 引脚组 | 功能 | 特性 |
|--------|------|------|
| Port 0 (P0.0-P0.7) | 双向 I/O / 地址/数据总线 | 开漏输出，需外部上拉 |
| Port 1 (P1.0-P1.7) | 双向 I/O | 内部上拉 |
| Port 2 (P2.0-P2.7) | 双向 I/O / 高 8 位地址 | 内部上拉 |
| Port 3 (P3.0-P3.7) | 双向 I/O / 复用功能 | 内部上拉 |

### 特殊引脚

| 引脚 | 功能 |
|------|------|
| RST | 复位（高电平有效） |
| XTAL1/XTAL2 | 晶振输入/输出 |
| EA | 外部程序存储器选择（高电平：内部 ROM，低电平：外部） |
| PSEN | 外部程序存储器读选通 |
| ALE | 地址锁存使能 |

## 三、存储器组织

### 内部 RAM 地址映射

| 地址范围 | 大小 | 用途 |
|---------|------|------|
| 00H-1FH | 32 B | 工作寄存器组 (Bank 0-3) |
| 20H-2FH | 16 B | 位寻址区 (128 位) |
| 30H-7FH | 80 B | 通用 RAM 和堆栈 |
| 80H-FFH | — | 特殊功能寄存器 (SFR) |

### SFR 关键寄存器

| 寄存器 | 地址 | 功能 |
|--------|------|------|
| P0-P3 | 80H-0A0H | I/O 端口锁存器 |
| TMOD | 89H | 定时器模式控制 |
| TCON | 88H | 定时器控制 |
| TH0/TL0 | 8CH/8AH | 定时器 0 计数寄存器 |
| TH1/TL1 | 8DH/8BH | 定时器 1 计数寄存器 |
| SCON | 98H | 串行口控制 |
| SBUF | 99H | 串行数据缓冲 |
| IE | A8H | 中断使能 |
| IP | B8H | 中断优先级 |
| PSW | D0H | 程序状态字 |

## 四、I/O 端口

51 单片机的 I/O 端口具有准双向结构：

- 输出：写锁存器驱动端口引脚
- 输入：必须先写 1 到锁存器，再读引脚

```c
// 准双向口输入步骤
P1 = 0xFF;          // 先写 1
unsigned char val = P1;  // 再读
```

Port 0 作为 I/O 时为开漏输出，需外接上拉电阻（通常 10 kΩ）。

## 五、定时器/计数器

### 模式选择 (TMOD)

| 模式 | 说明 | 位数 | 应用 |
|------|------|------|------|
| Mode 0 | 13 位定时器 | 13 | 兼容旧设计 |
| Mode 1 | 16 位定时器 | 16 | 通用定时 |
| Mode 2 | 8 位自动重装 | 8 | 产生波特率 |
| Mode 3 | 双 8 位定时器 | — | 需要额外定时器 |

### 定时器初始化示例

```c
// 定时器 0，模式 1，50ms 中断（12MHz 晶振）
TMOD |= 0x01;           // Timer0, Mode 1 (16-bit)
TH0 = 0x4C;             // 高 8 位初值
TL0 = 0x00;             // 低 8 位初值
ET0 = 1;                // 允许定时器 0 中断
EA = 1;                 // 允许总中断
TR0 = 1;                // 启动定时器

// 定时 50ms 计算: (65536 - 50000) * 12/12MHz = 15536 = 0x3CB0
// 实际初值 = 65536 - 50000 ≅ 15536
```

定时器溢出时间通用公式（16 位模式，Mode 1）：

$$T_{overflow} = (2^{16} - \text{TH} \times 256 - \text{TL}) \times T_{cyc}$$

其中 $T_{cyc} = 12 / f_{osc}$ 为机器周期，TH 和 TL 分别为定时器高 8 位和低 8 位初值。

常用定时初值（12 MHz 晶振，Mode 1 十六位模式）：

| 定时时间 | TH0 | TL0 | 初值计算 |
|---------|-----|-----|---------|
| 1 ms | 0xFC | 0x18 | $65536 - 1000$ |
| 5 ms | 0xEC | 0x78 | $65536 - 5000$ |
| 10 ms | 0xD8 | 0xF0 | $65536 - 10000$ |
| 50 ms | 0x3C | 0xB0 | $65536 - 50000$ |

## 六、中断系统

| 中断源 | 入口地址 | 触发条件 | 优先级 (默认) |
|--------|---------|---------|-------------|
| INT0 (P3.2) | 0003H | 电平/边沿触发 | 最高 |
| TF0 (Timer0) | 000BH | 定时器 0 溢出 | 次高 |
| INT1 (P3.3) | 0013H | 电平/边沿触发 | 中 |
| TF1 (Timer1) | 001BH | 定时器 1 溢出 | 低 |
| RI/TI (UART) | 0023H | 串口接收/发送完成 | 最低 |

中断延迟时间取决于当前指令执行剩余时间和长跳转指令时间：

$$t_{latency} = t_{current\_inst} + 2 \times t_{machine\_cycle}$$

标准 8051 在 12 MHz 下，中断响应时间约为 3-9 个机器周期（3-9 µs）。

不同指令执行时间对中断延迟的影响：

| 当前指令 | 周期数 | 额外延迟 |
|---------|--------|---------|
| 单周期指令（NOP） | 1 | 3-4 周期 |
| 双周期指令（MOV） | 2 | 4-5 周期 |
| 四周期指令（MUL） | 4 | 6-7 周期 |

```c
// 中断服务函数 (C51)
void timer0_isr(void) interrupt 1 {
    TH0 = 0x4C;    // 重装初值
    TL0 = 0x00;
    // 处理代码
}
```

| 寄存器 | 功能 |
|--------|------|
| IE | 中断使能控制 (EA, ET0, ET1, EX0, EX1, ES) |
| IP | 中断优先级设置 (PT0, PT1, PX0, PX1, PS) |
| TCON | 中断触发方式 (IT0, IT1) 和中止标志 |

## 七、串行通信 (UART)

### SCON 寄存器

| 位 | 功能 |
|----|------|
| SM0/SM1 | 模式选择 (00: 模式0, 01: 模式1, 10: 模式2, 11: 模式3) |
| REN | 允许接收 |
| TI | 发送中断标志 |
| RI | 接收中断标志 |

### 波特率计算 (模式 1, 8 位 UART)

$$Baud = \frac{2^{SMOD}}{32} \times \frac{f_{osc}}{12 \times (256 - TH1)}$$

常用波特率与 TH1 初值（12 MHz，SMOD=0）：

| 目标波特率 | 实际波特率 | 误差 | TH1 |
|-----------|-----------|------|-----|
| 2400 | 2403.8 | 0.16% | 0xE8 |
| 4800 | 4807.7 | 0.16% | 0xF4 |
| 9600 | 9615.4 | 0.16% | 0xFD |
| 19200 | 19230.8 | 0.16% | 0xFE |
| 57600 | 57692.3 | 0.16% | 0xFF |

```c
// 波特率 9600 (12MHz, SMOD=0)
TH1 = 0xFD;          // 256 - (12MHz / 12 / 32 / 9600) ≈ 253
TMOD |= 0x20;        // Timer1, Mode 2 (8 位自动重装)
SCON = 0x50;         // Mode 1, REN=1
TR1 = 1;             // 启动 Timer1

// 发送
void uart_send(unsigned char c) {
    SBUF = c;
    while (!TI);
    TI = 0;
}

// 接收
unsigned char uart_recv(void) {
    while (!RI);
    RI = 0;
    return SBUF;
}
```

## 九、C 语言编程

### Keil C51 / SDCC

```c
// LED 闪烁
#include <reg52.h>
sbit LED = P1^0;      // 位定义

void delay(unsigned int ms) {
    unsigned int i, j;
    for (i = 0; i < ms; i++)
        for (j = 0; j < 125; j++);
}

void main(void) {
    while (1) {
        LED = 0;       // 亮
        delay(500);
        LED = 1;       // 灭
        delay(500);
    }
}
```
