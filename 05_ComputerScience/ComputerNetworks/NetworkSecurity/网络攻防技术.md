# 网络攻防技术

## 一、网络安全威胁模型

网络安全攻防是信息安全的永恒主题。攻击者的目标包括窃取数据、破坏服务、获取未授权访问权限等。理解攻击者的动机和能力是建立有效防御的前提。

网络威胁建模的常用框架包括：STRIDE(欺骗、篡改、抵赖、信息泄露、拒绝服务、权限提升)和 MITRE ATT&CK(按战术和技术组织的攻击知识库)。攻击者的能力层次从脚本小子(使用现成工具)到国家级 APT(高级持续性威胁)不等。

## 二、网络攻击技术

### 2.1 侦察与信息收集

攻击的第一步是信息收集：

- **被动侦察**：OSINT(公开情报)、Whois 查询、DNS 枚举
- **主动侦察**：端口扫描(Nmap)、服务指纹识别、网络拓扑探测
- **社会工程学**：钓鱼邮件、钓鱼网站、电话欺诈

Nmap 扫描目标开放端口和服务版本：

```bash
nmap -sS -sV -p- -T4 target.com
```

### 2.2 漏洞利用

| 漏洞类型 | 描述 | 常见目标 |
|----------|------|----------|
| 缓冲区溢出 | 超出缓冲区边界的写操作 | 本地进程、网络服务 |
| SQL 注入 | 在输入中插入 SQL 语句操控数据库 | Web 应用 |
| XSS(跨站脚本) | 注入恶意脚本在用户浏览器执行 | Web 应用 |
| CSRF | 伪造用户请求执行未授权操作 | Web 应用 |
| 命令注入 | 在参数中注入操作系统命令 | Web 应用、API |
| SSRF | 诱导服务器向内部系统发起请求 | Web 应用 |
| 反序列化漏洞 | 不安全反序列化导致代码执行 | Java/Python 应用 |

### 2.3 拒绝服务攻击(DoS/DDoS)

DDoS 攻击通过大量分布式节点耗尽目标资源：

- **体积攻击**：UDP Flood, ICMP Flood, DNS 放大攻击
- **协议攻击**：SYN Flood, Ping of Death
- **应用层攻击**：HTTP Flood, Slowloris 慢速攻击

DNS 放大攻击利用小查询产生大响应：

$$
\text{放大倍数} = \frac{\text{响应大小}}{\text{查询大小}} \approx 50-100x
$$

### 2.4 中间人攻击(MITM)

MITM 攻击者位于通信双方之间拦截或篡改通信。ARP 欺骗、DNS 欺骗、SSL 剥离是常见 MITM 技术。防御方法包括端到端加密(TLS)、HSTS 和证书固定。

### 2.5 高级持续性威胁(APT)

APT 攻击由国家支持的组织执行，具有高隐蔽性、长周期和明确目标的特点。典型攻击链包括：初始入侵-建立据点-内网横向移动-权限提升-数据窃取-痕迹清理。

## 三、网络防御技术

### 3.1 防火墙

防火墙按预设规则控制网络流量：

| 防火墙类型 | 工作原理 | 处理层次 |
|-----------|----------|----------|
| 包过滤 | 检查 IP 头部和传输层端口 | 网络层+传输层 |
| 状态检测 | 跟踪连接状态，动态过滤 | 网络层-应用层 |
| 应用代理 | 代理应用层连接，深入检查内容 | 应用层 |
| 下一代防火墙(NGFW) | 深度包检测+入侵防御+应用识别 | 全层次 |

### 3.2 入侵检测与防御系统

- **NIDS(网络入侵检测)**：监控网络流量，匹配签名或检测异常行为
- **HIDS(主机入侵检测)**：监控系统日志、文件完整性和进程行为
- **IPS(入侵防御)**：在线部署，可主动阻断攻击流量

Snort 规则示例：
```
alert tcp $EXTERNAL_NET any -> $HOME_NET 80 (
  msg:"SQL Injection Attempt";
  content:"' OR 1=1--";
  sid:1000001;
)
```

### 3.3 加密与认证

- **TLS/SSL**：保护传输层安全，证书体系验证身份
- **IPSec**：保护 IP 层通信，用于 VPN
- **SSH**：安全远程登录和文件传输
- **WPA3**：无线网络安全协议

公钥基础设施(PKI)管理数字证书的生命周期：生成、签发、分发、吊销。

## 四、Web 安全

### 4.1 OWASP Top 10

OWASP Top 10是 Web 安全的重要参考，最新版(2021)主要风险：

1. 失效的访问控制
2. 密码学失效
3. 注入漏洞
4. 不安全的设计
5. 安全配置错误
6. 易受攻击和过时的组件
7. 身份验证和会话管理失效
8. 软件和数据完整性失效
9. 安全日志记录和监控失效
10. 服务端请求伪造(SSRF)

### 4.2 安全编码实践

```python
# 不安全：SQL 注入风险
cursor.execute(f"SELECT * FROM users WHERE username='{username}'")

# 安全：参数化查询
cursor.execute(
    "SELECT * FROM users WHERE username=?",
    (username,)
)
```

```python
# XSS 防御：输出编码
import html
safe_text = html.escape(user_input)
```

## 五、渗透测试方法论

渗透测试模拟攻击者行为评估系统安全性。标准流程：

1. **规划与侦察**：确定范围，收集信息
2. **扫描**：识别开放端口、服务版本、漏洞
3. **漏洞利用**：利用发现漏洞获取访问权限
4. **后渗透**：权限维持、横向移动、数据收集
5. **报告**：详细说明发现、风险和建议

**红队-蓝队演习**是更高级的对抗形式：红队模拟真实攻击者，蓝队负责防御，紫队协调双方。

## 六、零信任安全架构

零信任的核心原则："从不信任，始终验证"

- 微隔离：将网络划分为最小权限单元
- 持续验证：每次访问请求都进行身份验证和授权
- 最小权限：按需分配最低限度访问权限
- 设备健康检查：确保终端符合安全策略
