# 无线网络 (Wireless Networks)

## 1. 无线基础 (Wireless Fundamentals)

### 电磁频谱

无线通信使用电磁波传输数据，不同频段具有不同特性。

```
频率范围与命名：
  ┌────────────┬────────────┬──────────────┐
  │ 频段       │ 频率范围    │ 典型应用      │
  ├────────────┼────────────┼──────────────┤
  │ VLF        │ 3-30kHz    │ 潜艇通信     │
  │ LF         │ 30-300kHz  │ AM 广播       │
  │ MF         │ 300-3000kHz│ AM 广播       │
  │ HF         │ 3-30MHz    │ 短波通信      │
  │ VHF        │ 30-300MHz  │ TV、FM 广播   │
  │ UHF        │ 300-3000MHz│ 手机、Wi-Fi   │
  │ SHF        │ 3-30GHz    │ 卫星、5G、雷达│
  │ EHF        │ 30-300GHz  │ 6G 研究       │
  └────────────┴────────────┴──────────────┘

ISM 免许可频段：
  900MHz  (915MHz)   ───  Zigbee, LoRa
  2.4GHz  (2.4-2.5GHz) ─── Wi-Fi 4, 蓝牙, Zigbee
  5GHz    (5.15-5.85GHz) ── Wi-Fi 5/6
  6GHz    (5.925-7.125GHz) ── Wi-Fi 6E/7
```

### 调制技术

| 调制方式 | 原理 | 应用 |
|---------|------|------|
| ASK (幅移键控) | 幅度变化表示 0/1 | 光纤通信 |
| FSK (频移键控) | 频率变化表示 0/1 | 蓝牙 |
| PSK (相移键控) | 相位变化表示 0/1 | 卫星通信 |
| BPSK | 2 种相位 (1bit/符号) | 802.11 基础速率 |
| QPSK | 4 种相位 (2bit/符号) | 802.11a/g |
| QAM (正交振幅调制) | 幅度+相位组合 | Wi-Fi 4/5/6/7 |

```
QAM 对比：
  16-QAM:  4×4 = 16 种状态, 4bit/符号
  64-QAM:  8×8 = 64 种状态, 6bit/符号
  256-QAM: 16×16 = 256 种状态, 8bit/符号
  1024-QAM: 32×32 = 1024 种状态, 10bit/符号
  4096-QAM: 64×64 = 4096 种状态, 12bit/符号 (Wi-Fi 7)
```

### OFDM（正交频分复用）

OFDM 将信道划分为多个正交子载波并行传输，抗多径衰落能力强。

```
传统单载波: [====全带宽====]  → 易受频率选择性衰落

OFDM 多子载波:
  [载波1][载波2][载波3]...[载波N]
    │      │      │         │
  每个子载波窄带 → 平坦衰落 → 易均衡

OFDMA (OFDM Access):
  不同子载波分配给不同用户，Wi-Fi 6/5G 采用
```

## 2. IEEE 802.11 标准

### 标准演进对比

| 标准 | 频段 | 最大速率 | 调制 | 信道宽度 | 发布年份 | Wi-Fi 代 |
|------|------|---------|------|---------|---------|---------|
| 802.11 | 2.4GHz | 2Mbps | FHSS/DSSS | 22MHz | 1997 | — |
| 802.11b | 2.4GHz | 11Mbps | HR-DSSS | 22MHz | 1999 | Wi-Fi 1 |
| 802.11a | 5GHz | 54Mbps | OFDM | 20MHz | 1999 | Wi-Fi 2 |
| 802.11g | 2.4GHz | 54Mbps | OFDM | 20MHz | 2003 | Wi-Fi 3 |
| 802.11n | 2.4/5GHz | 600Mbps | OFDM + MIMO | 20/40MHz | 2009 | Wi-Fi 4 |
| 802.11ac | 5GHz | 6.9Gbps | OFDM + MU-MIMO | 20/40/80/160MHz | 2013 | Wi-Fi 5 |
| 802.11ax | 2.4/5/6GHz | 9.6Gbps | OFDMA + MU-MIMO | 20/40/80/160MHz | 2019 | Wi-Fi 6/6E |
| 802.11be | 2.4/5/6GHz | 46Gbps | OFDMA + MU-MIMO(16×16) | 320MHz | 2024 | Wi-Fi 7 |

### Wi-Fi 6/6E 核心特性

```
OFDMA (正交频分多址):
  ┌────────────────────────────────────────┐
  │ 传统 Wi-Fi: 每个用户独占整个信道         │
  │  ┌──────┐                              │
  │  │ 用户A │  → 其他用户等待               │
  │  └──────┘                              │
  │                                        │
  │ Wi-Fi 6 OFDMA: 子载波分给多个用户       │
  │  ┌──┬──┬──┬──┬──┬──┬──┬──┐            │
  │  │A │A │B │B │C │C │  │  │  RU分配     │
  │  └──┴──┴──┴──┴──┴──┴──┴──┘            │
  └────────────────────────────────────────┘

MU-MIMO (多用户多入多出):
  Wi-Fi 5: 下行 MU-MIMO (4×4)
  Wi-Fi 6: 下行+上行 MU-MIMO (8×8)
  Wi-Fi 7: 16×16 MU-MIMO

其他特性：
  - 1024-QAM (Wi-Fi 6) / 4096-QAM (Wi-Fi 7)
  - 目标唤醒时间 (TWT) — 省电
  - BSS Coloring — 减少同频干扰
  - 6GHz 频段 (Wi-Fi 6E/7) — 更多信道、更少干扰
```

## 3. 蜂窝网络 (Cellular Networks)

### 4G LTE

```
LTE 网络架构 (EPC):
  UE ─── eNB ─── MME ─── HSS
    │          └── S-GW ─── P-GW ─── Internet
    │                   │
    └───────────────────┘

关键指标：
  峰值速率: 下行 1Gbps, 上行 500Mbps
  时延: 10-30ms
  载波聚合: 最多 5 个载波
  多天线: 4×4 MIMO, 8×8 MIMO
```

### 5G NR

```
5G 架构 (5GC):
  UE ─── gNB ─── AMF ─── UDM
    │          └── UPF ─── DN (Data Network)
    │                   │
    └───────────────────┘

三大场景：
  eMBB (增强移动宽带)  ─── 高速下载
  URLLC (超可靠低时延) ─── 工业控制
  mMTC (大规模物联网)   ─── 海量连接

5G 关键技术：
  ┌────────────────────────────────────┐
  │ 毫米波 (mmWave, 24-52GHz)         │
  │ 大规模 MIMO (mMIMO, 64/128天线)    │
  │ 波束成形 (Beamforming)             │
  │ 网络切片 (Network Slicing)         │
  │ 边缘计算 (MEC)                     │
  │ OFDMA + 可缩放子载波间隔           │
  └────────────────────────────────────┘
```

### 4G vs 5G

| 指标 | 4G LTE | 5G NR |
|------|--------|-------|
| 峰值速率 | 1 Gbps | 20 Gbps |
| 用户面时延 | 10-30ms | 1-4ms |
| 连接密度 | 10^5/km² | 10^6/km² |
| 频段 | 0.7-2.6GHz | 0.7-52GHz |
| 信道带宽 | 20MHz | 100MHz (sub-6GHz), 400MHz (mmWave) |
| 天线数 | 2-8 | 64-256 |

### 蓝牙 (Bluetooth)

| 版本 | 速率 | 范围 | 特点 |
|------|------|------|------|
| 4.0 (BLE) | 1Mbps | 100m | 低功耗 |
| 5.0 | 2Mbps | 240m | 2倍速率/4倍距离 |
| 5.1 | 2Mbps | 240m | 方向定位 |
| 5.2 | 2Mbps | 240m | LE Audio |

### Zigbee

```
特性：
  标准: IEEE 802.15.4
  频段: 2.4GHz (全球), 915MHz (美国), 868MHz (欧洲)
  速率: 250kbps (2.4GHz)
  范围: 10-100m
  拓扑: 星型、树型、网状
  功耗: 极低 (纽扣电池可用数年)
  应用: 智能家居、工业自动化

对比 Wi-Fi:
  Zigbee: 低速率、低功耗、Mesh 组网
  Wi-Fi:  高速率、高功耗、星型拓扑
```

### LoRaWAN

```
特性：
  频段: 868MHz (欧洲), 915MHz (美国)
  速率: 0.3-50kbps
  范围: 2-15km (视距)
  拓扑: 星型 (终端→网关→网络服务器)
  功耗: 极低 (电池可用10年)
  应用: 智慧城市、农业、资产管理
```

| 技术 | 频率 | 范围 | 速率 | 应用 |
|------|------|------|------|------|
| NFC | 13.56MHz | <10cm | 106-424kbps | 移动支付、门禁 |
| RFID (被动) | 125kHz-915MHz | <10m | 变化 | 物流、门禁 |

## 5. 无线传播与信道

### 香农信道容量

$$ C = B \log_2(1 + \frac{S}{N}) $$

其中 $C$ 为信道容量 (bps)，$B$ 为带宽 (Hz)，$S/N$ 为信噪比。

### 路径损耗与衰落

**Friis 自由空间路径损耗**：

$$ P_r = P_t G_t G_r \left(\frac{\lambda}{4\pi d}\right)^2 $$

$$ \text{PL(dB)} = 20\log_{10}(d) + 20\log_{10}(f) + 32.44 $$

其中 $d$ 为距离 (km)，$f$ 为频率 (MHz)，$P_t$ 为发射功率，$P_r$ 为接收功率。

例：距离 100m, 频率 2.4GHz
  PL = 20·log10(0.1) + 20·log10(2400) + 32.44
     = -20 + 67.6 + 32.44 = 80.04 dB

衰落类型：
   大尺度衰落 (Large-scale): 路径损耗、阴影效应
   小尺度衰落 (Small-scale): 多径效应 → 瑞利/莱斯分布
```

### 多径效应

```
发送:              接收（多条路径）:
  [信号]───────┐       ┌────[信号1] (直射)
               ├───────┤────[信号2] (反射)
               │       └────[信号3] (绕射)
               └───障碍物───[信号4] (散射)

多径引起的效应：
  - 时间色散 (时延扩展) → 符号间干扰 (ISI)
  - 频率选择性衰落
  - 角度扩展

应对技术：
  OFDM (将宽带转为多个窄带子载波)
  MIMO (利用多径提升容量)
  均衡器 (Equalizer)
```

### CSMA/CA 与 CSMA/CD

| 特性 | CSMA/CD (有线) | CSMA/CA (无线) |
|------|----------------|----------------|
| 全称 | 载波监听多点接入/碰撞检测 | 载波监听多点接入/碰撞避免 |
| 检测方式 | 检测电压变化（碰撞检测） | 等待 ACK（碰撞避免） |
| 处理碰撞 | 检测到碰撞后停止并随机等待 | 避免碰撞（RTS/CTS） |
| 隐藏节点问题 | 不存在 | 存在（RTS/CTS 解决） |
| 应用 | 以太网 | Wi-Fi |

```
CSMA/CA 流程：
  1. 监听信道
  2. 空闲 → 等待 DIFS → 发送数据
  3. 忙 → 等待直到空闲 → 竞争窗口 (随机退避)
  4. 接收方收到后等待 SIFS → 回复 ACK
  5. 未收到 ACK → 重传

**退避时间**：$\text{Backoff} = \text{Random}() \times \text{SlotTime}$，其中 $\text{Random}() \in [0, \text{CW}]$，竞争窗口 $\text{CW}$ 随重传次数指数增长。

RTS/CTS 机制：
  发送方 ──RTS──► 接收方
  发送方 ◄──CTS── 接收方
  其他节点收到 CTS 后设置 NAV（网络分配向量）
  发送方 ──DATA─► 接收方
  发送方 ◄──ACK── 接收方
```

## 6. 无线安全

### WPA3 特性

```
WPA3-Personal (SAE):
  - 使用 SAE (Simultaneous Authentication of Equals) 替代 PSK
  - 抵抗离线字典攻击
  - 前向安全性（即使密码泄露，过去的会话仍安全）
  - 192-bit 安全套件 (WPA3-Enterprise)

WPA3 对比 WPA2:
  ┌────────────────┬────────────────┐
  │ WPA2           │ WPA3           │
  ├────────────────┼────────────────┤
  │ 4-way handshake│ SAE handshake  │
  │ PSK（易破解）   │ 同时认证       │
  │ CCMP-AES       │ GCMP-256       │
  │ 公开帧保护      │ 管理帧保护      │
  └────────────────┴────────────────┘
```

### 无线攻击与防御

| 攻击 | 原理 | 防御 |
|------|------|------|
| 暴力破解 Wi-Fi 密码 | 尝试所有密码组合 | WPA3、强密码、802.1X |
| Deauth 攻击 | 伪造解认证帧 | 802.11w (管理帧保护) |
| Evil Twin | 伪造同名热点 | 证书验证、VPN |
| KRACK | 破解 WPA2 4次握手 | 升级 WPA3 |
| 中间人攻击 | 拦截无线通信 | HTTPS、证书锁定 |

## 7. 移动性管理

### 切换 (Handover)

```
硬切换 (Hard Handover):
  先断开旧连接 → 再建立新连接
  "Break before Make"
  时延大，但实现简单
  传统蜂窝网络使用

软切换 (Soft Handover):
  保持旧连接的同时建立新连接
  "Make before Break"
  无缝切换，但耗费资源
  CDMA 网络使用

Wi-Fi 漫游:
  客户端根据信号强度决定切换
  802.11r (快速漫游)：简化认证过程
  802.11k (邻居报告)：协助客户端发现附近 AP
  802.11v (网络辅助)：网络主动引导客户端切换
```
