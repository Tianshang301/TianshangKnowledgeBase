# 游戏AI

游戏 AI（Game Artificial Intelligence）是为游戏中的非玩家角色（NPC）和系统赋予智能行为的技术，使其能够做出合理的决策、导航和适应玩家行为。

## 行为树

### 基本概念

行为树（Behavior Tree）是游戏 AI 最常用的行为组织结构：

- **树形结构**：层次化的节点组织
- **自上而下执行**：从根节点开始评估
- **状态返回**：每个节点返回 Success、Failure 或 Running
- **模块化**：行为可以组合和复用

### 节点类型

```
行为树节点类型：
├── 组合节点 (Composite)
│   ├── Sequence (顺序) - 所有子节点都成功才成功
│   ├── Selector (选择) - 任一子节点成功就成功
│   └── Parallel (并行) - 同时执行多个子节点
├── 装饰节点 (Decorator)
│   ├── Inverter (反转) - 反转子节点结果
│   ├── Repeater (重复) - 重复执行子节点
│   ├── Succeeder (总是成功) - 总是返回成功
│   └── Until Fail (直到失败) - 重复直到失败
└── 叶子节点 (Leaf)
    ├── Action (动作) - 执行具体行为
    └── Condition (条件) - 检查条件
```

### 行为树示例

```
敌人 AI 行为树：
├── Selector (选择)
│   ├── Sequence (战斗)
│   │   ├── Condition: 看到玩家?
│   │   ├── Action: 追击玩家
│   │   └── Action: 攻击玩家
│   ├── Sequence (搜索)
│   │   ├── Condition: 听到声音?
│   │   └── Action: 搜索声源位置
│   └── Action (巡逻)
│       └── Action: 沿路径点巡逻
```

### 黑板系统

黑板（Blackboard）是行为树的共享数据存储：

- **全局数据**：所有节点可访问的数据
- **感知信息**：敌人位置、威胁等级等
- **目标数据**：当前目标、移动目的地
- **状态信息**：生命值、弹药量等

## GOAP

### 目标导向行动规划

GOAP（Goal-Oriented Action Planning）是更灵活的 AI 规划系统：

- **目标驱动**：AI 根据目标选择行动
- **自动规划**：自动寻找达到目标的行动序列
- **动态适应**：根据情况重新规划

### 核心组件

```
GOAP 组件：
├── World State (世界状态)
│   └── 布尔/数值属性集合
├── Goals (目标)
│   └── 期望的世界状态
├── Actions (行动)
│   ├── 前置条件 (Preconditions)
│   ├── 效果 (Effects)
│   └── 代价 (Cost)
└── Planner (规划器)
    └── A* 搜索算法
```

### 规划过程

1. **定义当前世界状态**
2. **选择优先级最高的目标**
3. **搜索从当前状态到目标状态的行动序列**
4. **按顺序执行行动**
5. **如果失败，重新规划**

### GOAP 与行为树对比

| 特性 | 行为树 | GOAP |
|------|--------|------|
| 设计方式 | 手动设计结构 | 定义行动和目标 |
| 灵活性 | 固定结构 | 动态规划 |
| 性能 | 预测性强 | 规划有开销 |
| 适用场景 | 固定行为模式 | 复杂决策场景 |
| 调试 | 可视化友好 | 需要规划可视化 |

## 导航网格

### NavMesh 概述

导航网格（Navigation Mesh）是游戏 AI 寻路的核心技术：

- **可行区域**：定义 AI 可以行走的区域
- **凸多边形**：导航区域由凸多边形组成
- **连接关系**：多边形之间的连接定义可通行路径
- **动态更新**：支持障碍物的动态变化

### 寻路算法

- **A* 算法**：启发式搜索最短路径
- **JPS（Jump Point Search）**：优化的 A* 变体
- **分层寻路**：大地图使用分层搜索
- **路径平滑**：使路径更加自然

```csharp
// Unity NavMesh 寻路示例
NavMeshAgent agent;

void MoveToTarget(Vector3 target)
{
    if (NavMesh.CalculatePath(transform.position, target, 
        NavMesh.AllAreas, path))
    {
        agent.SetPath(path);
    }
}
```

### 动态避障

- **RVO（Reciprocal Velocity Obstacles）**：互惠速度障碍
- **ORCA（Optimal Reciprocal Collision Avoidance）**：最优互惠避碰
- **局部避障**：实时调整路径避开动态障碍
- **人群模拟**：模拟大规模人群流动

### 导航区域

- **Walkable**：可行走区域
- **Jump**：可跳跃区域
- **Crouch**：需要蹲下的区域
- **Cost**：不同区域的通行代价

## 群体模拟

### 群体行为

群体模拟（Crowd/Flock Simulation）模拟大量个体的集体行为：

- **分离（Separation）**：避免与邻近个体碰撞
- **对齐（Alignment）**：与邻近个体保持相同方向
- **聚合（Cohesion）**：向邻近个体的中心移动
- **避障（Obstacle Avoidance）**：避开环境障碍物

### Boids 算法

```
Boids 三规则：
1. 分离：远离太近的邻居
2. 对齐：朝向邻居的平均方向
3. 聚合：朝向邻居的中心移动

最终速度 = w1 * 分离 + w2 * 对齐 + w3 * 聚合
```

### 群体寻路

- **流场寻路**：为整个群体计算流场
- **潜在场**：结合目标吸引和障碍排斥
- **分组行为**：保持队形的群体移动
- **密度管理**：控制群体密度避免拥堵

### 优化技术

- **空间分区**：使用网格加速邻居搜索
- **LOD 行为**：远处个体使用简化行为
- **实例化渲染**：GPU 实例化渲染大量个体
- **多线程更新**：并行更新个体行为

## PCG 程序化生成

### 概念介绍

PCG（Procedural Content Generation）是用算法生成游戏内容：

- **地图生成**：程序化生成游戏地图
- **关卡设计**：自动设计游戏关卡
- **物品生成**：随机生成装备、物品
- **任务生成**：程序化生成任务和剧情

### 生成算法

- **随机游走**：生成迷宫和洞穴
- **细胞自动机**：生成自然地形
- **Perlin 噪声**：生成连续的随机地形
- **波函数坍缩（WFC）**：基于约束的生成
- **L-System**：生成植物和分形结构

### 地牢生成

```
地牢生成步骤：
1. 房间生成 - 随机生成房间位置和大小
2. 房间连接 - 使用 MST 或随机连接
3. 走廊生成 - 连接相邻房间
4. 房间类型 - 分配房间功能
5. 敌人放置 - 根据难度放置敌人
6. 物品放置 - 放置宝箱和道具
```

### 程序化纹理

- **噪声函数**：Perlin、Simplex、Worley 噪声
- **分形叠加**：多层噪声叠加（Fractal Brownian Motion）
- **纹理合成**：基于样本的纹理生成
- **材质混合**：程序化混合多种材质

## 感知系统

### 视觉感知

- **视锥体检测**：在视野范围内检测目标
- **遮挡测试**：检查目标是否被遮挡
- **边缘检测**：检测视野边缘的目标
- **动态视觉**：移动目标更容易被发现

### 听觉感知

- **声音传播**：模拟声音在空间中的传播
- **声音遮挡**：墙壁对声音的阻挡
- **声音强度**：距离影响声音强度
- **声音事件**：脚步声、枪声等声音事件

### 记忆系统

- **短期记忆**：最近感知到的信息
- **长期记忆**：重要的持久信息
- **记忆衰减**：随时间遗忘信息
- **记忆更新**：新信息覆盖旧信息

## 决策系统

### 有限状态机（FSM）

- **状态**：AI 的不同行为状态
- **转换**：状态之间的切换条件
- **层次 FSM**：嵌套的状态机结构
- **并行 FSM**：同时运行多个状态机

### 效用系统（Utility System）

- **评分函数**：为每个选项计算分数
- **权重调整**：根据情况调整选项权重
- **随机选择**：根据分数概率选择
- **上下文感知**：考虑当前情况的决策

### 决策流程

```
感知系统 → 决策系统 → 行为系统
    ↓           ↓           ↓
收集信息    评估选项    执行行为
更新记忆    选择行动    生成动画
```
