# 进程调度与同步

## 一、进程状态与模型

进程是程序的执行实例，包含程序代码、数据、堆栈和上下文信息。

进程的五状态模型：

```
新建 (New) -> 就绪 (Ready) <-> 运行 (Running) -> 终止 (Terminated)
                |                  |
                v                  v
            阻塞 (Blocked) <- 等待 IO/事件
                |
                v
           就绪 (条件满足)
```

进程控制块（PCB, Process Control Block）：

| 字段 | 说明 |
|------|------|
| PID | 进程唯一标识 |
| 状态 | 当前进程状态 |
| 程序计数器 | 下一条指令地址 |
| CPU 寄存器 | 上下文信息 |
| 内存限制 | 地址空间边界 |
| 已打开文件列表 | 文件描述符表 |
| 调度信息 | 优先级、时间片等 |
| 记账信息 | CPU 使用时间等 |

## 二、调度算法

调度算法的目标：最大化 CPU 利用率、最小化响应时间、公平分配。

核心调度算法对比：

| 算法 | 决策方式 | 抢占 | 饥饿风险 | 平均等待时间 | 适用场景 |
|------|----------|------|----------|-------------|----------|
| FCFS | 先来先服务 | 否 | 无 | 较长 | 批处理 |
| SJF | 最短作业优先 | 可选 | 长作业 | 最短 | 批处理 |
| SRTF | 最短剩余时间 | 是 | 长作业 | 最短 | 交互式 |
| 优先级调度 | 优先级 | 可选 | 低优先级 | 依赖优先级 | 实时系统 |
| 轮转 (RR) | 时间片轮转 | 是 | 无 | 适中 | 分时系统 |
| 多级反馈队列 | MLFQ | 是 | 有 | 较短 | 通用 OS |

轮转调度时间片选择：

$$
\text{上下文切换开销} = \frac{\text{切换时间}}{\text{时间片}}
$$

时间片过小导致频繁切换，过大退化为 FCFS。

```python
class RoundRobinScheduler:
    def __init__(self, quantum=4):
        self.ready_queue = []
        self.quantum = quantum

    def add_process(self, pid, burst_time):
        self.ready_queue.append({
            'pid': pid,
            'burst': burst_time,
            'remaining': burst_time
        })

    def run(self):
        time = 0
        while self.ready_queue:
            process = self.ready_queue.pop(0)
            execute_time = min(self.quantum, process['remaining'])

            print(f"Time {time}: Running P{process['pid']} "
                  f"for {execute_time} units")
            time += execute_time
            process['remaining'] -= execute_time

            if process['remaining'] > 0:
                self.ready_queue.append(process)
            else:
                print(f"Time {time}: P{process['pid']} completed")

            # 模拟新进程到达
            if time == 2:
                self.add_process(3, 3)
            if time == 5:
                self.add_process(4, 2)


scheduler = RoundRobinScheduler(quantum=3)
scheduler.add_process(1, 5)
scheduler.add_process(2, 8)
scheduler.run()
```

## 三、多级反馈队列

MLFQ（Multi-Level Feedback Queue）是现代操作系统最常用的调度算法。

MLFQ 规则：

```
规则1: 优先级越高，时间片越短
规则2: 新进程进入最高优先级队列
规则3a: 如果进程用完时间片，降低一级优先级
规则3b: 如果进程在时间片结束前让出 CPU，保持优先级
规则4: 周期性将所有进程提升到最高优先级（防止饥饿）
```

```c
// MLFQ 调度器简化实现
#define NUM_QUEUES 3
#define QUANTUM_Q0 2   // 最高优先级，最小时间片
#define QUANTUM_Q1 4
#define QUANTUM_Q2 8   // 最低优先级，最大时间片

typedef struct {
    int pid;
    int priority_level;
    int remaining_burst;
    int total_burst;
} Process;

typedef struct {
    Process *queue[100];
    int front, rear;
    int quantum;
} ReadyQueue;

ReadyQueue queues[NUM_QUEUES] = {
    { .quantum = QUANTUM_Q0 },
    { .quantum = QUANTUM_Q1 },
    { .quantum = QUANTUM_Q2 }
};

void schedule_mlfq() {
    for (int level = 0; level < NUM_QUEUES; level++) {
        if (!is_empty(&queues[level])) {
            Process *p = dequeue(&queues[level]);
            int exec_time = min(queues[level].quantum, p->remaining_burst);

            // 执行进程
            p->remaining_burst -= exec_time;

            if (p->remaining_burst > 0) {
                // 用完了时间片 -> 降级
                int new_level = min(level + 1, NUM_QUEUES - 1);
                enqueue(&queues[new_level], p);
            } else {
                // 进程完成
                free(p);
            }
            return;
        }
    }
}
```

## 四、进程同步机制

竞争条件（Race Condition）：多个进程同时访问和操作共享数据，结果取决于执行顺序。

临界区问题解决方案必须满足：

```
1. 互斥 (Mutual Exclusion): 同时只能一个进程在临界区
2. 前进 (Progress): 无进程在临界区时，应允许等待进程进入
3. 有限等待 (Bounded Waiting): 进程等待进入临界区的时间有限
```

Peterson 算法（软件解决方案）：

```c
#include <stdbool.h>
#include <threads.h>

bool flag[2] = {false, false};
int turn;

void process_0() {
    while (true) {
        flag[0] = true;
        turn = 1;
        while (flag[1] && turn == 1)
            ;  // 忙等待

        // 临界区
        // ...

        flag[0] = false;

        // 剩余区
    }
}

void process_1() {
    while (true) {
        flag[1] = true;
        turn = 0;
        while (flag[0] && turn == 0)
            ;

        // 临界区
        // ...

        flag[1] = false;
    }
}
```

## 五、信号量

信号量（Semaphore）由 Dijkstra 提出，是 OS 中最基础的同步工具。

```c
#include <semaphore.h>
#include <pthread.h>
#include <stdio.h>

sem_t mutex;
int shared_counter = 0;

void *worker(void *arg) {
    int id = *(int *)arg;

    for (int i = 0; i < 100000; i++) {
        sem_wait(&mutex);    // P 操作：请求资源
        shared_counter++;    // 临界区
        sem_post(&mutex);    // V 操作：释放资源
    }

    printf("Thread %d done\n", id);
    return NULL;
}

int main() {
    pthread_t threads[4];
    int ids[4] = {0, 1, 2, 3};

    sem_init(&mutex, 0, 1);  // 初始值=1，二进制信号量

    for (int i = 0; i < 4; i++)
        pthread_create(&threads[i], NULL, worker, &ids[i]);

    for (int i = 0; i < 4; i++)
        pthread_join(threads[i], NULL);

    printf("Final counter: %d (expected: 400000)\n", shared_counter);
    sem_destroy(&mutex);
    return 0;
}
```

经典同步问题：哲学家就餐

```c
#define N 5
sem_t chopsticks[N];

void philosopher(int i) {
    while (true) {
        think(i);
        sem_wait(&chopsticks[i]);           // 拿左筷子
        sem_wait(&chopsticks[(i + 1) % N]); // 拿右筷子
        eat(i);
        sem_post(&chopsticks[i]);           // 放左筷子
        sem_post(&chopsticks[(i + 1) % N]); // 放右筷子
    }
}
```

解决方案：限制最多4个哲学家同时就餐，或要求先拿编号小的筷子。

## 六、管程

管程（Monitor）提供高级同步抽象，包含条件变量。

```c
#include <pthread.h>

typedef struct {
    pthread_mutex_t lock;
    pthread_cond_t not_full;
    pthread_cond_t not_empty;
    int buffer[10];
    int count, in, out;
} MonitorQueue;

void monitor_put(MonitorQueue *mq, int item) {
    pthread_mutex_lock(&mq->lock);
    while (mq->count == 10)
        pthread_cond_wait(&mq->not_full, &mq->lock);

    mq->buffer[mq->in] = item;
    mq->in = (mq->in + 1) % 10;
    mq->count++;

    pthread_cond_signal(&mq->not_empty);
    pthread_mutex_unlock(&mq->lock);
}

int monitor_get(MonitorQueue *mq) {
    pthread_mutex_lock(&mq->lock);
    while (mq->count == 0)
        pthread_cond_wait(&mq->not_empty, &mq->lock);

    int item = mq->buffer[mq->out];
    mq->out = (mq->out + 1) % 10;
    mq->count--;

    pthread_cond_signal(&mq->not_full);
    pthread_mutex_unlock(&mq->lock);
    return item;
}
```

```bash
# 进程查看与管理
ps aux                    # 所有进程详细信息
```

## 八、调度与同步性能

同步原语性能对比：

| 原语 | 用户态/内核态 | 平均延迟 | 适用场景 |
|------|-------------|----------|----------|
| 自旋锁 | 用户态 | 10-50ns | 极短临界区 |
| 互斥锁 | 混合 | 100-200ns | 通用 |
| 读写锁 | 混合 | 100-300ns | 读多写少 |
| 信号量 | 内核态 | 1-5us | 进程间同步 |
| 原子操作 | 用户态 | 5-20ns | 简单计数 |
| RCU | 用户态 | 读无锁 | 读极多场景 |

上下文切换开销：

```
系统调用: ~100ns
线程上下文切换: ~1us
进程上下文切换: ~3-10us
TLB 刷新: ~50-100us (影响后续访问)
```
