# 死锁与并发控制

## 一、死锁的定义与必要条件

死锁是指两个或多个进程因互相等待对方持有的资源而永远无法继续执行的状态。

Coffman于1971年总结的死锁四个必要条件：

| 条件 | 描述 | 比喻 |
|------|------|------|
| 互斥 | 资源不能被共享 | 房间只能进一个人 |
| 持有并等待 | 进程持有资源时等待其他资源 | 拿着钥匙等另一把钥匙 |
| 不可剥夺 | 资源不能被强制夺取 | 别人不能抢你的钥匙 |
| 循环等待 | 存在进程-资源的环形链 | A等B, B等A |

## 二、死锁预防

预防策略通过打破四个必要条件之一来避免死锁。

打破互斥：使用共享访问模式（但有些资源天生不可共享）。

打破持有并等待：要求进程一次性申请所有资源。

```c
// 一次性申请所有资源
int acquire_all_resources(int res1, int res2, int res3) {
    pthread_mutex_lock(&global_mutex);
    if (resources_available(res1, res2, res3)) {
        allocate(res1);
        allocate(res2);
        allocate(res3);
        pthread_mutex_unlock(&global_mutex);
        return 0;  // success
    }
    pthread_mutex_unlock(&global_mutex);
    return -1;  // 资源不足，阻塞或重试
}
```

打破不可剥夺：如果进程申请的资源被拒绝，则释放已持有的资源。

打破循环等待：对资源进行全局编号，强制按顺序申请。

```c
// 资源排序法 - 按地址顺序加锁
void safe_lock_two(pthread_mutex_t *a, pthread_mutex_t *b) {
    if (a < b) {
        pthread_mutex_lock(a);
        pthread_mutex_lock(b);
    } else {
        pthread_mutex_lock(b);
        pthread_mutex_lock(a);
    }
}
```

## 三、死锁避免

银行家算法（Banker's Algorithm）是最著名的死锁避免算法：

设 $n$ 个进程，$m$ 种资源，定义如下数据结构：

| 矩阵 | 含义 | 维度 |
|------|------|------|
| Available | 各资源可用数量 | m |
| Max | 每个进程最大需求 | n x m |
| Allocation | 每个进程已分配资源 | n x m |
| Need | 每个进程还需要资源 | n x m = Max - Allocation |

安全性检查算法：

```python
def is_safe_state(available, max_demand, allocation):
    n = len(allocation)
    m = len(available)

    need = [[max_demand[i][j] - allocation[i][j]
             for j in range(m)] for i in range(n)]
    work = available[:]
    finish = [False] * n
    safe_sequence = []

    while len(safe_sequence) < n:
        found = False
        for i in range(n):
            if not finish[i] and all(need[i][j] <= work[j] for j in range(m)):
                # 可以分配
                for j in range(m):
                    work[j] += allocation[i][j]
                finish[i] = True
                safe_sequence.append(i)
                found = True
                break
        if not found:
            return False, []

    return True, safe_sequence


# 示例
available = [3, 3, 2]
max_demand = [
    [7, 5, 3],
    [3, 2, 2],
    [9, 0, 2],
    [2, 2, 2],
    [4, 3, 3]
]
allocation = [
    [0, 1, 0],
    [2, 0, 0],
    [3, 0, 2],
    [2, 1, 1],
    [0, 0, 2]
]

safe, seq = is_safe_state(available, max_demand, allocation)
print(f"Safe: {safe}, Sequence: {seq}")
```

## 四、死锁检测与恢复

死锁检测算法（基于资源分配图）：

```python
def detect_deadlock(allocation, request, available):
    n = len(allocation)
    m = len(available)

    work = available[:]
    finish = [all(allocation[i][j] == 0 for j in range(m))
              for i in range(n)]

    while True:
        found = False
        for i in range(n):
            if not finish[i] and all(request[i][j] <= work[j] for j in range(m)):
                for j in range(m):
                    work[j] += allocation[i][j]
                finish[i] = True
                found = True

        if not found:
            break

    deadlocked = [i for i in range(n) if not finish[i]]
    return deadlocked
```

恢复方法：

| 方法 | 描述 | 缺点 |
|------|------|------|
| 终止所有死锁进程 | 简单粗暴 | 损失大 |
| 逐个终止进程 | 逐步尝试 | 开销大 |
| 资源抢占 | 剥夺部分进程的资源 | 可能需要回滚 |
| 回滚到安全状态 | 恢复到检查点 | 需要事务支持 |

## 五、并发控制机制

并发控制的核心机制对比：

| 机制 | 粒度 | 等待方式 | 适用场景 | 死锁风险 |
|------|------|----------|----------|----------|
| 互斥锁 (Mutex) | 线程级 | 阻塞 | 短临界区 | 高 |
| 自旋锁 (Spinlock) | 线程级 | 忙等待 | 极短临界区 | 高 |
| 读写锁 (RWLock) | 线程级 | 阻塞 | 读多写少 | 中 |
| 信号量 (Semaphore) | 资源计数 | 阻塞 | 资源池管理 | 中 |
| 条件变量 (CondVar) | 条件等待 | 阻塞 | 生产者-消费者 | 低 |
| RCU | 数据级 | 无等待 | 读极多的场景 | 无 |

## 六、并发编程模型

生产者-消费者模型：

```c
#include <pthread.h>
#include <semaphore.h>
#include <stdio.h>

#define BUFFER_SIZE 10

int buffer[BUFFER_SIZE];
int in = 0, out = 0;

sem_t empty, full;
pthread_mutex_t mutex;

void *producer(void *arg) {
    for (int i = 0; i < 100; i++) {
        sem_wait(&empty);
        pthread_mutex_lock(&mutex);

        buffer[in] = i;
        printf("Produced: %d at slot %d\n", i, in);
        in = (in + 1) % BUFFER_SIZE;

        pthread_mutex_unlock(&mutex);
        sem_post(&full);
    }
    return NULL;
}

void *consumer(void *arg) {
    for (int i = 0; i < 100; i++) {
        sem_wait(&full);
        pthread_mutex_lock(&mutex);

        int item = buffer[out];
        printf("Consumed: %d from slot %d\n", item, out);
        out = (out + 1) % BUFFER_SIZE;

        pthread_mutex_unlock(&mutex);
        sem_post(&empty);
    }
    return NULL;
}

int main() {
    pthread_t prod, cons;

    sem_init(&empty, 0, BUFFER_SIZE);
    sem_init(&full, 0, 0);
    pthread_mutex_init(&mutex, NULL);

    pthread_create(&prod, NULL, producer, NULL);
    pthread_create(&cons, NULL, consumer, NULL);

    pthread_join(prod, NULL);
    pthread_join(cons, NULL);

    sem_destroy(&empty);
    sem_destroy(&full);
    pthread_mutex_destroy(&mutex);
    return 0;
}
```

## 七、高级并发原语

读写锁实现读优先：

```c
typedef struct {
    pthread_mutex_t lock;
    pthread_cond_t readers_proceed;
    pthread_cond_t writers_proceed;
    int readers_count;
    int writer_active;
    int writers_waiting;
} RWLock;

void read_lock(RWLock *rw) {
    pthread_mutex_lock(&rw->lock);
    while (rw->writer_active || rw->writers_waiting > 0) {
        pthread_cond_wait(&rw->readers_proceed, &rw->lock);
    }
    rw->readers_count++;
    pthread_mutex_unlock(&rw->lock);
}

void read_unlock(RWLock *rw) {
    pthread_mutex_lock(&rw->lock);
    rw->readers_count--;
    if (rw->readers_count == 0) {
        pthread_cond_signal(&rw->writers_proceed);
    }
    pthread_mutex_unlock(&rw->lock);
}
```

## 八、事务内存

事务内存（Transactional Memory）作为锁的替代方案：

```python
# 软件事务内存概念示例
from functools import wraps

def transactional(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        # 简化的事务模型
        attempts = 0
        while attempts < 3:
            try:
                with atomic_context():
                    return func(*args, **kwargs)
            except ConflictError:
                attempts += 1
        raise DeadlockError("Transaction failed after retries")
    return wrapper

@transactional
def transfer_money(account_from, account_to, amount):
    # 数据库操作，事务保证原子性
    if account_from.balance >= amount:
        account_from.balance -= amount
        account_to.balance += amount
```
