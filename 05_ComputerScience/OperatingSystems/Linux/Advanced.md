# 正则表达式进阶

## 前瞻与后顾 (Lookaround)

```regex
# 前瞻（Lookahead）
# 正向前瞻：匹配后跟特定模式的位置
(?=pattern)     # 正向前瞻（positive lookahead）

# 负向前瞻：匹配后不跟特定模式的位置
(?!pattern)     # 负向前瞻（negative lookahead）

# 后顾（Lookbehind）
# 正向后顾：匹配前有特定模式的位置
(?<=pattern)    # 正向后顾（positive lookbehind）

# 负向后顾：匹配前没有特定模式的位置
(?<!pattern)    # 负向后顾（negative lookbehind）
```

### 前瞻示例

```regex
# 正向前瞻示例
(?=\d{3})       # 后面跟着 3 位数字的位置
\w+(?=\.txt)    # 匹配 ".txt" 前的文件名
(?=\d{3})(?=\d{4})  # 同时满足多个条件

# 应用：千位分隔符
# 将 "1234567" 替换为 "1,234,567"
(?<=\d)(?=(\d{3})+(?!\d))  # 千位分隔位置

# 密码强度检查（正向前瞻实现）
(?=.*[a-z])     # 必须包含小写字母
(?=.*[A-Z])     # 必须包含大写字母
(?=.*\d)        # 必须包含数字
(?=.*[!@#$%])   # 必须包含特殊字符
.{8,}           # 至少 8 位

# 组合：
^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)(?=.*[!@#$%]).{8,}$

# 负向前瞻示例
(?!\d+$)        # 不以纯数字结尾
\b(?!un)\w+\b   # 不以 "un" 开头的单词
^(?!admin).*    # 不以 "admin" 开头的字符串
```

### 后顾示例

```regex
# 正向后顾示例
(?<=\$)         # 前面有美元符号的位置
(?<=\bMr\.?\s)\w+   # Mr. 后的名字
(?<=<[^>]+>)    # 在 HTML 标签内的位置

# 提取价格数字
(?<=\$)\d+(\.\d{2})?   # 匹配 $100.50 中的 100.50

# 提取用户名（邮箱前缀）
(?<=@)[\w.-]+   # 匹配邮箱域名

# 负向后顾示例
(?<!un)         # 前面没有 "un" 的位置
(?<!\d)         # 前面没有数字的位置
(?<!Mr\.?\s)    # 前面没有 Mr.

# 匹配非特定前缀的单词
\b(?<!un)\w+\b  # 匹配不以 "un" 开头的单词
(?<![A-Z])\w+   # 不匹配前有大写字母的单词
```

### 实战：URL 解析

```regex
# 匹配 URL 但不包含协议
(?<=https?://)[^\s/?#]+   # 提取域名（需后顾支持）

# 替代方案（无后顾时）
https?://([^\s/?#]+)      # 使用捕获组
```

## 占有量词

```regex
# 占有量词：匹配后不回溯
*+          # 占有零次或多次
++          # 占有一次或多次
?+          # 占有零次或一次
{n}+        # 占有精确次数
{n,}+       # 占有至少 n 次
{n,m}+      # 占有 n 到 m 次

# 示例
\w++        # 占有式匹配单词字符
\d++        # 占有式匹配数字
[^"]*+      # 占有式匹配非引号字符（用于提高性能）
A*+B        # 匹配 "AAAAAB"，一旦匹配 A*+ 不会回溯

# 与贪婪/懒惰对比
" 文本 "a" 和 "b" "  # 要匹配引号内容

".*"               # 贪婪：匹配 " 文本 "a" 和 "b" "（整个字符串）
".*?"              # 懒惰：匹配 " 文本 " 和 "a" 和 "b"（多次）
"[^"]*+"           # 占有：匹配 " 文本 "（安全快速）
```

## 原子组

```regex
# 原子组：组内回溯被禁用，整体作为一个原子
(?>pattern)     # 原子组

# 不匹配时不会回溯到组内
(?>a*)b         # 如果 "aaaa" 后没有 b，不会回溯减少 a 的数量

# 应用：防止灾难性回溯
# 问题模式：(a|aa|aaa)+b
# 原子组修复：(?>a|aa|aaa)+b
```

## 回溯与灾难性回溯

```regex
# 灾难性回溯示例
(a|aa|aaa)+b   # 输入 "aaaaaaaaac" 会导致指数级回溯
(.+)+          # 输入较长字符串可能导致超时
(\d+|[\d]+)b   # 嵌套量词导致回溯
<.*>.*<\/.*>   # HTML 匹配可能导致大量回溯

# 解决方案
(?:a|aa|aaa)+b     # 非捕获组（无效）
(?>a|aa|aaa)+b     # 原子组（有效）
a+b                # 简化（有效）

# 优化原则
# 1. 避免嵌套量词
# 2. 使用原子组
# 3. 使用占有量词
# 4. 使模式更具体
```

## 递归模式（PCRE）

```regex
# 递归匹配括号
\((?:[^()]++|(?R))*\)    # 匹配嵌套括号
\(([^()]|(?R))*\)        # 同上（另一种写法）

# 递归匹配 HTML
<(\w+)(?:[^>]*)>(?:[^<]|(?R))*</\1>  # 匹配嵌套 HTML 标签

# 示例应用
(1+2)*(3-(4/5))     # 可以被上面的模式匹配
<div><p>text</p></div>  # 匹配完整嵌套标签
```

## 命名捕获

```regex
# 各种正则引擎的命名捕获语法

# PCRE/PHP
(?P<name>pattern)   # 定义命名组 - PHP 风格
(?P=name)           # 反向引用命名组
(?&name)            # 递归调用命名组

# PCRE 新语法（PHP 7+）
(?<name>pattern)    # 定义命名组
\k<name>            # 反向引用
\g{name}            # 反向引用（另一种语法）

# .NET
(?<name>pattern)    # 定义命名组
\k<name>            # 反向引用

# Python
(?P<name>pattern)   # 定义命名组
(?P=name)           # 反向引用

# JavaScript（ES2018+）
(?<name>pattern)    # 定义命名组
\k<name>            # 反向引用

# 示例
(?<year>\d{4})-(?<month>\d{2})-(?<day>\d{2})
# 匹配 "2024-01-15"
# 可通过名字获取 year=2024, month=01, day=15

# 命名反向引用
(?<tag>\w+)>.*?</\k<tag>>
# 匹配 <div>...</div>、<span>...</span> 等
```

## 条件模式

```regex
# 条件匹配语法
(?(condition)yes-pattern|no-pattern)
(?(condition)yes-pattern)   # 无 else

# 基于捕获组的条件
(\()?[^()]+(?(1)\)) 
# 如果出现左括号，则必须有右括号

# 基于命名组的条件
(?<open>\()?[^()]+(?(open)\))

# 基于 Lookahead 的条件
(?(?=\d)\d{3}|[a-z]{3})
# 如果是数字开头，匹配 3 位数字；否则匹配 3 个小写字母

# 示例：匹配引号字符串
(["'])?[^"']+(?(1)\1)
# 如果出现引号，则匹配相同的结束引号
```

## Unicode 类别

```regex
# Unicode 属性（需启用 UCP 标志）

# 通用类别
\p{L}       # 任意字母
\p{Lu}      # 大写字母
\p{Ll}      # 小写字母
\p{Lt}      # 首字母大写
\p{N}       # 任意数字
\p{Nd}      # 十进制数字
\p{P}       # 标点符号
\p{S}       # 符号
\p{Z}       # 分隔符
\p{M}       # 组合标记
\p{C}       # 控制字符

# 脚本
\p{Han}     # 汉字（CJK）
\p{Hiragana} # 平假名
\p{Katakana} # 片假名
\p{Greek}   # 希腊字母
\p{Cyrillic} # 西里尔字母
\p{Arabic}  # 阿拉伯字母

# 否定
\P{L}       # 非字母

# 结合使用
[\p{L}\p{N}]    # 字母或数字
[\p{Han}\p{Hiragana}\p{Katakana}]  # 中日文字符

# 示例
\p{Han}+        # 匹配一个或多个汉字
[\p{Lu}]\w+     # 大写字母开头的单词（含 Unicode）
\b\p{L}+\b      # 匹配任意语言的单词
```
