# 内存管理详解 (Memory Management)

## 一、内存层次 (Memory Hierarchy)

| 层次 | 典型容量 | 访问时间 | 管理方式 |
|------|---------|---------|---------|
| 寄存器 | 1KB | 1ns | 编译器 |
| 缓存 (L1-L3) | 1-32MB | 2-20ns | 硬件 |
| 物理内存 | 4-512GB | 100ns | OS |
| 磁盘交换 | TB级 | 10ms | OS |

## 二、地址绑定 (Address Binding)

### 绑定阶段

| 阶段 | 时间 | 地址类型 |
|------|------|---------|
| **编译时 (Compile Time)** | 编译 | 绝对地址 |
| **加载时 (Load Time)** | 程序加载 | 可重定位地址 |
| **执行时 (Execution Time)** | 运行时 | 动态重定位（需MMU） |

### 逻辑地址 vs 物理地址

```
CPU生成逻辑地址
     ↓
MMU (内存管理单元) ─→ 物理地址 → 内存
     ↑
   页表/段表
```

## 三、连续分配 (Contiguous Allocation)

### 固定分区 vs 动态分区

| 特性 | 固定分区 | 动态分区 |
|------|---------|---------|
| 分区大小 | 固定 | 按需 |
| 内部碎片 | ✓ | ✗ |
| 外部碎片 | ✗ | ✓ |
| 灵活性 | 低 | 高 |

### 动态分区分配算法

| 算法 | 搜索策略 | 碎片大小 | 速度 |
|------|---------|---------|------|
| **First-Fit** | 首个足够大的 | 小碎片多 | 快 |
| **Best-Fit** | 最小的足够大 | 极小小碎片 | 中 |
| **Worst-Fit** | 最大的空闲块 | 碎片较大 | 中 |
| **Next-Fit** | 从上一次位置开始 | 分布均匀 | 快 |

### 紧缩 (Compaction)

移动已分配进程以合并空闲块。需运行时地址绑定支持。

### 伙伴系统 (Buddy System)

将内存按2的幂划分，相邻且大小相同的空闲块可合并。

```
申请64KB:
128KB → 分裂为 64KB + 64KB → 返回一个

释放64KB:
检查伙伴(相邻同大小)是否空闲 → 合并为128KB
```

## 四、分页 (Paging)

### 基本概念

物理内存划分为帧 (Frame)，逻辑空间划分为页 (Page)，帧和页大小相同。

```
逻辑地址: [页号(p)] [页内偏移(d)]
                ↓
           页表映射
                ↓
物理地址: [帧号(f)] [页内偏移(d)]
```

### 页表 (Page Table)

| 页表项字段 | 说明 |
|-----------|------|
| **Frame Number** | 物理帧号 |
| **Valid/Invalid** | 页是否在内存 |
| **Protection** | 读写权限 |
| **Dirty** | 是否被修改 |
| **Referenced** | 是否被访问 |
| **Caching** | 缓存策略 |

### TLB (Translation Lookaside Buffer)

TLB是页表的硬件缓存，存储最近使用的页表项。

```
EAT = (TLB_hit × TLB_time + TLB_miss × (TLB_time + 2×mem_time)) + mem_time

假设：TLB=2ns, mem=100ns, hit=99%
EAT = 0.99×2 + 0.01×202 + 100 = 103.98ns
```

### 页大小权衡

| 页大小 | 内部碎片 | 页表大小 | I/O效率 |
|--------|---------|---------|--------|
| 4KB（小） | 小 | 大 | 差 |
| 2MB（大） | 大 | 小 | 好 |

### 多级页表

x86-64四级页表结构：

```
逻辑地址 (48位):
┌─────┬─────┬─────┬─────┬──────┐
│ PML4│ PDPT│  PD │  PT │offset│
│ 9bit│ 9bit│9bit │9bit │12bit │
└──┬──┴──┬──┴──┬──┴──┬──┴──────┘
   ↓     ↓     ↓     ↓
  CR3 → PML4 → PDPT → PD → PT → 物理帧
```

| 级别 | 名称 | 覆盖范围 |
|------|------|---------|
| PML4 | 页映射表4 | 512GB |
| PDPT | 页目录指针表 | 1GB |
| PD | 页目录 | 2MB |
| PT | 页表 | 4KB |

### 哈希页表

用哈希表将虚拟地址映射到物理地址。解决稀疏地址空间问题。

### 倒排页表

对物理帧建表而非虚拟页，表大小与物理内存成比例。

```
物理帧0: <PID, 虚拟页号>
物理帧1: <PID, 虚拟页号>
...
物理帧N: <PID, 虚拟页号>
```

## 五、分段 (Segmentation)

程序按逻辑段划分：代码、数据、堆、栈等。

```
逻辑地址: [段号(s)] [段内偏移(d)]

段表:
┌─────┬────────┬──────┐
│ 段号 │ 基址   │ 段长  │
├─────┼────────┼──────┤
│  0  │ 0x4000 │  1K  │
│  1  │ 0x8000 │  2K  │
│  2  │ 0xA000 │  3K  │
│  3  │ 0xF000 │  1K  │
└─────┴────────┴──────┘
```

### 分段 + 分页 (x86架构)

```
逻辑地址: [段选择子] [偏移]
               ↓
         段描述符表 (GDT/LDT)
               ↓
         得到线性地址
               ↓
         页表转换
               ↓
         物理地址
```

| 组件 | 功能 |
|------|------|
| **GDT** (全局描述符表) | 内核段描述符 |
| **LDT** (局部描述符表) | 进程段描述符 |
| **段选择子** | 16位：索引+TI位+RPL |
| **CS, DS, SS** | 代码段、数据段、栈段寄存器 |

## 六、虚拟内存 (Virtual Memory)

将部分地址空间放在磁盘，按需加载到内存。

### 请求分页 (Demand Paging)

```
页面访问流程：
1. CPU访问逻辑地址
2. 查页表 → 有效位=0 → 触发缺页中断
3. 保存寄存器状态
4. 检查地址合法性
5. 从磁盘读取页面到空闲帧
6. 更新页表（有效位=1）
7. 重启指令
```

### 写时复制 (Copy-on-Write)

fork() 时父子进程共享物理内存，仅写入时才复制。

```
fork前：      fork后（共享）：  写入时：
进程A        进程A ← 共享 → 进程B  进程A  进程B
页0→帧0      页0→帧0       页0→帧0  页0→帧0 页0→帧2
页1→帧1      页1→帧1       页1→帧1  页1→帧1 页1→帧1
```

## 七、页面置换算法 (Page Replacement)

### 算法对比

| 算法 | 描述 | 实现 | Belady异常 | 性能 |
|------|------|------|-----------|------|
| **FIFO** | 换入最早页面 | 队列 | ✓ | 差 |
| **Optimal (MIN)** | 换出最远将来使用的 | 不可实现 | N/A | 最优 |
| **LRU** | 换出最久未使用的 | 计数/栈 | ✗ | 好 |
| **Clock (二次机会)** | 近似LRU，引用位扫描 | 硬件引用位 | ✗ | 较好 |
| **增强Clock** | (引用位,修改位) 组合 | 4类优先级 | ✗ | 好 |
| **NFU (老化)** | 定期右移引用位 | 软件计数器 | ✗ | 接近LRU |

### LRU实现

```c
// 近似LRU: 老化算法
#define BITS 8  // 历史位数

unsigned int nfu_counters[NUM_PAGES];

void clock_interrupt() {
    for (int i = 0; i < NUM_PAGES; i++) {
        nfu_counters[i] >>= 1;           // 右移
        if (referenced[i]) {
            nfu_counters[i] |= (1 << (BITS-1));  // 置最高位
            referenced[i] = 0;
        }
    }
}

int choose_victim() {
    int min = 0;
    for (int i = 1; i < NUM_PAGES; i++) {
        if (nfu_counters[i] < nfu_counters[min])
            min = i;
    }
    return min;  // 最小计数器 → 最近最少使用
}
```

### Clock算法

```
时钟指针扫描页面：
  若引用位=1 → 清零，继续
  若引用位=0 → 置换该页

增强型：
  (0,0) → 最佳置换
  (0,1) → 需写回
  (1,0) → 可能再用
  (1,1) → 最差
```

## 八、工作集与颠簸 (Working Set & Thrashing)

### 工作集模型

工作集是进程在时间窗口 $\Delta$ 内访问的页面集合。

```
若 工作集大小 > 可用物理帧 → 颠簸 (Thrashing)
```

### 缺页频率控制 (PFF)

```
if (缺页率 > 上限) → 增加分配帧数
if (缺页率 < 下限) → 减少分配帧数
```

### 分配策略

| 策略 | 描述 | 适用 |
|------|------|------|
| **固定分配** | 进程分配的帧数固定 | 简单系统 |
| **可变分配** | 根据缺页率动态调整 | 现代OS |
| **全局置换** | 从所有帧中选择换出 | 灵活性高 |
| **局部置换** | 仅从本进程帧中选择 | 隔离性好 |
