# 自动化测试框架

## 一、自动化测试概述

自动化测试是用工具和脚本代替手工执行的测试过程，是软件质量保障的核心手段。

测试金字塔（Test Pyramid）：

```
        /\
       /  \         E2E (End-to-End) - 少量
      /    \
     /      \       Integration - 适量
    /        \
   /          \     Unit (单元测试) - 大量
  /____________\
```

测试收益与成本：

| 测试层级 | 执行速度 | 维护成本 | 发现的问题 | 覆盖率 |
|----------|----------|----------|-----------|--------|
| 单元测试 | 毫秒级 | 低 | 逻辑错误 | 高 |
| 集成测试 | 秒级 | 中 | 接口问题 | 中 |
| E2E测试 | 分钟级 | 高 | 流程问题 | 低 |
| 手动测试 | 小时级 | 极高 | UX问题 | 不定 |

## 二、单元测试框架

主流单元测试框架对比：

| 语言 | 框架 | 断言风格 | Mock支持 | 覆盖率工具 |
|------|------|----------|----------|-----------|
| JavaScript | Jest | expect | 内置 | istanbul |
| JavaScript | Vitest | expect | 内置 | c8 |
| Python | pytest | assert | pytest-mock | coverage.py |
| Java | JUnit 5 | Assertions | Mockito | JaCoCo |
| C# | xUnit | Assert | Moq | Coverlet |
| Go | testing | t.Error | 内置 | go test -cover |
| Rust | cargo test | assert! | 内置 | tarpaulin |

单元测试示例（Jest）：

```javascript
// math.js - 被测函数
export function calculateTotal(items, taxRate) {
  const subtotal = items.reduce((sum, item) => sum + item.price * item.quantity, 0);
  const tax = subtotal * taxRate;
  return subtotal + tax;
}

// math.test.js - 测试用例
import { calculateTotal } from './math';

describe('calculateTotal', () => {
  const items = [
    { price: 10, quantity: 2 },
    { price: 5, quantity: 3 },
  ];

  test('calculates correct total without tax', () => {
    expect(calculateTotal(items, 0)).toBe(35);
  });

  test('applies tax correctly', () => {
    expect(calculateTotal(items, 0.1)).toBeCloseTo(38.5, 2);
  });

  test('returns 0 for empty items', () => {
    expect(calculateTotal([], 0.1)).toBe(0);
  });

  test('handles single item', () => {
    expect(calculateTotal([{ price: 20, quantity: 1 }], 0.08))
      .toBeCloseTo(21.6, 2);
  });
});
```

## 三、模拟与桩

测试替身（Test Double）类型：

| 类型 | 用途 | 示例 |
|------|------|------|
| Dummy | 仅为了满足参数 | 空对象 |
| Stub | 返回预设值 | 固定响应 |
| Spy | 记录调用信息 | 调用计数 |
| Mock | 预定义期望行为 | 验证交互 |
| Fake | 简化实现 | 内存数据库 |

```java
// Mockito示例 (Java)
@ExtendWith(MockitoExtension.class)
class OrderServiceTest {

    @Mock
    private PaymentGateway paymentGateway;

    @Mock
    private EmailService emailService;

    @InjectMocks
    private OrderService orderService;

    @Test
    void placeOrder_shouldProcessPaymentAndSendEmail() {
        // Arrange
        Order order = new Order("user@example.com", 100.0);
        when(paymentGateway.charge("user@example.com", 100.0))
            .thenReturn(new PaymentResult(true, "txn_123"));

        // Act
        Order result = orderService.placeOrder(order);

        // Assert
        assertThat(result.getStatus()).isEqualTo(OrderStatus.CONFIRMED);
        assertThat(result.getTransactionId()).isEqualTo("txn_123");

        verify(paymentGateway).charge("user@example.com", 100.0);
        verify(emailService).sendOrderConfirmation(result);
    }

    @Test
    void placeOrder_paymentFails_shouldThrowException() {
        Order order = new Order("user@example.com", 100.0);
        when(paymentGateway.charge(anyString(), anyDouble()))
            .thenReturn(new PaymentResult(false, null));

        assertThrows(PaymentException.class, () -> {
            orderService.placeOrder(order);
        });

        verify(emailService, never()).sendOrderConfirmation(any());
    }
}
```

## 四、集成测试

集成测试验证多个模块之间的交互是否正确。

```python
# pytest集成测试示例
import pytest
import httpx
from sqlalchemy import create_engine
from sqlalchemy.orm import sessionmaker

# 测试数据库配置
TEST_DATABASE_URL = "postgresql://test:test@localhost:5432/test_db"

@pytest.fixture
def db_session():
    engine = create_engine(TEST_DATABASE_URL)
    Session = sessionmaker(bind=engine)
    session = Session()
    yield session
    session.rollback()
    session.close()

@pytest.fixture
def test_client():
    return httpx.Client(base_url="http://localhost:8000")

class TestUserAPI:
    def test_create_user(self, db_session, test_client):
        # 发送请求
        response = test_client.post("/api/users", json={
            "name": "Alice",
            "email": "alice@example.com"
        })

        # 验证响应
        assert response.status_code == 201
        data = response.json()
        assert data["name"] == "Alice"
        assert data["id"] is not None

        # 验证数据库
        result = db_session.execute(
            "SELECT name FROM users WHERE id = :id",
            {"id": data["id"]}
        ).fetchone()
        assert result[0] == "Alice"

    def test_get_nonexistent_user(self, test_client):
        response = test_client.get("/api/users/99999")
        assert response.status_code == 404

    def test_duplicate_email(self, test_client, db_session):
        # 创建第一个用户
        test_client.post("/api/users", json={
            "name": "Alice", "email": "dup@example.com"
        })

        # 尝试创建重复邮箱
        response = test_client.post("/api/users", json={
            "name": "Bob", "email": "dup@example.com"
        })
        assert response.status_code == 409
```

## 五、端到端测试

E2E测试模拟真实用户操作流程。

```javascript
// Cypress E2E测试
describe('Shopping Cart Flow', () => {
  beforeEach(() => {
    cy.visit('/');
    cy.fixture('users').as('userData');
  });

  it('should complete a purchase successfully', () => {
    // 搜索商品
    cy.get('[data-cy=search-input]').type('wireless mouse');
    cy.get('[data-cy=search-button]').click();
    cy.get('[data-cy=product-card]').first().click();

    // 添加到购物车
    cy.get('[data-cy=add-to-cart]').click();
    cy.get('[data-cy=cart-count]').should('contain', '1');

    // 结算
    cy.get('[data-cy=checkout-button]').click();
    cy.get('[data-cy=email-input]').type('user@example.com');
    cy.get('[data-cy=address-input]').type('123 Main St');
    cy.get('[data-cy=place-order]').click();

    // 验证成功
    cy.get('[data-cy=order-confirmation]').should('be.visible');
    cy.get('[data-cy=order-number]').should('not.be.empty');
  });

  it('should handle empty cart checkout', () => {
    cy.visit('/cart');
    cy.get('[data-cy=empty-cart-message]').should('be.visible');
    cy.get('[data-cy=checkout-button]').should('be.disabled');
  });
});
```

## 六、API测试

```python
# pytest + requests REST API测试
import pytest
import requests
from jsonschema import validate

BASE_URL = "https://api.example.com/v1"

USER_SCHEMA = {
    "type": "object",
    "properties": {
        "id": {"type": "integer"},
        "name": {"type": "string"},
        "email": {"type": "string"},
        "created_at": {"type": "string"}
    },
    "required": ["id", "name", "email"]
}

class TestUserAPI:
    def setup_method(self):
        self.session = requests.Session()
        self.session.headers.update({
            "Authorization": "Bearer test_token",
            "Content-Type": "application/json"
        })

    def test_list_users_returns_valid_schema(self):
        response = self.session.get(f"{BASE_URL}/users", params={
            "page": 1, "per_page": 20
        })
        assert response.status_code == 200

        data = response.json()
        assert "data" in data
        assert "pagination" in data
        for user in data["data"]:
            validate(instance=user, schema=USER_SCHEMA)

    def test_pagination_works_correctly(self):
        page1 = self.session.get(f"{BASE_URL}/users", params={
            "page": 1, "per_page": 10
        }).json()
        page2 = self.session.get(f"{BASE_URL}/users", params={
            "page": 2, "per_page": 10
        }).json()

        assert len(page1["data"]) == 10
        assert page1["pagination"]["page"] == 1
        assert page2["pagination"]["page"] == 2
```

## 七、性能测试

```python
# locust性能测试
from locust import HttpUser, task, between
import random

class WebsiteUser(HttpUser):
    wait_time = between(1, 3)

    def on_start(self):
        """用户登录"""
        self.client.post("/login", json={
            "username": f"user_{random.randint(1, 100)}",
            "password": "test123"
        })

    @task(3)
    def view_products(self):
        self.client.get("/products", params={"page": 1})

    @task(2)
    def view_product_detail(self):
        product_id = random.randint(1, 1000)
        self.client.get(f"/products/{product_id}")

    @task(1)
    def add_to_cart(self):
        self.client.post("/cart", json={
            "product_id": random.randint(1, 1000),
            "quantity": 1
        })

    @task(1)
    def checkout(self):
        self.client.post("/checkout", json={
            "address": "123 Test St",
            "payment_method": "credit_card"
        })
```

性能测试指标：

| 指标 | 定义 | 良好值 | 警告值 | 危险值 |
|------|------|--------|--------|--------|
| 响应时间 (p50) | 50%请求的延迟 | < 200ms | 200-500ms | > 500ms |
| 响应时间 (p99) | 99%请求的延迟 | < 500ms | 500ms-2s | > 2s |
| 吞吐量 (RPS) | 每秒请求数 | 按需设定 | 下降20% | 下降50% |
| 错误率 | 失败请求占比 | < 0.1% | 0.1-1% | > 1% |
| 并发用户数 | 同时在线用户 | 按需设定 | 达到极限 | 系统崩溃 |
