# 软件架构 Software Architecture

软件架构是软件系统的高层结构设计，定义了系统的组件、组件间关系及其交互方式。好的软件架构能够满足功能和非功能需求（如性能、可扩展性、可维护性和安全性），是软件开发成功的关键基础。

常见的软件架构模式包括分层架构（如 MVC、三层架构）、微服务架构、事件驱动架构、管道-过滤器架构和微内核架构。微服务架构将应用拆分为独立部署的小型服务，每个服务专注于特定的业务功能，通过轻量级通信机制（如 REST、gRPC）交互。架构风格的选择取决于系统的规模、复杂度和业务需求。

架构设计需要遵循一些基本原则：关注点分离、单一职责、开闭原则和依赖反转。架构评估方法（如 ATAM、SAAM）用于评估架构决策的质量。领域驱动设计（DDD）帮助将业务需求映射到架构设计。架构文档（如 4+1 视图模型）用于沟通和交流架构决策。随着云原生技术的发展，架构设计也越来越关注弹性、可观测性和自动化运维。
