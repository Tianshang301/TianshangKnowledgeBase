# 数据集选型指南

## 一、数据集选择的核心考量

选择合适的数据集需平衡以下因素：
1. 任务匹配度：数据集是否包含解决目标问题所需的数据类型和标注
2. 规模与质量：数据量是否充足，标注是否正确一致
3. 领域相关性：数据分布是否接近实际应用场景
4. 可获取性：是否公开可用，是否有下载限制
5. 许可协议：是否允许研究或商业使用
6. 成本：内存和计算资源需求是否可承受
7. 时效性：数据是否过时，是否反映当前领域状态
8. 偏差评估：数据是否存在性别、种族、地域等代表性偏差

## 三、数据质量评估清单
检查以下维度的数据质量：
1. 完整性：
   - 缺失值比例及分布
   - 字段是否按预期填写
   - 是否有记录不完整

2. 一致性：
   - 格式和单位是否统一
   - 分类标签的命名规范是否一致
   - 跨字段逻辑关系是否合理

3. 准确性：
   - 标注是否正确（抽样验证）
   - 数值是否有异常值
   - 是否有录入错误
4. 时效性：
   - 数据采集时间
   - 标注完成时间
   - 在目标场景中是否仍有效
## 四、偏差与公平性考量

常见偏差类型：
| 偏差类型 | 说明 | 示例 |
|----------|------|------|
| 采样偏差 | 样本不能代表总体分布 | 面部识别数据集肤色分布不均 |
| 标注偏差 | 标注者主观判断差异 | 情感标注受文化背景影响 |
| 历史偏差 | 数据反映已有社会偏见 | 招聘数据中的性别偏好 |
| 测量偏差 | 特征测量方法系统性错误 | 传感器位置偏差 |
| 报告偏差 | 结果选择性报告 | 只报告表现好的子集结果 |

缓解策略：
- 分析数据在人口统计学维度上的分布
- 使用分层采样确保子群体均衡
- 对敏感属性进行去偏处理
- 进行子群体性能分析
- 数据集文档化（Datasheet for Datasets）
## 五、许可协议
| 许可类型 | 允许商业使用 | 允许修改 | 需要署名 | 常见数据集 |
|----------|-------------|----------|----------|-----------|
| CC0 | 是 | 是 | 否 | 部分政府开放数据 |
| CC BY 4.0 | 是 | 是 | 是 | 部分学术数据集 |
| CC BY-SA 4.0 | 是 | 是 | 是（相同方式共享）| 维基百科相关 |
| CC BY-NC 4.0 | 否 | 是 | 是 | 部分学术数据集 |
| MIT | 是 | 是 | 是 | 部分代码+数据项目 |
| 自定义学术许可 | 否 | 有条件 | 是 | ImageNet, COCO |

使用前务必阅读完整许可条款，特别是对模型再分发的限制。
