# 论文结构指南

## 一、学术论文标准结构：IMRaD 格式

IMRaD 是科研论文最通用的结构框架，四个核心部分为：

| 部分 | 英文 | 核心功能 | 常见篇幅占比 |
|------|------|----------|-------------|
| 引言 | Introduction | 说明研究背景、问题、目的 | 15-20% |
| 方法 | Methods | 描述实验/研究设计、数据采集与分析 | 20-25% |
| 结果 | Results | 呈现研究发现，客观陈述 | 25-30% |
| 讨论 | Discussion | 解释结果意义，与已有研究对比，局限与展望 | 25-30% |

其余必要组成部分：标题、摘要、关键词、致谢、参考文献、附录。

## 二、摘要写作
摘要类型有三种：

1. 信息性摘要：包含研究目的、方法、主要结果和结论，适用于大多数实证研究论文。通常 200-250 词。
2. 指示性摘要：仅概述论文主题和范围，不提供具体结果，适用于综述、评论类文章。通常 100-150 词。
3. 结构化摘要：按小标题分段（目的、方法、结果、结论），常见于医学和临床研究期刊。

写作原则：
- 避免引用参考文献
- 不使用缩略词（除非首次定义）
- 包含 3-6 个关键词，便于检索
- 结果部分给出具体数据而非笼统描述
- 最后一句总结核心结论

## 三、引言写作策略

采用"漏斗式"结构，从宏观到微观：

1. 第一层：研究领域的背景和重要性（广泛）
2. 第二层：已有研究进展，指出知识缺口
3. 第三层：本文的研究问题、目的和假设
4. 第四层：研究方法和论文结构预览

常见问题：引言过长、文献综述替代了引言、研究目的不明确、缺乏研究假设。

## 四、方法部分
核心原则：可重复性。读者应能根据描述完全复现研究。
内容清单：
- 研究设计类型（实验、观察、队列、横断面等）
- 样本/被试描述（来源、数量、纳入排除标准）
- 数据采集工具和流程
- 变量定义和测量方法
- 统计分析方法（包括使用的软件和版本）
- 伦理声明（如适用）

## 五、结果与讨论

结果部分要点：
- 先概述整体发现，再逐一详述
- 图表用于呈现关键数据，文字用于解释图表中的核心信息
- 不在此处解释结果的意义（留到讨论部分）
- 报告统计显著性时应同时报告效应量

讨论部分结构：
1. 重申主要发现
2. 与已有研究比较（支持或矛盾的发信）
3. 研究的优势和局限性
4. 对理论和实践的启示
5. 未来研究方向
6. 结论

## 六、跨学科结构对比

| 学科领域 | 常见结构 | 特点 |
|----------|----------|------|
| 自然科学/工程 | IMRaD 或其变体 | 方法、结果严格分离，数据驱动 |
| 社会科学 | IMRaD, 或章节式 | 理论框架单独成章常见 |
| 人文学科 | 主题章节式 | 论证驱动，不严格遵循 IMRaD |
| 计算机科学（会议） | 引言-方法-实验-结论 | 实验部分常包含消融研究 |
| 医学 | IMRaD + 结构化摘要 | CONSORT/STROBE 等报告规范 |
| 综述论文 | 引言-主题1/2/3-讨论 | 系统性综述需遵循 PRISMA |

## 七、结论撰写
- 长度：1-3 段或 3-5 条要点
- 内容：总结核心发现，阐明学术贡献，指出实际应用价值
- 避免：简单重复摘要内容，过度推广结论，提出未经数据支持的论断

## 八、常见错误清单
| 错误类型 | 表现 | 修正方法 |
|----------|------|----------|
| 逻辑断裂 | 各节之间缺乏过渡 | 每节末尾/开头加过渡句 |
| 过度引用 | 一段话中连续多个引用 | 只引关键文献，用自己的话综述 |
| 结果与讨论混淆 | 结果中解释意义 | 严格分离事实陈述与解释 |
| 文献遗漏 | 遗漏关键相关文献 | 投稿前做系统性文献检索 |
| 格式不一致 | 标题层级混乱 | 使用模板，统一各级标题格式 |
