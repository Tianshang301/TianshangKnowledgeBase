# 笔记法

## 概述

笔记（Note-Taking）是知识管理的核心环节。好的笔记不仅是信息的被动记录，更是与知识的主动对话——将外部信息转化为内在理解，并在不同概念之间建立连接。本模块收录主流笔记系统、数字笔记工具策略、以及笔记复习方法。

---

## 主流笔记系统

### 1. Zettelkasten（卡片盒笔记法）

由德国社会学家 Niklas Luhmann 发明，他凭借此方法一生出版了 70+ 本书和 400+ 论文。

**核心原则：**
- **原子化**：每张卡片只记录一个概念或想法
- **关联性**：每张卡片必须链接到其他卡片
- **用自己的话写**：不是摘抄，而是转述和理解
- **无分类**：不按主题归档，依靠链接形成网络

**实施步骤：**
1. **文献笔记**：阅读时简要记录关键信息（附来源）
2. **永久笔记**：将文献笔记中的观点与自己的思考结合，写成原子笔记
3. **建立链接**：将新笔记链接到已有笔记
4. **涌现主题**：通过链接网络发现研究主题和论文方向

**数字工具**：Obsidian、Roam Research、Logseq、RemNote

### 2. Cornell 笔记法（康奈尔笔记法）

由康奈尔大学 Walter Pauk 教授提出，将一页纸分为四个区域：

```
┌─────────────────────┬─────────────────────────────┐
│                     │                             │
│                     │                             │
│    提示列（Cue）      │       笔记正文（Notes）       │
│    （关键词/问题）    │      （课堂/阅读记录）         │
│                     │                             │
│                     │                             │
├─────────────────────┴─────────────────────────────┤
│                                                   │
│               总结（Summary）                       │
│            （3-5 句话概括本页）                     │
└───────────────────────────────────────────────────┘
```

**使用流程：**
1. 记录（Record）：在笔记区记录内容
2. 简化（Reduce）：下课后在提示列提炼关键词或问题
3. 复述（Recite）：遮住笔记区，只看提示列复述
4. 反思（Reflect）：在总结区写下整体理解和疑问
5. 复习（Review）：每周花 10 分钟回顾

### 3. PARA 方法

由 Tiago Forte 提出的数字信息组织系统。

- **P**rojects（项目）：有截止日期的任务
- **A**reas（领域）：需要长期负责的职责
- **R**esources（资源）：感兴趣的主题资料
- **A**rchives（归档）：已完成的或不再活跃的项目

### 4. 渐进式总结（Progressive Summarization）

Tiago Forte 提出的五层笔记精炼方法：

| 层级 | 操作 | 目的 |
|------|------|------|
| Layer 1 | 原始笔记 | 捕捉完整信息 |
| Layer 2 | 加粗重要段落 | 突出关键点 |
| Layer 3 | 高亮最核心的句子 | 提取精华 |
| Layer 4 | 用自己话写摘要 | 压缩理解 |
| Layer 5 | 重新混排成新文章 | 创造输出 |

**原则**：只在必要时进行下一层加工，不追求一次性完成所有层级。

## 数字笔记组织策略

### 文件夹结构 vs 标签系统 vs 双向链接

| 维度 | 文件夹 | 标签 | 双向链接 |
|------|--------|------|----------|
| 结构 | 树状，严格层级 | 扁平，灵活 | 网络状 |
| 维护成本 | 低 | 中 | 高 |
| 检索效率 | 依赖路径记忆 | 多维度查找 | 关联发现 |
| 推荐场景 | 归档检索 | 分类标记 | 知识发现 |

**最佳实践**：三者结合使用。文件夹做顶层归类，标签做横向分类，双向链接构建知识网络。

### 命名约定
- `YYYY-MM-DD-标题.md` — 日记/日志型笔记
- `[学科]/[概念名].md` — 永久笔记
- `[来源类型]-[来源名称].md` — 文献笔记

### 链接策略
- **链路笔记**：使用 `[[双向链接]]` 实现笔记间跳转
- **索引笔记**：建立 MOC（Map of Content），集中管理主题下所有相关笔记
- **标签索引**：用标签聚合跨目录的同类笔记

## 复习与回顾

好的笔记体系必须搭配定期复习才能发挥最大价值。

### 每日回顾
- 浏览当天新创建的笔记
- 修复残缺链接和格式问题
- 添加遗漏的标签

### 每周回顾
- 回顾 MOC，梳理主题进展
- 清理未被链接的孤立笔记（挂掉树叶）
- 决定哪些笔记需要进一步精炼（渐进式总结）

### 每月回顾
- 检查 PARA 中的项目进展
- 归档已完成的笔记
- 重新组织不再合理的目录结构

## 工具推荐

| 工具 | 特点 | 适用场景 |
|------|------|----------|
| Obsidian | 本地存储、双向链接、插件丰富 | 知识管理深度用户 |
| Logseq | 大纲式、开源、支持块引用 | 日常笔记和日记 |
| Notion | 全能型、数据库、团队协作 | 项目管理和知识库 |
| Roam Research | 块级引用、间隔重播 | 学术研究和写作 |
| RemNote | 内置间隔重复 | 学习和备考 |
| OneNote | 微软生态、手写支持 | 课堂笔记团队协作 |
