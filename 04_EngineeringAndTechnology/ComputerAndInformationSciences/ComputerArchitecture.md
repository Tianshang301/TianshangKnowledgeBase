# 计算机体系结构

## 一、冯·诺依曼体系结构

冯·诺依曼体系结构是现代计算机设计的基础，其核心思想包括：

- **存储程序概念**：指令和数据以相同方式存储在存储器中
- **五大部件**：运算器（ALU）、控制器、存储器、输入设备、输出设备
- **顺序执行**：指令逐条从存储器取出并执行

冯·诺依曼瓶颈：由于指令和数据共享同一总线，CPU 和存储器之间的数据传输速率成为系统性能的瓶颈。哈佛体系结构通过分离指令和数据通路缓解此问题，但增加了芯片引脚和缓存一致性复杂度。

### 1.1 现代改进

| 结构 | 特点 | 典型应用 |
|------|------|---------|
| 冯·诺依曼 | 统一存储，单总线 | x86桌面 CPU |
| 哈佛 | 分离指令和数据存储 | DSP、嵌入式 MCU |
| 改进哈佛 | 分离 L1 Cache，统一 L2/L3 | ARM Cortex 系列 |

## 二、CPU 组成

### 2.1 运算器（ALU）

算术逻辑单元执行算术运算（加、减、乘、除）和逻辑运算（与、或、非、异或）。其关键路径延迟决定了 CPU 的主频上限。现代 ALU 还包含专门硬件如乘法器（Booth 编码、Wallace 树）、除法器（SRT 算法）和桶形移位器。

### 2.2 控制器

控制器负责指令译码和控制信号的生成，包括：
- **程序计数器（PC）**：指向下一条指令地址
- **指令寄存器（IR）**：存储当前正在执行的指令
- **控制单元**：组合逻辑或微程序控制方式生成控制信号

**硬布线控制**：用组合逻辑电路直接生成控制信号，速度快但设计复杂（RISC 常用）。
**微程序控制**：将控制信号存储在控制存储器中，灵活但速度较慢（CISC 常用）。

### 2.3 寄存器

| 寄存器类型 | 功能 | 特点 |
|-----------|------|------|
| 通用寄存器 | 存放操作数和结果 | 程序员可见 |
| 专用寄存器 | PC、SP、PSW 等 | 特定用途 |
| 暂存寄存器 | 流水线级间缓冲 | 对程序员透明 |

**寄存器重命名**：通过物理寄存器到逻辑寄存器的映射（使用 RAT 表），消除 WAW 和 WAR 数据依赖，是乱序执行的核心技术之一。

## 三、指令集体系结构

### 3.1 指令分类

- **数据传送**：Load、Store、Move
- **算术运算**：Add、Sub、Mul、Div
- **逻辑运算**：And、Or、Xor、Shift、Rotate
- **控制转移**：Branch（条件）、Jump（无条件）、Call、Return
- **浮点运算**：FAdd、FMul、FDiv、FComp
- **系统指令**：特权指令、中断处理

### 3.2 寻址方式

| 寻址方式 | 有效地址 | 示例 |
|---------|---------|------|
| 立即数 | 指令本身包含操作数 | ADD R1, #10 |
| 寄存器 | EA = 寄存器值 | ADD R1, R2 |
| 直接 | EA = 地址字段 | ADD R1, (1000) |
| 基址 | EA = 基址寄存器 + 偏移 | ADD R1, 100(R2) |
| 变址 | EA = 变址寄存器 + 偏移 | ADD R1, (R2+R3) |
| PC 相对 | EA = PC + 偏移 | BEQ R1, loop |
| 寄存器间接 | EA = *(寄存器) | ADD R1, @R2 |

### 3.3 RISC vs CISC 对比

| 特性 | RISC | CISC |
|------|------|------|
| 指令长度 | 固定（通常32位） | 可变长度（1-15字节） |
| 指令格式 | 简单，种类少（<200条） | 复杂，种类多（>1000条） |
| 寻址方式 | 较少（通常2-3种） | 较多（10种以上） |
| 寄存器使用 | 大量通用寄存器（32+） | 寄存器较少（8-16） |
| 访存指令 | 仅 Load/Store 访存 | 指令可直接操作内存 |
| 译码复杂度 | 低，硬件译码 | 高，微程序译码 |
| CPI | 接近1 | 1-4 |
| 典型代表 | ARM、RISC-V、MIPS | x86、x86-64 |

### 3.4 指令流水线

经典五级流水线：**IF（取指）→ ID（译码）→ EX（执行）→ MEM（访存）→ WB（写回）**

| 阶段 | 操作 | 硬件组件 |
|------|------|---------|
| IF | 从指令 Cache 取指令，PC+4 | 指令存储器、PC、加法器 |
| ID | 指令译码，读寄存器堆 | 译码器、寄存器堆、符号扩展 |
| EX | 执行算术逻辑运算，计算地址 | ALU、加法器、移位器 |
| MEM | 数据 Cache 读写 | 数据存储器、写缓冲 |
| WB | 结果写回寄存器堆 | 写端口、旁路逻辑 |

流水线理想 CPI = 1，但因各类 hazards 导致实际 CPI > 1。

## 四、流水线冒险

### 4.1 结构冒险

多个指令同时竞争同一硬件资源，例如单端口存储器在 IF 和 MEM 阶段的冲突，或者乘法器被多条指令同时占用。解决方案：资源分离（如分离指令 Cache 和数据 Cache）、功能单元复制（多个乘法器）、流水线停顿。

### 4.2 数据冒险

指令之间存在数据依赖关系：
- **RAW（读后写）**：后续指令读取前一条指令写入的数据，真依赖（必须解决）
- **WAR（写后读）**：后续指令写入前一条指令读取的数据，反依赖（寄存器重命名解决）
- **WAW（写后写）**：两条指令写入同一目标，输出依赖（寄存器重命名解决）

**旁路（Forwarding）路径**：
- ALU 到 ALU 旁路：EX 阶段结果直接送到下一指令的 EX 输入
- MEM 到 ALU 旁路：MEM 阶段结果送到下一指令的 EX 输入
- 加载使用延迟：Load 指令后立即使用结果需要插入一个气泡（Load-use 延迟）

### 4.3 控制冒险

分支指令导致流水线无法确定下一条指令地址。解决方案：
- **分支预测**：静态（总是预测跳转/不跳转）和动态（两位饱和计数器、BTB、全局历史）
- **延迟分支**：在分支指令后插入不依赖分支结果的指令（延迟槽，MIPS 采用）
- **预测失败惩罚**：清空流水线（Flush），从正确地址重新取指

分支预测准确率是现代处理器性能的关键，典型预测器准确率达95%以上（如 TAGE 预测器）。

## 五、存储层次结构

### 5.1 层次化存储

$$ \text{速度} \uparrow, \text{容量} \downarrow, \text{成本} \uparrow $$

| 层级 | 容量 | 访问时间 | 管理单位 |
|------|------|---------|---------|
| 寄存器 | ~1KB | 0.3ns | 字 |
| L1 Cache | 32-64KB | 1ns | 块（64B） |
| L2 Cache | 256KB-1MB | 3-10ns | 块 |
| L3 Cache | 2-32MB | 10-40ns | 块 |
| 主存 | 8-512GB | 50-100ns | 页（4KB） |
| SSD | 512GB-2TB | 10-100μs | 页（4KB） |
| 磁盘 | TB 级 | 5-10ms | 文件 |

**局部性原理**：
- **时间局部性**：最近访问过的数据可能在短期内再次被访问
- **空间局部性**：访问过某地址后，其邻近地址也可能被访问

### 5.2 Cache 映射方式

- **直接映射**：每个内存块对应唯一 Cache 行（冲突率高，但硬件简单）
- **全相联映射**：任意内存块可映射到任意 Cache 行（冲突率低，但比较器昂贵）
- **组相联映射**：折中方案，n 路组相联（路数为性能与成本的平衡点，通常4-8路）

**替换策略**：LRU（最近最少使用，性能好但复杂）、Pseudo-LRU（树形 LRU 简化实现）、FIFO（简单但性能差）、随机（实现简单，性能尚可）。

**写策略**：
- **写直达（Write-Through）**：同时写 Cache 和主存（配合写缓冲使用）
- **写回（Write-Back）**：仅写 Cache，替换时写回主存（需脏位，减少总线流量）

### 5.3 虚拟内存

虚拟地址通过 MMU（内存管理单元）转换为物理地址，采用页表或 TLB 加速地址转换。

$$ \text{有效访问时间} = \text{TLB 命中率} \times \text{TLB 访问时间} + (1 - \text{TLB 命中率}) \times (\text{页表访问时间} + \text{内存访问时间}) $$

**多级页表**：减少页表空间占用（x86-64采用4级页表，RISC-V 采用 Sv39/Sv48）。

## 六、指令级并行

### 6.1 超标量处理器

每个时钟周期发射多条指令，需要更复杂的译码和冲突检测逻辑。发射宽度从早期的2路到现代高端处理器的6-8路以上。

### 6.2 乱序执行

指令按数据流顺序执行，而非程序顺序：
- **Tomasulo 算法**：使用保留站和通用数据总线（CDB）实现寄存器重命名
- **重排序缓冲（ROB）**：保证指令按程序顺序提交，处理异常精确和分支预测失败恢复

**执行过程**：取指 → 发射（分派到保留站）→ 执行（操作数就绪时）→ 写结果（CDB 广播）→ 提交（ROB 按序提交）

### 6.3 分支预测高级技术

- **两位饱和计数器**：对每个分支指令维护一个2-bit 状态机（强跳转/弱跳转/弱不跳/强不跳）
- **相关分支预测器**：利用全局分支历史寄存器（GHR）进行预测
- **Tournament 预测器**：组合基于局部历史和全局历史的预测器，通过选择器动态选择
- **分支目标缓冲（BTB）**：缓存分支指令的目标地址
- **返回地址栈（RAS）**：准确预测函数返回地址（压栈/弹栈机制）

## 七、多核与 GPU 架构

### 7.1 多核处理器

| 架构 | 特点 | 优势 | 劣势 |
|------|------|------|------|
| SMP | 所有核心共享内存，统一地址 | 编程简单 | 一致性开销大 |
| NUMA | 非一致内存访问延迟 | 扩展性好（AMD Epyc） | 数据局部性需优化 |
| SMT | 每核心多线程（超线程） | 提高资源利用率 | 单线程性能略降 |

**缓存一致性协议（MESI）**：
- **M（Modified）**：该行被修改，与其他 Cache 不一致
- **E（Exclusive）**：仅本 Cache 有此行，与主存一致
- **S（Shared）**：多个 Cache 共享此行，均与主存一致
- **I（Invalid）**：该行无效

**伪共享**：不同核心频繁修改同一 Cache 行中不同变量，导致不必要的无效化广播和性能下降。

### 7.2 GPU 架构

- **SIMT（单指令多线程）**：以线程束（NVIDIA Warp 为32线程，AMD Wavefront 为64线程）为单位执行
- **层次化线程结构**：Grid → Block → Thread
- **内存层次**：全局内存（大容量高延迟，~200-800 cycle）、共享内存（小容量低延迟，~5-30 cycle，块内共享）、局部内存、寄存器
- **流多处理器（SM）**：GPU 的基本计算单元，包含多个 CUDA 核心、共享内存和调度器

## 八、性能指标

### 8.1 基本指标

$$ \text{CPI} = \frac{\text{总时钟周期数}}{\text{指令总数}} $$

$$ \text{CPU 执行时间} = \text{指令数} \times \text{CPI} \times \text{时钟周期时间} $$

### 8.2 吞吐量与响应时间

- **响应时间（延迟）**：单个任务从开始到完成的时间
- **吞吐量**：单位时间内完成的任务数

$$ \text{吞吐量} = \frac{\text{任务数}}{\text{总时间}} $$

### 8.3 Amdahl 定律

系统加速比受限于系统中可并行化部分的比例。

$$ S = \frac{1}{(1-p) + p/n} $$

其中，$p$为可并行部分比例，$n$为处理器核心数，$S$为理论加速比。

**Gustafson 定律**：随着问题规模增大，可并行部分占比增加，加速比可超过 Amdahl 定律的预测。

### 8.4 功耗与散热

$$ P = CV^2f $$

其中$C$为电容负载，$V$为电压，$f$为频率。动态功耗与频率和电压平方成正比。漏电功耗（静态功耗）随工艺缩小而增大。

**暗硅**：受热功耗限制，芯片上不能所有区域同时运行，部分必须关闭。

### 8.5 性能评测基准

- **SPEC**：CPU 综合基准（SPECint、SPECfp）
- **Dhrystone**：整数性能（DMIPS）
- **Linpack**：浮点性能（GFLOPS/TFLOPS）
- **CoreMark**：嵌入式 CPU 基准
- **PARSEC**：多线程基准
- **MLPerf**：机器学习训练和推理基准
