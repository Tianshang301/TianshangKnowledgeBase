# 操作系统

## 一、操作系统概述

### 1.1 操作系统功能

操作系统是管理计算机硬件与软件资源的系统软件，核心功能包括：
- **进程管理**：进程创建、调度、同步、通信
- **内存管理**：地址空间分配、虚拟内存、内存保护
- **文件系统**：文件存储、目录管理、权限控制
- **设备管理**：I/O驱动、中断处理、DMA控制
- **用户接口**：命令行（CLI）和图形界面（GUI）

### 1.2 操作系统类型

| 类型 | 特点 | 典型系统 | 应用场景 |
|------|------|---------|---------|
| 批处理系统 | 作业批量提交，无交互 | OS/360 | 大型机数据处理 |
| 分时系统 | 时间片轮转，交互式 | UNIX、Linux | 多用户多任务 |
| 实时系统 | 严格时间约束 | VxWorks、RT-Linux | 工业控制、航天 |
| 分布式系统 | 多机协同，资源共享 | Amoeba、Plan 9 | 集群计算 |
| 嵌入式系统 | 资源受限，专用性强 | FreeRTOS、μC/OS | 物联网、移动设备 |

### 1.3 操作系统发展历程

1. **手工操作阶段**（1940s）：插拔线路，无OS
2. **批处理系统**（1950s-60s）：FMS、IBYS等
3. **多道程序系统**（1960s）：OS/360
4. **分时系统**（1970s）：Unix、Multics
5. **个人计算机**（1980s）：MS-DOS、Mac OS
6. **网络/分布式系统**（1990s-至今）：Windows NT、Linux

### 1.4 系统调用与API

用户程序通过系统调用进入内核态执行特权操作。常见系统调用分类：
- **进程控制**：fork、exec、wait、exit
- **文件操作**：open、read、write、close、lseek
- **设备管理**：ioctl、mmap
- **信息维护**：getpid、alarm、sleep
- **通信**：pipe、shmget、msgget、semget、socket

## 二、进程管理

### 2.1 进程与控制块

**进程**是程序的一次执行过程，包含代码段、数据段和堆栈段。

**进程控制块（PCB）** 保存进程信息：
- 进程ID、状态、程序计数器
- CPU寄存器集合
- 内存管理信息（页表指针等）
- I/O状态信息（打开文件列表）
- 调度信息（优先级、时间片剩余量、状态）

### 2.2 进程状态与转换

进程三态模型：**就绪（Ready）→ 运行（Running）→ 阻塞（Blocked）**

进程五态模型增加：**新建（New）和终止（Terminated）**

| 状态转换 | 触发条件 | 说明 |
|---------|---------|------|
| 新建→就绪 | 进程创建完成 | 进入就绪队列 |
| 就绪→运行 | 调度程序选择 | 获得CPU |
| 运行→就绪 | 时间片用完或被抢占 | 失去CPU |
| 运行→阻塞 | 等待I/O或事件 | 主动放弃CPU |
| 阻塞→就绪 | I/O完成或事件到达 | 回到就绪队列 |

### 2.3 上下文切换

操作系统保存当前进程状态，恢复下一个进程状态的过程。上下文切换开销直接影响系统性能，包括：
- 保存/恢复通用寄存器、PC、PSW
- TLB刷新（切换不同地址空间时）
- Cache污染（新进程的工作集与旧进程不同）

切换时间通常为微秒级，调度频率过高会显著降低系统吞吐量。

### 2.4 线程

线程是CPU调度的基本单位，同一进程内的线程共享地址空间和文件描述符。

**多线程模型对比**：
| 模型 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| 多对一 | 多个用户线程映射到一个内核线程 | 切换快 | 不能并行 |
| 一对一 | 每个用户线程对应一个内核线程 | 可并行 | 内核开销大 |
| 多对多 | 多个用户线程映射到多个内核线程 | 灵活 | 实现复杂 |

### 2.5 进程间通信（IPC）

- **管道（Pipe）**：单向字节流，父子进程使用
- **命名管道（FIFO）**：无亲缘关系进程间通信
- **消息队列**：结构化消息传递
- **共享内存**：最快通信方式，需同步机制配合
- **信号量**：同步与互斥
- **信号（Signal）**：异步事件通知
- **Socket**：跨网络进程通信

## 三、CPU调度

### 3.1 调度算法

| 算法 | 策略 | 优点 | 缺点 |
|------|------|------|------|
| FCFS | 先来先服务 | 公平简单 | 护航效应，平均等待时间长 |
| SJF | 最短作业优先 | 平均等待时间最优 | 需要预知执行时间 |
| SRTF | 最短剩余时间优先 | SJF抢占式版本 | 可能导致饥饿 |
| 优先级 | 优先级高优先 | 支持紧急任务 | 低优先级可能饥饿（老化解决） |
| Round Robin | 时间片轮转（通常10-100ms） | 响应时间短 | 时间片选择影响性能 |
| 多级反馈队列 | 多队列+动态优先级 | 平衡响应和吞吐 | 参数配置复杂 |

### 3.2 调度评价指标

- **CPU利用率**：CPU忙碌时间 / 总时间
- **吞吐量**：单位时间完成的进程数
- **周转时间**：$T_{\text{完成}} - T_{\text{提交}}$
- **等待时间**：进程在就绪队列中等待的总时间
- **响应时间**：从提交到首次响应的时间

### 3.3 实时调度

- **速率单调调度（RMS）**：周期越短优先级越高，静态优先级，可调度性上限约69%
- **最早截止时间优先（EDF）**：截止时间越早优先级越高，动态优先级，可调度性上限100%

## 四、进程同步与死锁

### 4.1 同步机制

竞争条件：多个进程并发访问和操作同一共享数据，结果取决于访问顺序。

- **互斥锁（Mutex）**：二进制信号量
- **信号量（Semaphore）**：$P$（wait，减1，可能阻塞）和$V$（signal，加1，可能唤醒）操作
- **管程（Monitor）**：高级抽象，条件变量（wait/signal/broadcast）
- **读写锁**：多个读者或单个写者
- **自旋锁**：忙等，适合短期等待

**经典同步问题**：
1. **生产者-消费者**：有界缓冲区，empty/full/mutex三个信号量
2. **读者-写者**：读者优先（读者不等待）或写者优先
3. **哲学家就餐**：防止死锁的经典示例

### 4.2 死锁

**死锁必要条件**（四个必须同时成立）：
1. **互斥**：资源不可共享
2. **持有并等待**：进程持有资源的同时等待其他资源
3. **不可抢占**：资源只能由持有者主动释放
4. **循环等待**：存在进程-资源循环等待链

### 4.3 死锁处理

| 策略 | 方法 | 特点 |
|------|------|------|
| 预防 | 破坏四个必要条件之一 | 保守，资源利用率低 |
| 避免 | 银行家算法 | 安全性检查，需预知最大需求 |
| 检测与恢复 | 资源分配图检测，终止或回滚 | 允许死锁发生 |
| 忽略 | 鸵鸟算法 | 大多数桌面OS的策略 |

**银行家算法**：系统处于安全状态时，存在安全序列满足所有进程的最大需求。资源分配前检查各进程的Need（最大需求 - 已分配）与Available资源量。

## 五、内存管理

### 5.1 内存分配方式

- **连续分配**：固定分区（内碎片）、动态分区（外碎片），分配算法有首次适应、最佳适应、最差适应、下次适应
- **非连续分配**：分页、分段、段页式

### 5.2 分页与分段

| 特性 | 分页 | 分段 |
|------|------|------|
| 单位大小 | 固定（4KB或更大） | 可变（逻辑段） |
| 用户视角 | 透明 | 可见 |
| 地址空间 | 一维线性 | 二维（段号+偏移） |
| 碎片问题 | 内部碎片 | 外部碎片（首次适配缓解） |
| 保护与共享 | 困难 | 容易 |

**段页式**：先分段再分页，结合段式逻辑组织和页式物理管理优势。

### 5.3 虚拟内存

虚拟内存允许执行大于物理内存的程序，采用**按需调页（Demand Paging）** 机制。

$$ \text{有效访问时间} = (1-p) \times \text{内存访问时间} + p \times \text{缺页处理时间} $$

其中$p$为缺页率。缺页处理时间包括：磁盘寻道延迟 + 传输延迟 + 页表更新 + 进程上下文切换。

**工作集模型**：进程当前频繁访问的页面集合。窗口大小影响工作集精度。
**抖动（Thrashing）**：进程频繁缺页，CPU大部分时间用于调页。原因：工作集超过可用物理内存。

### 5.4 页面置换算法

| 算法 | 策略 | 特点 |
|------|------|------|
| FIFO | 替换最早进入的页面 | 实现简单，Belady异常（增加内存反增缺页） |
| LRU | 替换最久未使用的页面 | 性能好，硬件实现开销大 |
| LFU | 替换访问次数最少的页面 | 不适应访问模式变化 |
| 时钟算法 | 近似LRU，使用位循环扫描 | 效率与开销的折中 |
| 最优置换（OPT） | 替换未来最久不用的页面 | 理论最优，仅作比较基准 |

## 六、文件系统

### 6.1 文件逻辑结构

- **顺序文件**：记录按顺序存放
- **索引文件**：按索引号快速访问
- **索引顺序文件**：结合索引和顺序访问

### 6.2 文件分配方法

| 方法 | 特点 | 缺点 |
|------|------|------|
| 连续分配 | 顺序读取快 | 外部碎片，文件扩展困难 |
| 链接分配 | 无外部碎片 | 随机存取慢 |
| FAT表 | 集中式文件分配表 | FAT表占较大空间 |
| 索引分配 | 支持直接存取 | 小文件索引块浪费 |
| Unix混合索引 | 多级索引+直接块 | 兼顾大小文件 |

### 6.3 空闲空间管理

- **位图**：bit数组表示盘块使用情况
- **空闲链表**：链接所有空闲盘块
- **成组链接法**：UNIX系统中常用

### 6.4 目录结构

- **单级目录** → **两级目录** → **树形目录**（现代OS标准）→ **无环图目录**（支持共享）

### 6.5 文件系统示例

| 文件系统 | 特点 | 最大文件 | 日志 | 压缩 |
|---------|------|---------|------|------|
| FAT32 | 兼容性好 | 4GB | 无 | 无 |
| NTFS | 功能全面 | 16EB | 是 | 支持 |
| ext4 | Linux默认 | 16TB | 是（extents） | 无 |
| APFS | macOS | 8EB | 是（快照） | 是 |

## 七、I/O管理

### 7.1 I/O控制方式

- **程序查询**：CPU轮询设备状态（忙等，浪费CPU周期）
- **中断驱动**：设备完成后发中断，CPU处理中断（高效，但频繁中断开销大）
- **DMA**：直接内存访问，无需CPU逐字节传输，适用于批量数据传输（磁盘、GPU）

### 7.2 缓冲与SPOOLing

- **单缓冲**、**双缓冲**、**循环缓冲**
- **SPOOLing**（假脱机）：用磁盘模拟独占设备，如打印机队列

### 7.3 磁盘调度

| 算法 | 策略 | 特点 |
|------|------|------|
| FCFS | 按请求顺序 | 公平但寻道时间长 |
| SSTF | 最短寻道时间优先 | 可能饥饿 |
| SCAN | 电梯算法 | 无饥饿，中间区域优先 |
| C-SCAN | 单向扫描，快速归零 | 等待时间均匀 |

## 八、Linux内核基础

Linux内核采用**单内核结构**，关键组件：
- **进程调度**：完全公平调度器（CFS），红黑树，虚拟运行时间vruntime
- **内存管理**：伙伴系统（大块分配）、slab分配器（对象缓存）、KSM（内存合并）
- **虚拟文件系统（VFS）**：统一接口，支持ext4、XFS、Btrfs等
- **IPC**：pipe、msg、shm、sem、socket、signal
- **中断**：上半部（hardirq，紧急简短）+ 下半部（softirq/tasklet/workqueue）

**系统调用流程**：用户态 → libc封装 → syscall指令 → 内核sys_call_table → 具体函数 → 返回用户态。
